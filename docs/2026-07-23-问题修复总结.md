# 2026-07-23 问题修复总结

## 一、Token 设置-登录 页面

### 1. 复制 Token 按钮报错
- **现象**：`Cannot read properties of undefined (reading 'writeText')`
- **原因**：`navigator.clipboard` 在非 HTTPS 环境（HTTP 局域网访问）下不可用
- **修复**：新增 `copyToClipboard()` 兼容函数，优先用 `navigator.clipboard`，不可用时降级到 `textarea + execCommand('copy')`

### 2. Token 管理新增"刷新 Token"按钮
- **需求**：通过 Android Telecom API 从手机 APP 自动提取最新 Token
- **实现**：
  - 新增后端接口 `POST /api/token/refresh-from-telecom`
  - 调用 `GET {apiUrl}/api/v1/audi/token` 获取手机 APP 中的 JWT Token
  - 保存到 Redis + AuthService 内存
  - 前端"📱 刷新 Token"按钮调用该接口

### 3. Token 续期逻辑改为通过 Telecom API
- **原逻辑**：定时器每 12 小时检查，Token 剩余不足 6 小时时调用 `getMemberInfo()` 假设续期成功
- **新逻辑**：改为调用 Telecom API 从手机 APP 提取最新 Token 并替换，获取真正最新的 Token

## 三、Cookie 状态面板与续期逻辑

### 10. 测试连接失败后刷新页面状态仍显示绿色
- **现象**：点"测试连接"失败后状态面板变红，但刷新页面又变回绿色
- **原因**：失败状态只在前端 DOM 修改，没有落库
- **修复**：
  - `test-xiaohongshu` / `test-zhihu` 后端接口：成功写 success 日志，失败写 failed 日志到数据库
  - 前端 `isExpired` 判断条件放宽：只要最后日志 `status=failed` 即显示红色（原来要求含"失效"关键词）

### 11. 实际搜索时发现 Cookie 失效未更新状态
- **修复**：`xiaohongshu-search.ts` 和 `zhihu-search.ts` 的 `search()` 方法中，检测到权限/鉴权类错误时自动写 failed 日志到数据库

### 12. Cookie 续期后把有效 Cookie 覆盖为无效
- **现象**：Cookie 本来有效，自动续期跑完后变成无效了
- **原因**：续期从浏览器提取的新 Cookie 没有验证有效性就直接保存 + 写 success 日志
- **修复**：续期成功后必须调 `testConnection()` / `GET /api/v4/me` 验证新 Cookie，验证通过才保存

### 13. 原 Cookie 有效 + 新 Cookie 验证失败时误标为失效
- **现象**：原 Cookie 明明有效，但浏览器环境问题导致提取的新 Cookie 验证失败，被写了 failed 日志
- **修复**：这种情况保留原 Cookie 不变 + 写 success 日志（续期失败不等于 Cookie 失效）

### 14. 续期时浏览器页面显示未登录态
- **现象**：知乎 Cookie 有效，但续期打开的浏览器页面是未登录状态
- **原因**：使用 `launchPersistentContext` 持久化目录，目录里的旧 Cookie 覆盖了注入的有效 Cookie
- **修复**：Cookie 有效分支改用普通 `chromium.launch()` + `context.addCookies()` 注入数据库 Cookie，不使用持久化目录
- **验证**：测试确认注入 Cookie 后打开知乎为登录态

### 15. "刷新 Cookie"按钮不出二维码直接报成功
- **现象**：点"刷新 Cookie"后没出二维码，直接提示成功（实际 Cookie 无效）
- **原因**：`refreshCookie()` 方法用持久化浏览器检测到"已登录"→ 提取 Cookie 保存→ 没验证有效性就报成功
- **修复**：`refreshCookie` 中检测到已登录后提取 Cookie，增加 `testConnection` 验证。验证失败则不保存，继续走扫码流程出二维码

### 16. 无法换号（持久化目录自动登录旧账号）
- **修复**：清除 `data/browser_user_data/xiaohongshu` 目录后重新扫码

## 四、服务稳定性

### 17. 启动时 TimeoutNaNWarning
- **现象**：`TimeoutNaNWarning: NaN is not a number. Timeout duration was set to 1.`
- **原因**：调度器 `executeWithRandomOffset` 中 `task.randomOffsetMin` 为 undefined，`undefined * 60000 = NaN`
- **修复**：加 `|| 0` 兜底和 `isNaN` 防御

### 18. 浏览器关闭超时后服务崩溃
- **现象**：Cookie 刷新完成后服务进程退出
- **原因**：`browser.close()` 超时后 `Promise.race` 选择了 timeout 分支，但 closePromise 后续 reject 时无人捕获，触发 `unhandledRejection` → `process.exit(1)`
- **修复**：
  - `browser.close()` 后立即 `.catch(() => {})` 吞掉潜在 reject
  - `unhandledRejection` 处理器改为只记录错误不退出进程
