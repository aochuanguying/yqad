### ⚠️ 详情功能 - 需要特定条件

#### 尝试的方案

**方案 1: 纯 API (xhshow + requests)**
- ❌ HTTP 500 错误
- 原因：需要更高权限的 Cookie

**方案 2: xhshow + Playwright**
- ❌ HTTP 500 错误
- 原因：同上

**方案 3: 纯 Playwright (网页抓取)**
- ⚠️ 页面显示"笔记暂时无法浏览"
- 原因：Cookie 权限不足或笔记状态问题

#### 问题根源

根据历史经验，获取详情需要：
1. ✅ xhshow 生成正确的签名
2. ✅ 有效的 Cookie（包含完整登录态）
3. ✅ **笔记处于可访问状态**（非私密、未删除）
4. ✅ **可能需要先浏览后访问**（防止反爬）

## 📋 五、Cookie 获取指南

### 5.1 完整步骤

1. 访问 https://www.xiaohongshu.com
2. 登录账号
3. 按 F12 打开开发者工具
4. 刷新页面
5. 在 Network 标签中找到任意请求
6. 右键 → Copy → Copy as cURL

### 5.2 必需字段

Cookie 必须包含以下字段：
- `a1` - 用于签名
- `web_session` - 会话标识
- `id_token` - 身份令牌
- 其他登录态相关字段

## ⚠️ 六、常见问题

### Q1: 详情 API 返回 500 错误？

**A**: 可能原因：
1. Cookie 权限不足
2. 笔记已删除或设为私密
3. API 端点需要其他参数
4. 需要先在浏览器中访问过该笔记

**解决方案**:
- 使用搜索 API 代替
- 尝试其他笔记 ID
- 获取更完整的 Cookie

### Q2: Playwright 访问显示"页面不见了"？

**A**: 可能原因：
1. 笔记确实不可访问
2. Cookie 无效或过期
3. 被检测到是自动化访问

**解决方案**:
- 使用真实有效的笔记 ID
- 更新 Cookie
- 添加浏览器指纹伪装

### Q3: 搜索 API 返回空结果？

**A**: 可能原因：
1. Cookie 无效
2. 关键词太冷门
3. page_size < 10

**解决方案**:
- 验证 Cookie 有效性
- 更换热门关键词
- 确保 page_size >= 10
