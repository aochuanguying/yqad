# 多平台优化数据库迁移说明

本文档详细说明了多平台发帖优化功能所需的数据库迁移步骤。

## 迁移概述

本次迁移为 `internet_reference_config` 和 `internet_reference_platforms` 表添加多平台支持字段，包括：
- 平台区分字段
- 优先级配置
- 频率限制
- 成功率统计

## 迁移前准备

### 1. 备份数据库

```bash
# 备份整个数据库
mysqldump -u [用户名] -p[密码] [数据库名] > backup_$(date +%Y%m%d_%H%M%S).sql

# 或只备份相关表
mysqldump -u [用户名] -p[密码] [数据库名] \
  internet_reference_config \
  internet_reference_platforms \
  > backup_tables_$(date +%Y%m%d_%H%M%S).sql
```

### 2. 检查表结构

```sql
-- 查看当前表结构
DESCRIBE internet_reference_config;
DESCRIBE internet_reference_platforms;

-- 查看现有数据
SELECT * FROM internet_reference_config;
SELECT * FROM internet_reference_platforms;
```

## 迁移步骤

### 步骤 1：执行数据库迁移脚本

迁移脚本位置：`scripts/db/migrations/001_add_platform_support.sql`

```bash

mysql -u [用户名] -p[密码] [数据库名] < scripts/db/migrations/001_add_platform_support.sql

mysql -u [用户名] -p[密码]
```

## 迁移脚本详解

### 1. 添加 platform 字段到 internet_reference_config

```sql
ALTER TABLE internet_reference_config 
ADD COLUMN platform VARCHAR(20) NOT NULL DEFAULT 'all' 
COMMENT '平台：xiaohongshu|zhihu|autohome|all';
```

**说明**：
- `VARCHAR(20)`: 存储平台标识符
- `NOT NULL DEFAULT 'all'`: 默认为通用配置
- 支持的值：`xiaohongshu`, `zhihu`, `autohome`, `all`

### 2. 添加优先级等字段到 internet_reference_platforms

```sql
ALTER TABLE internet_reference_platforms
ADD COLUMN priority INT DEFAULT 5 COMMENT '优先级 1-10',
ADD COLUMN rate_limit_per_hour INT DEFAULT 50 COMMENT '每小时频率限制',
ADD COLUMN success_rate DECIMAL(5,2) DEFAULT 100.00 COMMENT '历史成功率（%）';
```

**说明**：
- `priority`: 平台优先级，范围 1-10，数值越大优先级越高
- `rate_limit_per_hour`: 每小时最大查询次数
- `success_rate`: 历史成功率，用于智能选择算法

### 3. 初始化三个平台配置

```sql
-- 插入搜索词配置
INSERT INTO internet_reference_config (platform, search_keywords, rate_limit_per_hour) VALUES
  ('xiaohongshu', '奥迪 Q5L，奥迪 A4L，奥迪上下班，奥迪自驾游，奥迪真香', 20),
  ('zhihu', '如何评价奥迪 Q5L，奥迪 Q5L 值得购买吗，奥迪 Q5L vs 宝马 X3', 100),
  ('autohome', '提车，油耗，改装，异响，保养，Q5L,A4L,A6L', 50);

-- 插入平台优先级配置
INSERT INTO internet_reference_platforms (platform_name, platform_display, priority, rate_limit_per_hour) VALUES
  ('xiaohongshu', '小红书', 8, 20),
  ('zhihu', '知乎', 7, 100),
  ('autohome', '汽车之家', 8, 50);
```

|------|-----------|----------|------|

|------|--------|----------|------|

| 知乎 | 7 | 100 次/小时 | 中等优先级，频率限制宽松 |

## 回滚方案

### 回滚步骤 1：清除新增数据

```sql
-- 删除新增的平台配置
DELETE FROM internet_reference_config 
WHERE platform IN ('xiaohongshu', 'zhihu', 'autohome');

-- 删除新增的平台优先级
DELETE FROM internet_reference_platforms 
WHERE platform_name IN ('xiaohongshu', 'zhihu', 'autohome');
```

### 回滚步骤 2：删除新增字段

```sql
-- 删除 internet_reference_config 的 platform 字段
ALTER TABLE internet_reference_config 
DROP COLUMN platform;

-- 删除 internet_reference_platforms 的新增字段
ALTER TABLE internet_reference_platforms 
DROP COLUMN priority,
DROP COLUMN rate_limit_per_hour,
DROP COLUMN success_rate;
```

### 回滚步骤 3：恢复备份（如果需要）

```bash
# 恢复备份数据
mysql -u [用户名] -p[密码] [数据库名] < backup_YYYYMMDD_HHMMSS.sql
```

## 常见问题

### Q1: 迁移失败，提示字段已存在

**解决方案**：
```sql
-- 检查字段是否已存在
SELECT COLUMN_NAME 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = 'internet_reference_config' 
  AND COLUMN_NAME = 'platform';

-- 如果已存在但需要修改
ALTER TABLE internet_reference_config 
MODIFY COLUMN platform VARCHAR(20) NOT NULL DEFAULT 'all';
```

### Q2: 数据插入失败，提示主键冲突

**解决方案**：
```sql
-- 检查是否已存在数据
SELECT * FROM internet_reference_platforms 
WHERE platform_name IN ('xiaohongshu', 'zhihu', 'autohome');

-- 如果已存在，使用 UPDATE 而不是 INSERT
UPDATE internet_reference_platforms 
SET priority = 8, rate_limit_per_hour = 20 
WHERE platform_name = 'xiaohongshu';
```

### Q3: Redis 缓存未清除

**解决方案**：
```bash
# 手动清除 Redis 缓存
redis-cli
> KEYS internet_ref:*
> DEL internet_ref:platform_config
> DEL internet_ref:platform_priority
> DEL internet_ref:search_keywords:*
```
