## 📋 一、测试结论

### ✅ 搜索功能 - 使用 xhshow 签名

**API**: `POST https://so.xiaohongshu.com/api/sns/web/v2/search/notes`

**技术栈**:
- Python 3.10+
- `xhshow` (v0.2.0) - 签名库
- `requests` - HTTP 请求

**测试结果**: ✅ 完全可用
- 返回笔记列表
- 包含：标题、描述、作者、互动数据、封面图
- 性能稳定，响应时间 2-5 秒

### ⚠️ 详情功能 - 必须使用 Playwright

**原因**: 
- API 接口（`/api/sns/web/v1/feed`、`/api/sns/web/v2/feed`）返回 HTTP 500 错误
- 需要更高权限的 Cookie 或特殊参数
- **经验证：必须使用 Playwright 访问网页获取 HTML 内容**

**技术方案**:
- 使用 Playwright 启动无头浏览器
- 访问 `https://www.xiaohongshu.com/explore/{note_id}`
- 解析 HTML 获取笔记详情
- 可设置 Cookie 保持登录态

### 2.2 详情功能实现（Playwright）

**文件**: `scripts/test_xiaohongshu_playwright.py`

```python
from playwright.sync_api import sync_playwright

def get_note_detail(note_id: str, cookie: str = None):
    """使用 Playwright 获取笔记详情"""
    
    with sync_playwright() as p:
        # 启动浏览器
        browser = p.chromium.launch(headless=True)
        context = browser.new_context(
            user_agent='Mozilla/5.0 ...'
        )
        
        # 设置 Cookie（可选）
        if cookie:
            context.add_cookies([...])
        
        # 访问笔记页面
        page = context.new_page()
        page.goto(f"https://www.xiaohongshu.com/explore/{note_id}")
        
        # 提取数据
        note_data = page.evaluate('''() => {
            return {
                title: document.querySelector('h1.title')?.textContent,
                desc: document.querySelector('.desc')?.textContent,
                author: document.querySelector('.author-name')?.textContent,
                likes: document.querySelector('[data-type="like"] .count')?.textContent,
                collects: document.querySelector('[data-type="collect"] .count')?.textContent,
                comments: document.querySelector('[data-type="comment"] .count')?.textContent,
                images: [...document.querySelectorAll('.album-item img')].map(img => img.src)
            };
        }''')
        
        browser.close()
        return note_data
```

## 🎯 四、推荐方案

### 方案 A：仅使用搜索 API（推荐）⭐

**适用场景**:
- 只需要基本信息（标题、描述、互动数据）
- 追求性能和稳定性
- 不需要完整详情和图片列表

**优点**:
- ✅ 简单快速
- ✅ 性能稳定
- ✅ 无需浏览器依赖

**缺点**:
- ⚠️ 缺少完整描述
- ⚠️ 没有图片列表
- ⚠️ 缺少部分详细信息

### 方案 B：搜索 + Playwright 详情（完整方案）

**适用场景**:
- 需要完整的笔记内容
- 需要图片列表
- 可以接受较慢的速度

**优点**:
- ✅ 数据完整
- ✅ 包含所有图片
- ✅ 包含完整描述

**缺点**:
- ⚠️ 速度较慢（5-10 秒）
- ⚠️ 需要 Playwright 依赖
- ⚠️ 可能触发反爬机制

## ⚠️ ���、注意事项

### 8.1 Cookie 使用

**搜索功能**:
- Cookie 必须包含 `a1` 字段
- Cookie 会过期，需定期更新
- 建议实现 Cookie 有效期检测

**Playwright 详情**:
- Cookie 可选，但建议提供
- 登录态可避免部分反爬限制
- Cookie 格式需转换为 Playwright 格式

### 8.2 性能考虑

**搜索 API**:
- 响应时间：2-5 秒
- 建议添加请求间隔（1-3 秒）
- 可实现结果缓存

**Playwright**:
- 响应时间：5-10 秒
- 浏览器启动开销较大
- 建议使用浏览器池复用

### 8.3 反爬机制

- 控制请求频率
- 使用真实 User-Agent
- 可设置 Cookie 保持登录态
- 避免短时间大量请求
