## 二、三个搜索平台

### 2.1 小红书（XiaohongshuSearch）

**技术方案：** Python xhshow 库（mns0301 签名算法）直接调小红书 Web API

**调用链：**
```
search(keywords, maxResults)
  → ensureCookieLoaded()      // 从数据库加载最新 Cookie（5 分钟缓存）
  → checkRateLimit()          // 滑动窗口频率限制（5 次/小时）
  → requestDelay()            // 随机延迟 8-15 秒（防风控）
  → searchViaPython()         // spawn Python 脚本
```

**Python 脚本逻辑：**
- 使用 xhshow 库生成 XYS_ 格式签名
- POST `https://so.xiaohongshu.com/api/sns/web/v2/search/notes`
- 返回笔记列表（标题、描述、作者、互动数据、封面图、xsec_token）

**Cookie 来源：** 数据库 `network_post_config.xiaohongshu_cookie`

**搜索词策略：** 从预配置词库按小时轮换，避免频繁更换触发风控

---

### 2.2 知乎（ZhihuSearch）

**技术方案：** 知乎开放平台官方 API（搜索）+ Python Playwright（正文提取）

**调用链：**
```
search(keywords, maxResults)
  → loadZhihuConfig()                    // 从数据库加载配置（5 分钟缓存）
  → searchViaApiWithContent()
      → searchViaApiOnly()               // 步骤 1：调知乎 API 获取搜索结果
      → fetchContentViaPython()          // 步骤 2：Playwright 提取正文和图片
```

**步骤 1 - 搜索 API：**
- GET `https://developer.zhihu.com/api/v1/content/zhihu_search`
- Headers: `Authorization: Bearer {accessSecret}`
- 返回文章/回答列表（标题、URL、摘要）

**步骤 2 - 正文提取：**
- Python 脚本 `test_zhihu_content.py` 通过 stdin 接收数据
- Playwright 打开每个 URL，提取正文和图片
- Cookie 用于绕过安全验证
- 超时 90 秒，失败时降级为仅 API 结果（无正文）

**配置来源：**
- Access Secret: `network_post_config.zhihu_access_secret`
- Cookie: `network_post_config.zhihu_cookie`

**搜索词策略：** 优先选含"如何/评价/对比"的专业问句，否则选最长关键词

---

### 2.3 汽车之家（AutohomeSearch）

**技术方案：** Python Playwright 访问搜索页面，提取论坛帖子

**调用链：**
```
search(keywords, maxResults)
  → checkRateLimit()          // 滑动窗口频率限制（30 次/小时）
  → searchDelay()             // 随机延迟 2-5 秒
  → searchViaPython()         // spawn Python 脚本（带 --fetch-content）
```

**Python 脚本逻辑（test_autohome.py）：**
- Playwright 打开 `https://sou.autohome.com.cn?q={keyword}`
- 等待 5 秒让 SPA 渲染完成
- 使用选择器 `a[href*="club.autohome.com.cn/bbs"]` 提取论坛帖子链接
- 从卡片文本中正则提取回复数、浏览数、时间
- 如果带 `--fetch-content`，打开帖子 URL 提取正文

**无需 Cookie，使用搜索页面公开接口**

**搜索词策略：** 优先选含"提车/作业/用车"等论坛术语的词；无论坛术语时自动拼接后缀（每小时轮换：提车/用车/作业/油耗/保养）

## 三、Cookie 管理

### 3.1 三条获取路径

| 路径 | 触发方式 | 实现 |
|------|---------|------|
| 手动设置 | Web 页面输入 Cookie → 保存 | `POST /api/network-post-config` |
| 扫码设置 | Web 页面点"刷新 Cookie" | Playwright 生成二维码 → 等待扫码 → 提取 Cookie |
| 自动刷新 | 定时任务（cron 配置） | `smartRefreshCookie()` |

### 3.2 自动刷新逻辑（smartRefreshCookie）

```
定时任务触发
    │
    ├── Step 1: API 测试 Cookie 字符串是否有效
    │       ├── 有效 → 启动浏览器 → 刷新续期 → 保存 → 成功 ✅
    │       └── 失效 → 进入 Step 2
    │
    ├── Step 2: 启动浏览器（加载持久化目录），检查 session
    │       ├── 已登录 → 刷新续期 → 提取新 Cookie → 保存 → 成功 ✅
    │       └── 未登录 → 进入 Step 3
    │
    └── Step 3: API 和浏览器 session 都失效
            → 记录失败日志 → 标记"需要手动刷新"
            → 用户在 Web 页面看到"已过期"状态
```

### 3.3 Cookie 状态判断

前端通过 `GET /api/network-post-config/cookie/status/{platform}` 获取状态：
- `isValid = hasCookie && 最近48小时内刷新过 && 最后日志不是failed`
- `expired = true` 时显示红色"已过期"提示

## 四、搜索管理器 - 平台选择策略

### 4.1 选择算法

```
selectNextPlatform()
  → calculateBasePriorities()      // 数据库配置的基础优先级
  → adjustByRateLimit()            // 使用率 >80% 优先级 -3，>50% 优先级 -1
  → adjustBySuccessRate()          // 成功率 >90% 优先级 +1，<50% 优先级 -2
  → weightedRandomSelect()         // 加权随机 + 最近使用惩罚 -3
```

### 4.2 降级策略

当选中的平台搜索失败（throw Error）时：
- 搜索管理器捕获异常
- 遍历其他可用平台尝试搜索
- 全部失败则返回空结果

### 4.3 频率限制

| 层级 | 实现 | 说明 |
|------|------|------|
| 外层（入口） | Redis `searchRateLimitStorage` | 全局频率（如 10 次/小时） |
| 内层（平台） | Redis `incrementQueryCount(platform)` | 每平台独立计数 |
| 平台内部 | 滑动窗口 `requestTimestamps[]` | 进程内存，精确控制 |

## 五、发帖消费流程

### 5.1 自由发帖模式（tryInternetReferenceMode）

```
1. canQuery() — 检查频率限制（Redis）
2. search() — 通过搜索管理器获取参考素材
3. AI 改写 — 基于参考素材生成原创内容
4. 抄袭检测 — detectPlagiarism() 对比相似度
5. 合规性检查 — complianceCheckOrchestrator
6. 发布 — 通过 AutoJS 远程执行
```

### 5.2 参考素材格式（SearchResult）

```typescript
interface SearchResult {
  title: string;       // 标题
  content: string;     // 正文内容
  source: string;      // 来源平台
  imageUrls?: string[]; // 图片 URL 列表
  url?: string;        // 原文链接
  author?: string;     // 作者
  likes?: number;      // 点赞/浏览数
  comments?: number;   // 评论/回复数
  xsecToken?: string;  // 小红书详情 token
}
```

## 六、Web 页面功能

### 6.1 测试连接

| 平台 | 验证内容 | 调用方式 |
|------|---------|---------|
| 知乎 | API 搜索 + Cookie 有效性 | `testZhihuConnection(accessSecret)` + `https /api/v4/me` |
| 小红书 | Cookie → xhshow 搜索 | `testXiaohongshuConnection(cookie)` → `XiaohongshuSearch.testConnection()` |
| 汽车之家 | Playwright 搜索 | `testAutohomeConnection()` → `test_autohome.py '奥迪Q5L提车' 5` |

### 6.2 刷新 Cookie（扫码）

- 异步任务模式：前端发起 → 返回 taskId → 轮询状态
- 两个平台各自独立的任务 Map（带 10 分钟自动清理）
- 并发控制：`isRefreshing` 锁防止同时启动多个浏览器

## 七、关键配置

| 配置项 | 存储位置 | 说明 |
|--------|---------|------|
| 搜索关键词 | `internet_reference_config.search_keywords` | JSON 数组 |
| 最大结果数 | `internet_reference_config.max_results` | 默认 5 |
| 频率限制 | `internet_reference_config.rate_limit_per_hour` | 默认 10 |
| 平台优先级 | `internet_reference_platforms.priority` | 1-10 |
| 平台启用 | `internet_reference_platforms.enabled` | 布尔值 |
| Cookie 刷新定时 | `scheduler_config.cookie_refresh_cron` | 默认 `0 2 * * *` |
