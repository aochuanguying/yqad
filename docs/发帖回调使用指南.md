## 历史回顾

本文档记录的是已废弃的直接 API 发帖方式的回调流程。

### 旧流程（已废弃）

1. 调用 `POST /api/posts/generate` 生成内容
2. 手动或脚本调用奥迪 API 发布
3. 回调 `POST /api/posts/confirm` 更新日志

### 当前流程（推荐）

使用 AutoJS 远程发帖：

1. **配置 AutoJS API**
   - 在 Web 管理界面配置 AutoJS 服务器地址
   - 配置发帖 API Token

2. **触发发帖**
   - 点击"立即发帖"按钮
   - 调用 `POST /api/posts/execute`

3. **脚本执行**
   - AutoJS 脚本在手机上运行
   - 使用真实 APP 发帖

4. **自动回调**
   - 脚本自动回调服务端
   - 更新 `post_logs` 状态

详细使用：[AutoJS API 使用说明](./AUTOJS_API_USAGE.md)

---

## 为什么废弃直接 API 发帖？

1. **容易被识别**：直接 API 调用容��被服务器识别为非真实设备
2. **验证码复杂**：需要生成 vrfCode 等验证码
3. **稳定性差**：API 参数变化可能导致发帖失败
4. **有更好的方案**：AutoJS 方式使用真实 APP，完全模拟人工操作

---

## 当前发帖方式对比

| 特性 | 直接 API 发帖（废弃） | AutoJS 远程发帖（推荐） |
|------|---------------------|------------------------|
| 实现方式 | 直接调用奥迪 API | AutoJS 脚本在手机上执行 |
| 验证码 | 需要 vrfCode | 不需要（APP 自动处理） |
| 稳定性 | 较差 | 高 |
| 维护成本 | 高 | 低 |
| 推荐度 | ❌ 不推荐 | ✅ 强烈推荐 |

---

## 总结

**不要使用直接 API 发帖方式**，请使用 AutoJS 远程发帖。
