## 2. 部署步骤

### 步骤 1: 数据库迁移

#### 1.1 备份数据库

```bash
# 备份整个数据库
mysqldump -u [用户名] -p[密码] [数据库名] > backup_$(date +%Y%m%d_%H%M%S).sql

# 备份相关表
mysqldump -u [用户名] -p[密码] [数据库名] \
  internet_reference_config \
  internet_reference_platforms \
  > backup_tables_$(date +%Y%m%d_%H%M%S).sql
```

#### 1.2 执行迁移脚本

```bash
# 进入项目目录
cd /Users/mac/Documents/workspace/krio/yqad

# 执行迁移脚本
mysql -u [用户名] -p[密码] [数据库名] < scripts/db/migrations/001_add_platform_support.sql
```

#### 1.3 验证迁移结果

```bash
# 连接 MySQL
mysql -u [用户名] -p[密码] [数据库名]

# 执行验证 SQL
source scripts/verify-migration.sql;
```

验证 SQL 内容（`scripts/verify-migration.sql`）：
```sql
-- 检查表结构
DESCRIBE internet_reference_config;
DESCRIBE internet_reference_platforms;

-- 检查数据
SELECT platform, search_keywords FROM internet_reference_config;
SELECT platform_name, priority, rate_limit_per_hour, success_rate FROM internet_reference_platforms;

-- 检查数据完整性
SELECT COUNT(*) as count FROM internet_reference_platforms WHERE platform_name IN ('xiaohongshu', 'zhihu', 'autohome');
```

## 4. 回滚方案

### 4.1 回滚触发条件

出现以下情况时应该立即回滚：

1. **严重 Bug**: 发帖功能完全不可用
2. **性能问题**: API 响应时间 > 500ms
3. **数据错误**: 平台配置数据丢失或错误
4. **缓存故障**: Redis 连接失败或缓存雪崩

### 4.2 回滚步骤

#### 步骤 1: 停止服务

```bash
# 停止 PM2 服务
pm2 stop yqad

# 或者停止 Node 进程
pkill -f "node dist/index.js"
```

#### 步骤 2: 代码回滚

```bash
cd /Users/mac/Documents/workspace/krio/yqad

# 回滚到上一个版本
git reset --hard HEAD~1

# 或者回滚到指定版本
git reset --hard [commit-hash]

# 重新编译
npm run build
```

#### 步骤 3: 数据库回滚

##### 方案 A: 使用备份恢复（推荐）

```bash
# 恢复备份
mysql -u [用户名] -p[密码] [数据库名] < backup_YYYYMMDD_HHMMSS.sql
```

##### 方案 B: 手动回滚 SQL

```bash
mysql -u [用户名] -p[密码] [数据库名]
```

执行回滚 SQL：

```sql
-- 删除新增数据
DELETE FROM internet_reference_config 
WHERE platform IN ('xiaohongshu', 'zhihu', 'autohome');

DELETE FROM internet_reference_platforms 
WHERE platform_name IN ('xiaohongshu', 'zhihu', 'autohome');

-- 删除新增字段
ALTER TABLE internet_reference_config 
DROP COLUMN platform;

ALTER TABLE internet_reference_platforms 
DROP COLUMN priority,
DROP COLUMN rate_limit_per_hour,
DROP COLUMN success_rate;
```

#### 步骤 4: 清除 Redis 缓存

```bash
redis-cli
> KEYS internet_ref:*
> DEL internet_ref:platform_config
> DEL internet_ref:platform_priority
> DEL internet_ref:search_keywords:*
> QUIT
```

#### 步骤 5: 重启服务

```bash
# 启动服务
pm2 start yqad

# 或者
npm start

# 检查服务状态
pm2 status
pm2 logs yqad
```

### 4.3 回滚验证

回滚后需要验证：

```bash
# 1. 检查服务是否正常启动
pm2 status

# 2. 检查数据库结构
mysql -u [用户名] -p[密码] [数据库名] -e "DESCRIBE internet_reference_config;"

# 3. 检查基本发帖功能
```

## 5. 故障排查

### 5.1 常见问题

#### 问题 1: 数据库迁移失败

**错误信息**: `Table 'internet_reference_config' already exists`

**解决方案**:
```sql
-- 检查表是否已存在
SHOW TABLES LIKE 'internet_reference_config';

-- 如果已存在，检查字段
DESCRIBE internet_reference_config;

-- 如果字段已存在，跳过迁移或使用 ALTER TABLE MODIFY
```

#### 问题 2: Redis 连接失败

**错误信息**: `Redis connection failed`

**解决方案**:
```bash
# 检查 Redis 服务状态
systemctl status redis

# 测试 Redis 连接
redis-cli ping

# 检查配置文件中的 Redis 连接信息
cat config/default.yaml | grep -A 5 "redis:"
```

#### 问题 3: 缓存命中率低

**现象**: 命中率 < 50%

**解决方案**:
```typescript
// 检查缓存是否正常工作
import { internetReferenceCache } from './src/storage/redis/internet-reference-cache';

const stats = await internetReferenceCache.getStatistics();
console.log('缓存统计:', stats);

// 如果命中率低，检查：
// 1. Redis 服务是否正常
// 2. 缓存键是否正确
// 3. 缓存过期时间是否合理
```

#### 问题 4: 平台选择算法异常

**现象**: 总是选择同一个平台

**解决方案**:
```typescript
// 检查平台优先级
const priorities = await internetReferenceStorage.getPlatformPriorities();
console.log('优先级:', priorities);

// 检查成功率
const configs = await internetReferenceStorage.getAllPlatformConfigs();
console.log('成功率:', configs.map(c => ({
  platform: c.platformName,
  successRate: c.successRate
})));

// 如果数据异常，重置配置
await internetReferenceStorage.updatePlatformPriority('xiaohongshu', 8);
await internetReferenceStorage.updatePlatformPriority('zhihu', 7);
await internetReferenceStorage.updatePlatformPriority('autohome', 8);
```
