### 2. 重试机制 ✅

**优化前**:
- 无重试机制
- 网络波动导致失败

**优化后**:
- 自动重试 2 次
- 指数退避延迟（2 秒）
- 详细的重试日志

**实现代码**:
```python
async def fetch_post_content_with_retry(url, use_new_selector=True, max_retries=2):
    """带重试机制的正文获取函数"""
    retry_delay = 2  # 重试延迟（秒）
    
    for attempt in range(max_retries + 1):
        try:
            result = await fetch_post_content(url, use_new_selector)
            if result[1]:  # content 不为空
                return result
            
            if attempt < max_retries:
                print(f"⚠️ 第 {attempt + 1} 次尝试未获取到内容，{retry_delay}秒后重试...")
                await asyncio.sleep(retry_delay)
                
        except Exception as e:
            if attempt < max_retries:
                print(f"⚠️ 第 {attempt + 1} 次尝试失败：{e}，{retry_delay}秒后重试...")
                await asyncio.sleep(retry_delay)
            else:
                print(f"❌ 所有 {max_retries + 1} 次尝试均失败")
                return "", "", []
    
    return "", "", []
```

**重试策略**:
- 最大重试次数：2 次
- 重试延迟：2 秒
- 触发条件：
  - 网络异常
  - 内容为空
  - 选择器失效

### 3. 页面结构监控 ✅

**优化前**:
- 选择器失效时无提示
- 难以发现页面结构变更

**优化后**:
- 实时监控选择器有效性
- 自动 fallback 到其他选择器
- 详细日志记录

**实现代码**:
```python
# 主选择器
content_element = await page.query_selector(".fn-main .post")

if content_element:
    selector_used = ".fn-main .post"
    # 提取内容...
else:
    # ⚠️ 页面结构监控：记录警告
    print(f"⚠️ 警告：选择器 '.fn-main .post' 未找到元素，URL: {url}")
    print(f"   可能页面结构已变更，尝试 fallback 选择器...")
    
    # Fallback 选择器列表
    fallback_selectors = [
        ".post",
        ".thread-content",
        ".main-content",
        "article",
        "div.content"
    ]
    
    for fallback_selector in fallback_selectors:
        content_element = await page.query_selector(fallback_selector)
        if content_element:
            selector_used = f"fallback: {fallback_selector}"
            print(f"✓ 使用 fallback 选择器：'{fallback_selector}'")
            break
```

**Fallback 选择器**:
1. `.post` - 通用帖子选择器
2. `.thread-content` - 主题内容选择器
3. `.main-content` - 主内容选择器
4. `article` - HTML5 文章标签
5. `div.content` - 通用内容容器

**监控日志示例**:

## 🎓 最佳实践

### 1. 并发控制

```python
# 使用信号量控制并发数
semaphore = asyncio.Semaphore(max_concurrent)

async with semaphore:
    # 并发执行
```

**优点**:
- 避免资源耗尽
- 控制内存占用
- 平衡性能和稳定性

### 2. 重试策略

```python
# 指数退避
for attempt in range(max_retries):
    try:
        return await operation()
    except Exception:
        await asyncio.sleep(retry_delay * (2 ** attempt))
```

**优点**:
- 处理临时故障
- 避免雪崩效应
- 提高成功率

### 3. Fallback 机制

```python
# 多层 fallback
selectors = [".primary", ".secondary", ".tertiary"]
for selector in selectors:
    if await page.query_selector(selector):
        break
```

**优点**:
- 提高容错性
- 延长维护周期
- 降低紧急修复需求
