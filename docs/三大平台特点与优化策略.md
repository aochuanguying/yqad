## 小红书平台详解

### 1. 平台特点

#### 1.1 内容生态

**核心特征**：
- **种草文化**：用户分享真实使用体验，容易形成购买决策
- **精致生活**：内容追求美感、品质、仪式感
- **女性视角**：70%+ 用户为女性，注重细节和情感共鸣
- **年轻化**：90 后、00 后为主，接受新事物快

**内容调性**：
```
✅ 真实体验分享
✅ 精美图片 + 简短文字
✅ 情绪化表达（绝绝子、真香、踩雷）
✅ 实用干货（攻略、测评、对比）
❌ 硬广推销
❌ 过于专业化
❌ 长篇大论
```

#### 1.2 汽车内容特点

**热门话题**：
- 提车日记（仪式感、交付体验）
- 用车心得（油耗、空间、驾驶感受）
- 改装装饰（内饰、外观、实用配件）
- 自驾游记（风景、路线、拍照打卡）
- 对比评测（同价位车型对比）

**内容结构**：
```
标题：[emoji] 核心卖点 + 情绪表达
正文：
- 个人情况介绍（车型、配置、购车时间）
- 使用场景（上下班、接送孩子、自驾游）
- 优点列举（3-5 点，配精美图片）
- 小缺点（增加真实性）
- 总结推荐
标签：#奥迪 Q5L #豪华 SUV #用车日常
```

#### 1.3 用户行为

**浏览习惯**：
- 快速滑动，被封面图吸引后停留
- 平均阅读时间 30-60 秒
- 喜欢收藏"有用"的内容
- 容易被"同款"、"推荐"影响

**互动特点**：
- 点赞门槛低（内容好就赞）
- 收藏率高（认为以后用得上）
- 评论询问细节（"落地价多少？"、"油耗多少？"）

### 2. 技术实现

**文件位置**: [`src/services/internet-search/xiaohongshu-search.ts`](file:///Users/mac/Documents/workspace/krio/yqad/src/services/internet-search/xiaohongshu-search.ts)

**核心特点**：
- 使用 `xhshow` Python 库生成签名

- 需要 Cookie 中的 `a1` 值用于签名

**API 端点**：
```
POST https://so.xiaohongshu.com/api/sns/web/v2/search/notes
Headers:
  - x-s: XYS_签名
  - x-t: 时间戳
  - Cookie: 包含 a1
```

**返回数据结构**：
```typescript
interface XiaohongshuNote {

    url: string;           // 封面图 URL
  };

```

### 3. 优势与劣势

**优势**：
- ✅ 内容真实、生活化，容易引发共鸣
- ✅ 图片质量高，适合视觉展示
- ✅ 用户群体与奥迪目标客户重叠（年轻、追求品质）
- ✅ 种草效应强，容易形成购买决策

**劣势**：
- ❌ 内容深度有限，不适合技术分析
- ❌ 需要 Cookie 和签名，维护成本高
- ❌ 频率控制严格，容易触发风控

## 知乎平台详解

### 1. 平台特点

#### 1.1 内容生态

**核心特征**：
- **专业背书**：各领域专家、从业者分享专业知识
- **理性分析**：基于数据、逻辑、经验的分析
- **深度内容**：长文、系列回答、专栏文章
- **问答驱动**：围绕问题展开，针对性强

**内容调性**：
```
✅ 专业术语、数据分析
✅ 逻辑清晰、论证充分
✅ 图文并茂（图表、数据图）
✅ 引用来源、案例支撑
❌ 情绪化表达
❌ 缺乏依据的主观判断
❌ 过于营销化
```

#### 1.2 汽车内容特点

**热门话题**：
- 车型对比（同级别车型深度对比）
- 技术解析（发动机、变速箱、底盘技术）
- 购车建议（预算内最佳选择）
- 用车成本分析（油耗、保养、保险）
- 行业趋势（新能源、智能化）

**内容结构**：
```
标题：如何评价 XXX？/ XXX 和 XXX 怎么选？
正文：
- 背景介绍（市场定位、竞品分析）
- 技术参数对比（表格、数据）
- 实际体验（驾驶感受、空间表现）
- 优缺点分析（客观中立）
- 购买建议（适合人群、配置推荐）
- 总结
```

#### 1.3 用户行为

**浏览习惯**：
- 愿意花时间阅读长文（5-10 分钟）
- 关注作者背景（认证信息、专业领域）
- 看重赞同数和评论质量
- 喜欢收藏"干货"内容

**互动特点**：
- 赞同/反对（专业认可）
- 专业讨论（补充、质疑、辩论）
- 追问细节（"请问 XXX 怎么样？"）

### 2. 技术实现

**文件位置**: [`src/services/internet-search/zhihu-search.ts`](file:///Users/mac/Documents/workspace/krio/yqad/src/services/internet-search/zhihu-search.ts)

**核心特点**：
- 使用知乎官方开放平台 API
- 需要 Access Secret（从知乎开放平台申请）
- 每日免费 1000 次调用
- 返回数据包含权威等级、排序分数

**API 端点**：
```
GET https://developer.zhihu.com/api/v1/content/zhihu_search
Headers:
  - Authorization: Bearer {ACCESS_SECRET}
  - X-Request-Timestamp: {timestamp}
Params:
  - Query: 搜索关键词
  - Count: 结果数量（最大 10）
```

**��回数据结构**：
```typescript
interface ZhihuResult {
  Code: number;           // 0 表示成功
  Message: string;
  Data: {
    HasMore: boolean;
    SearchHashId: string;
    Items: [{
      Title: string;
      ContentText: string;
      AuthorName: string;
      AuthorBadge: string;     // 专业认证
      VoteUpCount: number;     // 赞同数
      CommentCount: number;
      AuthorityLevel: string;  // 权威等级
      RankingScore: number;    // 排序分数
      Url: string;
    }]
  };
}
```

### 3. 优势与劣势

**优势**：
- ✅ 内容专业、深度，适合技术分析
- ✅ 官方 API，稳定可靠
- ✅ 用户群体专业，内容质量高
- ✅ 权威等级、排序分数可用于质量筛选

**劣势**：
- ❌ 内容过于专业，可能不适合日常分享
- ❌ 每日调用次数有限（1000 次/天）
- ❌ 需要申请 Access Secret
- ❌ 图片质量一般

## 分平台搜索词优化策略

### ⚠️ 技术实现差异对搜索词的影响

**重要提示**：由于三个平台的技术实现方式完全不同，搜索词的选择必须考虑技术限制！

| 平台 | 技术实现 | 搜索机制 | 搜索词选择要点 |
|------|----------|----------|----------------|
| **小红书** | 官方 API（签名算法） | 全文搜索（标题 + 描述 + 标签） | ✅ 支持模糊匹配<br>✅ 支持多关键词组合<br>⚠️ 需要避免敏感词（触发风控）<br>⚠️ 签名算法限制，不能频繁更换搜索词 |
| **知乎** | 官方 API（标准 RESTful） | 问题/文章标题匹配 | ✅ 支持问题形式搜索<br>✅ 支持专业术语<br>✅ 稳定性最好，搜索词选择最自由 |
| **汽车之家** | 爬虫（HTML 解析） | 论坛帖子标题过滤 | ⚠️ 只能匹配帖子标题<br>⚠️ 需要精确匹配（不支持模糊）<br>❌ 不支持多关键词组合<br>⚠️ 受 HTML 结构影响大 |

### 1. 小红书搜索词优化

#### 1.1 搜索词特点

**技术限制下的搜索词选择**：

由于小红书使用**官方 API + 签名算法**，搜索词选择需要注意：

1. **签名算法限制**：
   - 每次搜索都需要生成 XYS_签名
   - 频繁更换搜索词会触发风控
   - **建议**：预先配置好搜索词库，每次从中选择，而不是动态生成

2. **全文搜索机制**：
   - API 会搜索笔记的标题、描述、标签
   - **建议**：使用"车型 + 场景"组合，命中率高
   - 示例：`"奥迪 Q5L 自驾游"` 比单独搜 `"自驾游"` 更好

3. **敏感词过滤**：
   - 签名算法会检查搜索词是否包含敏感内容
   - **避免**：广告词、营销词、政治敏感词
   - **推荐**：生活化、场景化、中性词

小红书用户搜索习惯：
- 喜欢用**场景词**（上下班、自驾游、接送孩子）
- 喜欢用**感受词**（真香、绝绝子、踩雷）
- 喜欢用**标签词**（#奥迪 Q5L #豪华 SUV）
- 搜索词短小精悍，2-5 个字为主

#### 1.2 推荐搜索词库

**车型相关**：
```

奥迪 A4L
奥迪 A6L
奥迪 e-tron

```

**场景相关**：
```

奥迪自驾游
奥迪接送孩子
奥迪露营
奥迪长途驾驶
```

**感受相关**：
```
奥迪真香
奥迪油耗
奥迪空间

奥迪内饰
```

**对比相关**：
```

奥迪 vs 宝马
```

#### 1.3 搜索词组合策略

**策略**：`车型 + 场景 + 感受`

示例：
```
✅ "奥迪 Q5L 自驾游"
✅ "奥迪 A4L 上下班 油耗"
✅ "奥迪 Q5L 接送孩子 空间"
✅ "奥迪 e-tron 露营 真香"
```

#### 1.4 搜索词更新频率

- **每周更新**：根据热点事件、季节变化
- **每月分析**：分析搜索效果，优化词库
- **季度调整**：根据车型改款、新车发布调整

### 2. 知乎搜索词优化

#### 2.1 搜索词特点

**技术限制下的搜索词选择**：

由于知乎使用**官方开放平台 API**，搜索词选择需要注意：

1. **API 参数限制**：
   - `Query` 参数支持较长搜索词（最大 100 字符）
   - `Count` 参数最大为 10（每次最多返回 10 条）
   - **建议**：使用精确的问题形式，充分利用字符限制

2. **标准 RESTful API**：
   - 搜索机制最稳定，不受 HTML 结构影响
   - 支持问题形式、陈述句等多种形式
   - **建议**：可以使用完整的问句，如"如何评价奥迪 Q5L 的油耗表现"

3. **权威等级和排序分数**：
   - API 返回 `AuthorityLevel` 和 `RankingScore`
   - **建议**：搜索词要能匹配专业内容，便于后续筛选高质量回答
   - 示例：`"奥迪 Q5L 油耗实测"` 比 `"奥迪 Q5L 怎么样"` 更能匹配专业回答

知乎用户搜索习惯：
- 喜欢用**问题形式**（如何评价、怎么选）
- 喜欢用**专业术语**（油耗、动力、操控）
- 喜欢用**对比分析**（vs、对比、区别）
- 搜索词较长，5-15 个字为主

#### 2.2 推荐搜索词库

**问题形式**：
```
如何评价奥迪 Q5L
奥迪 Q5L 值得购买吗
奥迪 A4L 怎么样
奥迪 Q5L 和 GLC 怎么选
奥迪 e-tron 优缺点
```

**技术分析**：
```
奥迪 Q5L 油耗实测
奥迪 Q5L 动力表现
奥迪 Q5L 空间评测
奥迪 Q5L 保养成本
奥迪 Q5L 驾驶感受
```

**对比分析**：

由于汽车之家使用**爬虫 + HTML 解析**，搜索词选择需要特别注意：

1. **只能匹配帖子标题**：
   - 爬虫只解析论坛帖子列表页的标题
   - **不支持**全文搜索（内容不会被搜索）
   - **建议**：搜索词必须是论坛帖子标题的高频词
   - 示例：`"提车作业"`、`"油耗实测"`、`"改装案例"`

2. **精确匹配限制**：
   - HTML 解析使用关键词过滤（`includes()` 或正则）
   - **不支持**模糊匹配、语义匹配
   - **建议**：使用论坛常用术语，而不是口语化表达
   - 示例：用 `"提车"` 而不是 `"提车日记"`，用 `"油耗"` 而不是"`油耗表现"`

3. **不支持多关键词组合**：
   - 爬虫脚本通常只搜索单个关键词
   - **不能**像 API 那样传递多个关键词
   - **建议**：使用高频单独词，而不是组合词
   - 示例：`"Q5L 改装"` 可能匹配不到，应该分开搜 `"Q5L"` 和 `"改装"`

4. **受 HTML 结构影响**：
   - 论坛改版会导致爬虫失效
   - **建议**：搜索词要保守，使用经典、稳定的术语
   - **避免**：网络流行语、新造词

5. **论坛 ID 限制**：
   - 需要在特定论坛 ID 下搜索（692: A4L, 812: Q5L 等）
   - **建议**：搜索词要符合对应论坛的习惯

// 根据参考素材来源选择提示词风格
function selectPromptStyle(references: InternetReference[]): string {
  const platformCount = {
    xiaohongshu: 0,
    zhihu: 0,
    autohome: 0,
  };
  
  // 统计参考素材来源
  for (const ref of references) {
    if (ref.source === '小红书') platformCount.xiaohongshu++;
    else if (ref.source === '知乎') platformCount.zhihu++;
    else if (ref.source === '汽车之家') platformCount.autohome++;
  }
  
  // 选择占比最高的平台风格
  const maxPlatform = Object.keys(platformCount).reduce(
    (a, b) => platformCount[a] > platformCount[b] ? a : b
  );
  
  return maxPlatform;
}

// 根据平台风格生成内容
async function generatePostWithPlatformStyle(
  topic: string,
  references: InternetReference[]
): Promise<GeneratedPost> {
  const style = selectPromptStyle(references);
  
  let systemPrompt: string;
  let userPrompt: string;
  
  switch (style) {
    case 'xiaohongshu':
      systemPrompt = buildXiaohongshuSystemPrompt();
      userPrompt = buildXiaohongshuUserPrompt(topic, references);
      break;
    case 'zhihu':
      systemPrompt = buildZhihuSystemPrompt();
      userPrompt = buildZhihuUserPrompt(topic, references);
      break;
    case 'autohome':
      systemPrompt = buildAutohomeSystemPrompt();
      userPrompt = buildAutohomeUserPrompt(topic, references);
      break;
    default:
      systemPrompt = buildPostSystemPrompt();
      userPrompt = buildPostUserPrompt(topic);
  }

}

// 根据平台技术实现选择搜索词
function selectSearchKeyword(platform: string): string {
  const config = getInternetReferenceConfig();
  const keywords = config.searchKeywords;
  
  switch (platform) {
    case 'xiaohongshu':
      // 小红书：从词库中随机选择，避免频繁更换
      return selectFromPredefinedKeywords(keywords);
    
    case 'zhihu':
      // 知乎：可以使用较长的专业问句
      return selectLongProfessionalQuery(keywords);
    
    case 'autohome':
      // 汽车之家：使用单个高频术语
      return selectSingleHighFrequencyKeyword(keywords);
    
    default:
      return keywords[0];
  }
}

// 小红书：从预定义词库选择
function selectFromPredefinedKeywords(keywords: string[]): string {
  // 避免频繁更换，每小时切换一次
  const hour = Math.floor(Date.now() / 3600000);
  const index = hour % keywords.length;
  return keywords[index];
}

// 知乎：选择较长的专业问句
function selectLongProfessionalQuery(keywords: string[]): string {
  // 优先选择包含"如何"、"评价"、"对比"的词
  const professionalKeywords = keywords.filter(
    k => k.includes('如何') || k.includes('评价') || k.includes('vs')
  );
  return professionalKeywords[0] || keywords[0];
}

// 汽车之家：选择单个高频术语
function selectSingleHighFrequencyKeyword(keywords: string[]): string {
  // 优先选择短词、论坛术语
  const shortKeywords = keywords.filter(k => k.length <= 4);
  return shortKeywords[0] || keywords[0];
}
