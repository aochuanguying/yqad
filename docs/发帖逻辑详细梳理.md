## 一、发帖模式对比

### 1.1 三种发帖模式

| 模式 | 触发方式 | 内容来源 | 素材来源 | 使用场景 |
|------|----------|----------|----------|----------|
| **主题发帖** | 定时/手动 | 预配置主题 + 子方向 | 本地素材优先 | 日常发帖，精华帖 |
| **自由发帖** | 定时/手动 | 互联网参考 + AI 改写 | 混合素材（本地 + 网络） | 无主题时兜底 |
| **API 生成** | 远程 API | 主题/自由模式 | 混合素材 | AutoJS 远程调用 |

### 1.2 模式选择逻辑

```typescript
// 位置：src/services/auto-post.ts

async performDailyPosts(count, mode, isManual) {
  const allTopics = await getAllTopics();
  const availableTopics = allTopics.filter(t => t.status === 'unused');
  
  for (let i = 0; i < count; i++) {
    const topic = availableTopics[topicIndex++];
    
    if (topic) {
      // 有可用主题 → 主题发帖模式
      await this.postWithTopic(topic, mode, triggerType);
    } else {
      // 无可用主题 → 自由发帖模式
      await this.postFreeStyle(mode, triggerType);
    }
  }
}
```

```typescript
async performComplianceCheck(input): Promise<ComplianceCheckResult> {
  const result = { passed: true, rejectReasons: [] };
  
  // 1. 内容去重检查
  const similarityResult = await contentDeduplicationService.checkSimilarity(
    input.title, input.content
  );
  if (similarityResult.isDuplicate) {
    result.passed = false;
    result.rejectReasons.push(`内容重复度过高 (${similarityResult.maxSimilarity * 100}%)`);
  }
  
  // 2. 敏感词检查（两级检测）
  const enhancedResult = await enhancedSensitiveWordService.detectAndReplace(input.content);
  if (!enhancedResult.passed) {
    result.passed = false;
    result.rejectReasons.push(enhancedResult.rejectReason);
  } else if (enhancedResult.filteredText !== input.content) {
    result.filteredContent = enhancedResult.filteredText;  // 使用替换后内容
  }
  
  // 3. 内容质量评分
  const qualityScore = await contentQualityScoringService.calculateScore({
    title: input.title,
    content: result.filteredContent || input.content,
    imageCount: input.imageCount,
    similarityScore: result.similarityCheck?.maxSimilarity
  });
  if (qualityScore.finalScore < 60) {
    result.passed = false;
    result.rejectReasons.push(`内容质量评分过低 (${qualityScore.finalScore}/60)`);
  }
  
  // 4. 发帖间隔检查
  if (input.topicId) {
    const intervalResult = await postingIntervalControlService.checkPostingInterval(
      input.topicId, input.topicName
    );
    if (!intervalResult.allowed && !intervalResult.canOverride) {
      result.passed = false;
      result.rejectReasons.push(`发帖间隔不足：${intervalResult.reason}`);
    }
  }
  
  // 5. 生成合规报告
  const reportId = await complianceCheckReportService.saveReport({
    ...result,
    checkDuration: Date.now() - startTime
  });
  result.reportId = reportId;
  
  return result;
}
```

```typescript
async selectBalancedSubDirection(topicId: string): Promise<number> {
  const subDirections = await topicUsageStorage.getSubDirectionStats(topicId);
  
  // 计算权重（使用次数的倒数）
  const totalInverseWeight = subDirections.reduce((sum, usage) => {
    return sum + 1 / (usage.usedCount + 1);
  }, 0);
  
  // 轮盘赌选择
  const random = Math.random() * totalInverseWeight;
  let cumulative = 0;
  
  for (let i = 0; i < subDirections.length; i++) {
    const inverseWeight = 1 / (subDirections[i].usedCount + 1);
    cumulative += inverseWeight;
    
    if (random <= cumulative) {
      return i;  // 选择使用次数少的
    }
  }
  
  return 0;
}
```

## 十、常见问题排查

### Q1: 发帖失败，如何定位问题？

**排查步骤**:
1. 查看 `post_logs` 表，找到对应的日志记录
2. 检查 `status` 字段（success/failed/pending）
3. 查看 `error_message` 字段
4. 根据错误信息定位到具体步骤

**常见错误**:
- `标题去重失败`: 调整 avoidRepeatDays 或增加 maxGenerateRetries
- `图片上传失败`: 检查 CDN 配置和网络
- `合规性检查未通过`: 查看 rejectReasons 详情
- `发布失败`: 检查 API Token 和网络
