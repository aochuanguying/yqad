# 汽车之家页面结构监控指南

## 📋 功能概述

当汽车之家页面结构发生变更，导致主选择器 `.fn-main .post` 失效时，系统会自动：

1. **检测失效**：Python 脚本检测到选择器无法找到元素
2. **发送警告**：自动调用 API 将警告信息保存到数据库
3. **页面提示**：在网络发帖配置页面显示警告横幅
4. **自动降级**：尝试 5 层 fallback 选择器继续工作

## 🎯 使用场景

### 场景 1：页面结构变更

当汽车之家修改了 HTML 结构，`.fn-main .post` 选择器失效时：

**系统行为**：
1. ⚠️ Python 脚本检测到选择器失效
2. 📡 自动发送警告到配置页面
3. 💾 警告信息保存到数据库
4. 🔄 自动尝试 fallback 选择器

**用户可见**：
- 打开"论坛设置" → "网络发帖"页面
- 看到黄色警告横幅提示页面结构可能变更

### 场景 2：警告清除

确认警告已解决或无需关注时：

**操作**：
1. 点击警告横幅上的"✓ 我知道了"按钮
2. 警告从页面清除
3. 数据库中的警告信息被清空

## 🔧 技术实现

### 1. 数据库迁移

```sql
-- Migration 032
ALTER TABLE network_post_config 
ADD COLUMN autohome_selector_warning TEXT DEFAULT NULL 
COMMENT '汽车之家选择器失效警告信息'
AFTER autohome_enabled;
```

**执行迁移**：
```bash
npx ts-node scripts/apply-migration-032.ts
```

### 2. 后端 API

**获取警告**：
```
GET /api/network-post-config/autohome-warning
```

**响应示例**：
```json
{
  "success": true,
  "warning": "页面结构可能已变更...",
  "hasWarning": true
}
```

**更新警告**：
```
POST /api/network-post-config/autohome-warning
Content-Type: application/json

{
  "warning": "警告信息",
  "clear": false  // true 表示清除警告
}
```

### 3. Python 脚本检测

**检测逻辑** (`scripts/test_autohome.py`)：
```python
content_element = await page.query_selector(".fn-main .post")

if not content_element:
    # ⚠️ 选择器失效
    print(f"⚠️ 警告：选择器 '.fn-main .post' 未找到元素")
    
    # 📡 发送警告到 API
    warning_data = {
        "warning": "页面结构可能已变更，主选择器失效",
        "timestamp": datetime.now().isoformat()
    }
    urllib.request.urlopen(api_url, data=warning_data, timeout=5)
    
    # 🔄 尝试 fallback 选择器
    fallback_selectors = [".post", ".thread-content", ...]
```

### 4. 前端显示

**警告横幅** (`index.html`)：
```html
<div id="autohome-warning-box" class="hidden p-4 bg-amber-50 border border-amber-200 rounded-md">
  <p class="text-sm text-amber-800">
    <strong>⚠️ 警告：</strong>
    <span id="autohome-warning-text"></span>
  </p>
  <button onclick="clearAutohomeWarning()">✓ 我知道了</button>
</div>
```

**加载逻辑**：
```javascript
// 页面加载时检查警告
async function loadAutohomeWarning() {
  const resp = await fetch('/api/network-post-config/autohome-warning');
  const data = await resp.json();
  
  if (data.hasWarning) {
    document.getElementById('autohome-warning-text').textContent = data.warning;
    document.getElementById('autohome-warning-box').classList.remove('hidden');
  }
}
```

## 📊 警告信息示例

### 警告内容

```
页面结构可能已变更，主选择器 '.fn-main .post' 失效，已自动使用 fallback 选择器
```

### 警告级别

- **⚠️ 黄色警告**：选择器失效，但 fallback 成功
- **❌ 红色错误**：所有选择器都失败（需要立即处理）

## 🎯 操作流程

### 开发者角度

1. **收到告警**
   - 查看数据库中的警告信息
   - 检查汽车之家页面是否真的变更

2. **验证问题**
   ```bash
   python3 scripts/test_autohome.py "奥迪 Q5L" 3 --fetch-content
   ```
   - 查看日志中的选择器使用情况
   - 确认 fallback 是否有效

3. **修复选择器**
   - 如果 fallback 有效：更新文档说明新的选择器
   - 如果 fallback 无效：立即修复主选择器

4. **清除警告**
   - 手动调用 API 清除警告
   - 或等待用户点击"我知道了"

### 用户角度

1. **看到警告**
   - 打开"论坛设置" → "网络发帖"
   - 看到黄色警告横幅

2. **理解警告**
   - 警告说明页面结构可能变更
   - 系统已自动尝试其他选择器

3. **处理警告**
   - 如果内容获取正常：点击"我知道了"清除警告
   - 如果内容获取异常：联系开发者

## 📈 性能影响

- **API 调用**：异步非阻塞，不影响主流程
- **数据库存储**：TEXT 字段，占用空间小
- **页面加载**：仅在配置页面加载时检查，影响可忽略

## 🛠️ 故障排查

### 问题 1：警告不显示

**检查步骤**：
1. 确认数据库迁移已执行
2. 检查 API 是否正常：
   ```bash
   curl http://localhost:8080/api/network-post-config/autohome-warning
   ```
3. 检查前端控制台是否有错误

### 问题 2：警告无法清除

**检查步骤**：
1. 检查数据库连接
2. 手动清除：
   ```sql
   UPDATE network_post_config 
   SET autohome_selector_warning = NULL 
   WHERE id = 1;
   ```

### 问题 3：Python 脚本发送警告失败

**检查步骤**：
1. 确认服务已启动（默认端口 8080）
2. 检查网络是否可达
3. 查看 Python 日志中的错误信息
