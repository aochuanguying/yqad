# 汽车之家爬虫技术方案

## 1. 背景

### 1.1 需求
- 爬取汽车之家论坛帖子，作为内容创作参考素材
- 需要获取帖子标题、正文内容、作者、互动数据等
- 主要针对奥迪相关论坛（A4L、Q5L、A6L 等）

### 1.2 历史问题
- **旧 URL 失效**：原论坛 URL 格式 `club.autohome.com.cn/bbs/forum-{id}-1.html` 全部返回 404
- **页面改为 SPA**：新页面使用 Vue 动态渲染，curl/requests 无法获取内容
- **需要登录**：官方 API 需要登录态，直接调用返回"未登录"

---

## 2. 技术方案

### 2.1 核心思路

```
搜索 API (sou.api.autohome.com.cn)
         ↓
    返回帖子列表（标题、URL、作者、图片、回复数、浏览数）
         ↓
    Playwright 打开帖子 URL
         ↓
    提取完整正文内容
```

### 2.2 技术栈

| 组件 | 技术 | 作用 |
|------|------|------|
| 搜索 API | HTTP requests | 获取帖子列表 |
| 正文获取 | Playwright + Chromium | 打开 SPA 页面，提取渲染后的内容 |
| 反检测 | stealth.min.js | 隐藏自动化浏览器特征 |

### 2.3 关键发现

#### 搜索 API
- **URL**: `https://sou.api.autohome.com.cn/v1/search`
- **无需 Cookie**：可直接调用
- **关键参数**:
  - `source=pc`：平台来源
  - `q`：搜索关键词
  - `entry=40`：入口类型（论坛搜索）
  - `pid=90300024`：产品 ID
  - `ext`：扩展参数（JSON），**必须包含** `plat`、`pf`、`q`、`offset`、`size`

#### 搜索结果类型
- `type: "box"` - 论坛推荐框（非帖子列表）
- `type: "card"` - 帖子卡片（需要的结果）

#### 关键词策略
- ✅ 精准词：`"奥迪 Q5L"`、`"提车作业"`、`"奥迪 A4L"` → 返回 card
- ❌ 宽泛词：`"奥迪"` → 返回 box

## 3. 实现细节

### 3.1 Python 脚本 (`scripts/test_autohome.py`)

#### 功能
1. `search_autohome(keyword, limit)` - 调用搜索 API
2. `fetch_post_content(url)` - 用 Playwright 获取正文
3. `search_with_content(keyword, limit, fetch_content=True)` - 组合两者

#### 正文提取逻辑

**最新实现**（已验证）：

```python
# 1. 打开帖子页面
await page.goto(url, wait_until="domcontentloaded")
await asyncio.sleep(5)

# 2. 使用精准选择器定位正文
content_element = await page.query_selector(".fn-main .post")

# 3. 直接提取正文文本和图片
content_text = await content_element.inner_text()
images = await content_element.query_selector_all("img")
```

**选择器验证**：
- ✅ `.fn-main .post` - 所有测试帖子都成功找到
- ❌ `.fn-container .fn-main .post` - 未找到（`.fn-container` 不是必需的）

**优势**：
- 代码简化 ~70 行
- 自动过滤导航和 UI 元素
- 提取更精准，性能更好

#### 调用方式
```bash
# 仅搜索（快速）
python3 scripts/test_autohome.py "奥迪 Q5L" 5

# 搜索 + 获取正文（慢，前 3 条）
python3 scripts/test_autohome.py "奥迪 Q5L" 5 --fetch-content
```

### 3.2 TypeScript 服务 (`src/services/internet-search/autohome-search.ts`)

#### 调用流程
```typescript
search(keywords, maxResults)
  ↓
searchViaPython(keyword, maxResults)
  ↓
spawn('python3', ['test_autohome.py', keyword, maxResults, '--fetch-content'])
  ↓
解析 JSON 结果 → SearchResult[]
```

#### 超时设置
- 30 秒（因为 Playwright 获取正文较慢）

```json
              "reply": 85,
              "view": 39301
            }
```

### 4.2 SearchResult 格式

```typescript
{
  title: string;       // 帖子标题
  content: string;     // 帖子正文（Playwright 获取）
  source: '汽车之家';

  author: string;      // 作者
  likes: number;       // 回复数
  comments: number;    // 回复数

}
```

## 7. 性能与优化

### 7.1 性能数据

| 操作 | 耗时 |
|------|------|
| 搜索 API 调用 | ~1 秒 |
| Playwright 启动 | ~2 秒 |
| 打开单个帖子 | ~3-5 秒 |
| 获取 3 条正文 | ~15-20 秒 |
| **总计** | **~20-25 秒** |

### 7.2 优化空间

1. **并发获取**：同时打开多个帖子 URL（目前串行）
2. **缓存机制**：相同 URL 不重复获取
3. **超时优化**：根据帖子数量动态调整超时时间
4. **降级策略**：Playwright 失败时返回空正文

## 8. 问题与解决方案

### 8.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 搜索返回 box 而非 card | 关键词太宽泛 | 优化关键词选择策略 |
| Playwright 获取失败 | 页面反爬/网络问题 | 注入 stealth 脚本，增加重试 |
| 正文提取不完整 | 页面结构复杂 | 优化文本过滤逻辑 |
| 超时 | Playwright 太慢 | 增加超时时间到 30 秒 |

### 8.2 已知限制

1. **速度慢**：Playwright 获取正文需要 15-25 秒
2. **资源消耗**：启动 Chromium 占用内存较大
3. **稳定性**：依赖汽车之家页面结构，`.fn-main .post` 选择器失效需更新

### 8.3 选择器说明

**当前使用的选择器**：
- `.fn-main .post` ✅ （经过 5+ 帖子验证）

**已淘汰的选择器**：
- `.fn-container .fn-main .post` ❌
- 复杂文本过滤逻辑 ❌
