```

       │
       ▼
┌─────────────────────────────────────────────────────────┐
│                   AutoPostService                        │
```

## 一、触发方式

发帖不再使用定时调度，改为**外部 AutoJS 脚本通过 API 触发**。

- 入口文件：`src/services/auto-post.ts` → `AutoPostService.performDailyPosts()`
- 触发方式：AutoJS 脚本调用 Web API → 路由转发到 AutoPostService
- 调度器（`src/scheduler/index.ts`）仅负责：评论、素材整理、Cookie 刷新、车辆监控

## 二、Cookie 配置与管理

### 存储

所有平台 Cookie 统一存储在 MySQL 表 `network_post_config`，通过 `NetworkPostConfigStorage` 管理。

| 平台 | Cookie 字段 | 版本字段 | 是否必须 |
|------|------------|---------|---------|
| 小红书 | `xiaohongshu_cookie` | `xiaohongshu_cookie_version` | 搜索必须 |
| 知乎 | `zhihu_cookie` | `zhihu_cookie_version` | Playwright 提取正文需要 |
| 汽车之家 | `autohome_cookie` | - | 搜索 API 不需要 Cookie |

### 配置入口

Web 管理界面 → 论坛设置 → 网络发帖 → 各平台配置区

## 三、Cookie 自动续期

### 调度

- Cron 配置：从数据库 `scheduler_config` 读取 `cookie_refresh_cron`
- 注册位置：`src/scheduler/index.ts` → 注册小红书和知乎两个刷新任务
- Handler：`src/index.ts` → `refreshXiaohongshuCookie` / `refreshZhihuCookie`

### 续期逻辑（smartRefreshCookie）

```
1. 验证当前 Cookie 有效性
   ├─ 小红书：调用 XiaohongshuSearch.testConnection()（xhshow 签名搜索）
   └─ 知乎：GET https://www.zhihu.com/api/v4/me（验证 HTTP 200）

2. Cookie 有效 → 注入浏览器续期
   ├─ 使用普通浏览器 context（非持久化，避免旧 session 覆盖）
   ├─ addCookies 注入数据库中的有效 Cookie
   ├─ 打开平台首页（登录态）→ 刷新页面
   ├─ 提取新 Cookie → 验证有效性
   │   ├─ 验证通过 → 保存 + success 日志
   │   └─ 验证失败 → 不保存，保留原 Cookie + success 日志
   └─ 异常 → 不影响原 Cookie

3. Cookie 无效 → 尝试浏览器 session 续期
   ├─ 使用持久化浏览器目录
   ├─ 检测是否登录态 → 提取 Cookie → 验证
   │   ├─ 验证通过 → 保存 + success 日志
   │   └─ 验证失败 → failed 日志
   └─ 也失效 → failed 日志 + requiresManualRefresh

4. 状态面板联动
   └─ 最后一条日志 status=failed → 页面显示红色"Cookie 已失效"
```

### 手动刷新（refreshCookie）

Web 页面点"🔄 刷新 Cookie" → 弹出二维码扫码：

```
1. 打开持久化浏览器检查登录态
   ├─ 已登录 → 提取 Cookie → 验证
   │   ├─ 验证通过 → 保存并返回成功
   │   └─ 验证失败 → 进入扫码流程
   └─ 未登录 → 进入扫码流程

2. 扫码流程
   ├─ 打开登录页
   ├─ 截取二维码 → Base64 返回前端显示
   ├─ 轮询等待扫码（最多 2 轮，每轮 2-3 分钟）
   ├─ 扫码成功 → 提取 Cookie → 保存
   └─ 超时/失败 → 返回错误
```

## 四、搜索帖子

### 搜索管理器

文件：`src/services/internet-search/search-manager.ts`

职责：
- 管理所有平台搜索服务（小红书/知乎/汽车之家）
- 实现平台轮询策略（根据优先级和权重选择平台）
- 搜索词优化（PlatformAwareKeywordSelector）

### 平台搜索词策略

| 平台 | 策略 | 示例 |
|------|------|------|
| 小红书 | 从词库按小时轮换 | "奥迪Q5L用车" |
| 知乎 | 优先专业问句，否则选最长词 | "如何评价奥迪Q5L油耗" |
| 汽车之家 | 自动拼接论坛术语后缀 | "奥迪Q5L提车" |

### 各平台搜索实现

#### 小红书（`xiaohongshu-search.ts`）

```
Cookie → xhshow sign_headers 生成完整请求头 → POST 搜索 API
→ 获取笔记列表 → 对前 2 条补充完整正文（getNoteDetail API）
```

- 签名方式：`Xhshow.sign_headers(method, uri, cookies, payload, x_rap)`
- 搜索 API：`POST https://so.xiaohongshu.com/api/sns/web/v2/search/notes`
- 详情 API：`POST https://edith.xiaohongshu.com/api/sns/web/v1/feed`
- 频率控制：滑动窗口限流，请求间随机延迟 1-5 秒

#### 知乎（`zhihu-search.ts`）

```
步骤 1：Access Secret → 知乎开放平台 API 搜索
步骤 2：Python + Playwright → 打开 URL 提取正文和图片
```

- 搜索 API：`GET https://developer.zhihu.com/api/v1/content/zhihu_search`
- 正文提取：`scripts/test_zhihu_content.py`（Playwright + BeautifulSoup）
- Cookie 作用：注入 Playwright 绕过知乎安全验证
- URL 清理：去掉 `?utm_medium=...` 追踪参数

#### 汽车之家（`autohome-search.ts`）

```
Python + Playwright → 搜索 + 提取正文
```

- 脚本：`scripts/test_autohome.py`
- 不需要 Cookie
- 正文选择器：`.fn-main .post`
- 支持 `--fetch-content` 参数获取正文
- 支持 `--detail` 参数获取单帖详情

## 五、获取帖子详情

### 小红书笔记详情

```typescript
// xiaohongshu-search.ts → getNoteDetail()
// 使用 xhshow 签名调用 feed API
POST https://edith.xiaohongshu.com/api/sns/web/v1/feed
payload: { source_note_id, image_formats, xsec_source, xsec_token }
```

返回：标题、完整正文、图片列表、互动数据

### 知乎正文提取

```
Python 脚本（test_zhihu_content.py）：
1. 解析 Cookie → 注入 Playwright 浏览器
2. 打开知乎 URL → 等待页面加载
3. 使用选择器提取：
   - 标题：h1.QuestionHeader-title / h1.Post-Title
   - 正文：.Post-RichText / .RichText / .AnswerCard .RichText
   - 图片：正文内 img 元素
4. Fallback：page.content() + BeautifulSoup 解析 HTML
```

### 汽车之家正文提取

```
Python 脚本（test_autohome.py）：
1. Playwright 打开帖子 URL
2. 选择器：.fn-main .post
3. 提取文本段落和图片
```

## 七、AI 内容生成

### 入口

文件：`src/ai/content-generator.ts` → `generatePost()`

### 分平台提示词

根据参考素材来源自动选择风格：

| 平台 | 风格特点 | 字数要求 |
|------|---------|---------|
| 小红书 | 口语化、emoji、情绪化 | 100-500 字 |
| 知乎 | 专业术语、数据分析、逻辑清晰 | 800-2000 字 |
| 汽车之家 | 真实车主口吻、价格数据 | 500-1500 字 |

### 生成流程

```
1. 选择提示词风格（selectPromptBuilder）
2. 构建 systemPrompt + userPrompt
3. 调用 AI 服务（支持多 Provider 降级）
4. 解析标题和正文
5. 正文长度约束（enforceLength）
6. 内容适配性检查（emoji 数量、段落长度等）
```

### 去重机制

- **标题去重**：与历史标题精确匹配，重复最多重试 2 次
- **语义去重**：ChromaDB 向量相似度检测（阈值可配）

## 八、发帖 Pipeline（postWithTopic）

文件：`src/services/auto-post.ts`

### 8 步 Pipeline

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1 | `selectSubDirectionAndOutline` | 从主题子方向中选择（均衡使用），50% 概率生成提纲变体 |
| 2 | `generateContentWithDedup` | AI 生成 + 标题去重 + 语义去重（ChromaDB） |
| 3 | `selectMaterials` | 本地素材优先，无本地素材时用网络参考图片 |
| 4 | `uploadImagesToCDN` | 通过一汽奥迪 API 上传图片，获取 CDN URL |
| 5 | `matchHotTopics` | 获取 APP 热门话题，AI 匹配关联话题 |
| 6 | `applyDiversityTransforms` | 30% 概率变换标题，20% 概率变换内容（增加多样性） |
| 7 | `performComplianceCheck` | 合规性检查（敏感词、政治内容等） |
| 8 | `publishAndRecord` | 调 API 发帖 + 记录到 MySQL + ChromaDB 索引 |

### 自由生成模式（无主题时）

```
postFreeStyle() → tryInternetReferenceMode()
1. 检查频率限制
2. 搜索互联网参考素材（search-manager 轮询各平台）
3. AI 改写生成原创内容（带参考素材的提示词）
4. 抄袭检测
5. 发布
```

## 九、素材选择策略

文件：`src/services/hybrid-material-service.ts`

### 优先级模式

| 模式 | 说明 |
|------|------|
| `local-first` | 优先使用本地素材库（processed 目录） |
| `internet-first` | 优先使用网络下载的参考图片 |
| `hybrid` | 混合模式（按比例组合） |

### 匹配逻辑

```
1. 从帖子标题提取关键词（AI）
2. ChromaDB 语义搜索匹配本地素材
3. 素材质量评分（清晰度/构图/光线/相关性/新鲜度）
4. 去重：避免重复使用同一张图
5. 记录使用追溯（关联帖子 ID）
```

## 十一、错误处理与降级

| 场景 | 降级策略 |
|------|---------|
| Cookie 失效 | 写 failed 日志，前端显示红色，等待手动刷新 |
| 搜索平台全部失败 | 回退到 AI 直接生成（不依赖参考素材） |
| 图片上传失败 | 以纯文字方式继续发帖 |
| ChromaDB 不可用 | 降级为仅标题去重 |
| AI 主模型失败 | 自动切换备用模型（FallbackChain） |
| Redis 不可用 | 降级到内存存储 |
