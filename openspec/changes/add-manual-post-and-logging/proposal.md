## 为什么

当前项目已支持自动发帖和远程 API 触发发帖，但缺少手工立即发帖的功能，用户无法在管理页面手动触发发帖操作。同时，发帖后缺乏完整的日志记录，用户无法查看历史发帖的详细信息（如发帖时间、自动/手动、发帖内容等）。

## 变更内容

1. **新增手工立即发帖功能**：参考现有的"立即评论"功能，在 Web 管理页面添加"立即发帖"按钮，支持手动触发发帖任务
2. **新增发帖日志功能**：记录每次发帖的完整信息（时间、触发方式、标题、内容、状态等），并在 Web 管理页面提供日志查看功能
3. **扩展发帖 API**：新增手工发帖 API 端点和日志查询 API 端点
4. **发帖日志持久化**：将发帖日志保存到磁盘，支持分页查询和历史记录管理

## 功能 (Capabilities)

### 新增功能
- `manual-post-trigger`: 支持在 Web 管理页面手动触发发帖任务，参考立即评论的实现方式
- `post-logging`: 记录发帖日志，包括时间、触发方式（自动/手动）、标题、内容、状态等，并支持页面查看

### 修改功能
- `auto-post`: 发帖服务需要支持区分自动/手动触发方式，并在日志中记录
- `remote-post-api`: 发帖 API 需要区分触发方式，手工发帖成功后立即扣减主题使用次数（如使用主题），无需等待回调
- `config-web-ui`: Web 管理页面需要增加"立即发帖"按钮和发帖日志查看页面

## 影响

- **代码影响**：
  - `src/services/auto-post.ts`：需要增加日志记录逻辑
  - `src/web/routes/posts-routes.ts`：需要增加手工发帖 API 和日志查询 API
  - `src/web/index.html`：需要增加"立即发帖"按钮和日志查看 UI
  - 新增 `src/services/post-logging-service.ts`：发帖日志管理服务
  - 新增 `src/types/post-logging.ts`：发帖日志类型定义

- **API 影响**：
  - 新增 `POST /api/posts/execute`：手工立即发帖 API
  - 新增 `GET /api/posts/logs`：查询发帖日志 API
  - 新增 `GET /api/posts/logs/:id`：查询单条发帖日志详情

- **数据持久化**：
  - 新增 `data/post-logs.json`：发帖日志持久化文件

- **UI 影响**：
  - 发帖配置页面增加"立即发帖"按钮
  - 新增"发帖日志"标签页，支持日志查看、筛选、分页
