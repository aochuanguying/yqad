## 修改需求

### 需求：系统必须自动生成并发布优质帖子

**修改说明**：增加发帖日志记录要求

系统必须定时发布帖子到社区。发帖时必须优先使用已配置且未使用的主题（含关联素材和提纲），当所有主题用完或无主题配置时，回退到基于 AI 自由生成的模式。系统必须在可用时优先按"精华策略"生成精华候选帖；当精华候选无法满足最低门槛时，系统必须自动降级为普通帖子发布。**每次发帖完成后，系统必须记录发帖日志，包括触发方式（auto）、发帖模式、使用的主题、标题、内容、图片、状态等信息。**

#### 场景：优先使用已配置主题发帖

- **当** 自动发帖任务触发且存在"未使用"状态的主题
- **那么** 系统必须选取一个未使用的主题，使用其方向描述和内容提纲作为 AI 生成的约束，生成帖子内容，并附带主题关联的素材图片
- **并且** 发帖完成后必须记录日志，triggerType 为"auto"，postType 为"topic"，包含 topicId 和 topicName

#### 场景：无可用主题时自由生成

- **当** 自动发帖任务触发且所有主题均已使用（或无主题配置）
- **那么** 系统必须回退到原有模式：从内容分析结果中选取热点话题，调用 AI 自由生成帖子
- **并且** 发帖完成后必须记录日志，triggerType 为"auto"，postType 为"free"，topicId 和 topicName 为空

#### 场景：发帖后标记主题已使用

- **当** 使用预配置主题成功发帖
- **那么** 系统必须将该主题标记为"已使用"并记录使用时间
- **并且** 必须在日志中记录使用的主题 ID 和主题名称
