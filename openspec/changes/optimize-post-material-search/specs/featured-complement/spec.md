## 新增需求

### 需求：系统必须优化精华补图策略
系统必须实现基于匹配分的精华补图算法，避免为了凑数混入无关图片。

#### 场景：主选图不足
- **当** 主选图流程返回的图片数量少于 `minImages` 要求
- **那么** 系统必须启动补图流程

#### 场景：按匹配分补图
- **当** 补图流程启动
- **那么** 系统必须从索引匹配得分排序中继续选取下一批候选，而非全库随机

#### 场景：候选耗尽降级
- **当** 索引中无更多候选可用
- **那么** 系统必须记录日志并降级为普通帖，禁止混入无关图片

#### 场景：补图数量限制
- **当** 补图时
- **那么** 系统必须限制补图数量不超过 `needed + 8` 张，避免过度补图
