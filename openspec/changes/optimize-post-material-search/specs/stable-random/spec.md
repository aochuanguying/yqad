## 新增需求

### 需求：系统必须提供稳定随机选图机制
系统必须实现基于确定性种子的随机选图算法，确保相同主题和日期下选图结果可复现。

#### 场景：生成稳定种子
- **当** 需要为选图生成随机种子
- **那么** 系统必须使用 `topic.id + useCount + 日期` 格式生成种子字符串

#### 场景：种子哈希
- **当** 生成种子后
- **那么** 系统必须对种子字符串进行 MD5 哈希，生成 32 位无符号整数

#### 场景：稳定随机选择
- **当** 从候选池中随机选取图片
- **那么** 系统必须使用基于种子的伪随机算法进行 Fisher-Yates 洗牌

#### 场景：相同条件结果一致
- **当** 同一主题在同一使用次数和同一日期下选图
- **那么** 系统必须返回相同的选图结果

#### 场景：不同日期结果变化
- **当** 日期发生变化
- **那么** 系统必须返回不同的选图结果，保持内容多样性
