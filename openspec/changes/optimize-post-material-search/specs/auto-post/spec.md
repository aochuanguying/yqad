## 修改需求

### 需求：系统必须自动生成并发布优质帖子
系统必须定时发布帖子到社区。发帖时必须优先使用已配置且未使用的主题（含关联素材和提纲），当所有主题用完或无主题配置时，回退到基于 AI 自由生成的模式。系统必须在可用时优先按"精华策略"生成精华候选帖；当精华候选无法满足最低门槛时，系统必须自动降级为普通帖子发布。选图时必须使用加权匹配算法和稳定随机策略。

#### 场景：优先使用已配置主题发帖
- **当** 自动发帖任务触发且存在"未使用"状态的主题
- **那么** 系统必须选取一个未使用的主题，使用其方向描述和内容提纲作为 AI 生成的约束，生成帖子内容
- **并且** 选图时必须使用加权匹配算法从素材索引中选取匹配度最高的图片
- **并且** 选图时必须使用基于 `topic.id + useCount + 日期` 的稳定随机种子
- **并且** 必须附带主题关联的素材图片（如配置了 materialPaths）

#### 场景：精华候选不达标时按匹配分补图
- **当** 精华候选帖图片数量不足 `minImages` 要求
- **那么** 系统必须从索引匹配得分排序中继续选取下一批候选进行补图
- **禁止** 从全库随机选取无关图片进行补图
- **当** 索引中无更多候选可用时
- **那么** 系统必须记录日志并降级为普通帖发布
