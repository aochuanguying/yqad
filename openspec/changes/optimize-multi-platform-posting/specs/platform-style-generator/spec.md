## 新增需求

### 需求：系统必须根据平台内容风格生成 AI 提示词

系统应当根据不同平台的内容调性（小红书种草风、知乎专业风、汽车之家车主风）生成差异化的 AI 提示词，以提升生成内容的平台适配性和质量。

#### 场景：小红书风格提示词生成
- **当** 参考素材主要来自小红书时
- **那么** 系统必须生成包含 emoji、简短段落、情绪化表达（真香、绝绝子）的提示词，要求 AI 生成 100-500 字的种草风格内容

#### 场景：知乎风格提示词生成
- **当** 参考素材主要来自知乎时
- **那么** 系统必须生成包含专业术语、数据分析、逻辑论证的提示词，要求 AI 生成 800-2000 字的深度分析内容

#### 场景：汽车之家风格提示词生成
- **当** 参考素材主要来自汽车之家时
- **那么** 系统必须生成包含真实车主视角、具体配置价格、实拍图片描述的提示词，要求 AI 生成 500-1500 字的车主分享内容

#### 场景：混合素材风格选择
- **当** 参考素材来自多个平台且占比相近时
- **那么** 系统必须选择占比最高的平台风格，如果占比相同则优先使用小红书风格

### 需求：系统必须支持提示词模板管理

系统应当为每个平台提供可配置的提示词模板，支持动态更新和版本管理。

#### 场景：加载平台提示词模板
- **当** 调用 `getPromptTemplate(platform)` 接口时
- **那么** 系统必须返回对应平台的完整提示词模板，包括 systemPrompt 和 userPrompt

#### 场景：模板变量替换
- **当** 生成提示词时
- **那么** 系统必须将模板中的变量（如{topic}、{referenceTexts}）替换为实际值

#### 场景：模板版本控制
- **当** 提示词模板更新时
- **那么** 系统必须保留历史版本记录，支持回滚到任意历史版本

### 需求：系统必须评估生成内容的平台适配性

系统应当对 AI 生成的内容进行平台适配性评分，确保内容符合目标平台的风格要求。

#### 场景：小红书内容适配性检查
- **当** 生成小红书风格内容时
- **那么** 系统必须检查内容是否包含 emoji（至少 3 个）、段落是否简短（每段 1-3 句）、是否包含情绪化表达

#### 场景：知乎内容适配性检查
- **当** 生成知乎风格内容时
- **那么** 系统必须检查内容是否包含专业术语、是否有数据支撑、是否有逻辑论证结构

#### 场景：汽车之家内容适配性检查
- **当** 生成汽车之家风格内容时
- **那么** 系统必须检查内容是否包含具体配置信息、是否有价格明细、是否使用真实车主口吻

#### 场景：适配性评分过低处理
- **当** 生成内容的平台适配性评分低于 60 分时
- **那么** 系统必须重新生成内容或回退到通用风格
