## 新增需求

### 需求：系统必须根据平台技术实现选择搜索词

系统应当根据不同平台的技术实现特点（小红书签名算法、知乎 RESTful API、汽车之家爬虫）智能选择搜索词，以提升搜索命中率和素材质量。

#### 场景：小红书平台搜索词选择
- **当** 搜索平台为小红书时
- **那么** 系统必须从预配置词库中选择搜索词，避免频繁更换触发风控，优先选择"车型 + 场景"组合词（2-5 字）

#### 场景：知乎平台搜索词选择
- **当** 搜索平台为知乎时
- **那么** 系统必须使用专业术语和问题形式搜索词（5-15 字），充分利用 API 的 100 字符限制

#### 场景：汽车之家平台搜索词选择
- **当** 搜索平台为汽车之家时
- **那么** 系统必须使用单个高频论坛术语（2-4 字），避免长句和组合词，仅匹配帖子标题

#### 场景：搜索词配置更新
- **当** 数据库中的搜索词配置发生更新时
- **那么** 系统必须在 5 分钟内同步新配置，并清除 Redis 缓存

### 需求：系统必须支持分平台搜索词配置管理

系统应当为每个平台独立配置搜索词库、频率限制和优先级，支持动态更新和热加载。

#### 场景：获取分平台配置
- **当** 调用 `getConfigByPlatform(platform)` 接口时
- **那么** 系统必须返回对应平台的完整配置，包括 search_keywords、rateLimitPerHour、priority 等字段

#### 场景：配置不存在时的回退
- **当** 请求的平台配置不存在时
- **那么** 系统必须返回通用配置（platform='all'），如果通用配置也不存在则使用默认配置

#### 场景：配置缓存
- **当** 首次获取平台配置时
- **那么** 系统必须将配置缓存到 Redis，过期时间设置为 5 分钟

### 需求：系统必须记录搜索词使用效果

系统应当记录每次搜索词的使用效果（搜索成功率、素材质量评分），用于优化搜索词选择策略。

#### 场景：记录搜索成功
- **当** 搜索 API 调用成功并返回结果时
- **那么** 系统必须记录搜索平台、搜索词、返回数量、耗时等指标

#### 场景：记录素材质量
- **当** 搜索到的素材被选中使用时
- **那么** 系统必须记录该搜索词对应的素材质量评分，用于后续优化

#### 场景：效果分析
- **当** 调用 `analyzeKeywordEffectiveness(platform)` 接口时
- **那么** 系统必须返回各搜索词的效果排名，包括成功率、平均素材质量等指标
