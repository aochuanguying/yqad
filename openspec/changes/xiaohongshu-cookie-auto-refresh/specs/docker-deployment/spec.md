## 新增需求

### 需求：Dockerfile 配置
系统必须提供 Dockerfile 支持容器化部署。

#### 场景：构建镜像
- **当** 执行 `docker build`
- **那么** 系统基于 `python:3.10-slim` 镜像
- **那么** 系统安装中文字体包（fonts-wqy-zenhei, fonts-noto-cjk）
- **那么** 系统安装 Playwright 和 Chromium 浏览器
- **那么** 系统安装 Node.js 运行时

#### 场景：容器启动
- **当** 容器启动时
- **那么** 系统自动启动定时调度器
- **那么** 系统自动启动 Web 服务
- **那么** 系统监听指定端口（默认 3000）

### 需求：Docker Compose 配置
系统必须提供 docker-compose.yml 简化部署。

#### 场景：服务编排
- **当** 执行 `docker-compose up -d`
- **那么** 系统启动 Cookie 管理服务
- **那么** 系统挂载浏览器数据卷（`./browser_data:/tmp/xiaohongshu_browser_data`）
- **那么** 系统挂载二维码目录（`./qr_codes:/tmp/qr_codes`）
- **那么** 系统设置环境变量（DB_HOST, COOKIE_ENCRYPTION_KEY 等）

#### 场景：资源限制
- **当** 容器运行时
- **那么** CPU 限制为 1.0 核
- **那么** 内存限制为 512MB
- **那么** 重启策略为 `unless-stopped`

### 需求：卷挂载配置
系统必须支持持久化存储浏览器数据和二维码。

#### 场景：浏览器数据持久化
- **当** 容器重启
- **那么** 浏览器用户数据保持不变
- **那么** 用户无需重新扫码
- **那么** Cookie 保持登录状态

#### 场景：二维码访问
- **当** 生成新的二维码
- **那么** 二维码文件保存到挂载目录
- **那么** 用户可通过文件管理器访问
- **那么** 用户用手机扫码登录

### 需求：群晖部署适配
系统必须适配群晖 NAS218+ 环境。

#### 场景：架构兼容
- **当** 部署到群晖 DS218+（x86_64）
- **那么** Docker 镜像架构匹配
- **那么** 所有依赖正常安装

#### 场景：权限管理
- **当** 容器访问挂载卷
- **那么** 目录权限正确（755）
- **那么** 容器用户有读写权限
