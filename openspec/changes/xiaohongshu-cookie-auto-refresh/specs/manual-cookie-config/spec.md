## 新增需求

### 需求：前端手工配置 Cookie
系统必须在前端页面提供手工配置小红书 Cookie 的功能。

#### 场景：打开配置页面
- **当** 用户访问网络配置页面
- **那么** 系统显示小红书 Cookie 配置输入框
- **那么** 系统显示当前 Cookie 状态（已配置/未配置）
- **那么** 系统显示最后刷新时间

#### 场景：保存 Cookie
- **当** 用户在输入框中输入 Cookie 并点击保存
- **那么** 系统验证 Cookie 格式（包含必需字段）
- **那么** 系统将 Cookie 保存到数据库
- **那么** 系统更新 `cookie_version` 版本号
- **那么** 系统提示保存成功

#### 场景：验证 Cookie 有效性
- **当** 用户点击"测试连接"按钮
- **那么** 系统使用配置的 Cookie 调用小红书 API
- **那么** 系统返回测试结果（成功/失败）
- **那么** 如果失败，提示可能的原因（Cookie 过期/格式错误）

### 需求：Cookie 格式验证
系统必须验证手工输入的 Cookie 格式是否正确。

#### 场景：Cookie 格式正确
- **当** 用户输入的 Cookie 包含必需字段（a1, web_session, id_token）
- **那么** 系统判定为有效格式
- **那么** 系统允许保存

#### 场景：Cookie 格式错误
- **当** 用户输入的 Cookie 缺少必需字段
- **那么** 系统判定为无效格式
- **那么** 系统阻止保存
- **那么** 系统提示缺少的字段

### 需求：自动刷新与手工配置协同
系统必须协调自动刷新和手工配置的关系。

#### 场景：手工配置后自动刷新
- **当** 用户手工配置 Cookie
- **那么** 系统重置下次刷新时间（24 小时后）
- **那么** 系统记录配置来源为"manual"

#### 场景：自动刷新覆盖手工配置
- **当** 自动刷新任务执行
- **那么** 系统用新 Cookie 覆盖现有 Cookie
- **那么** 系统记录配置来源为"auto"
