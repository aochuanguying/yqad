## 新增需求

### 需求：扫码登录生成二维码
系统必须生成小红书登录二维码图片，保存到指定目录，文件名包含时间戳。小红书网页版登录需要扫描两次二维码。

#### 场景：成功生成第一次二维码
- **当** 用户触发扫码登录
- **那么** 系统在 `/tmp/qr_codes` 目录生成 PNG 格式二维码图片
- **那么** 文件名格式为 `qr_first_YYYYMMDD_HHMMSS.png`
- **那么** 二维码有效期为 5 分钟

#### 场景：成功生成第二次二维码
- **当** 用户完成第一次扫码
- **那么** 系统生成第二次二维码图片
- **那么** 文件名格式为 `qr_second_YYYYMMDD_HHMMSS.png`
- **那么** 二维码有效期为 5 分钟

#### 场景：二维码目录不存在
- **当** `/tmp/qr_codes` 目录不存在
- **那么** 系统自动创建该目录
- **那么** 目录权限设置为 755

### 需求：监听登录状态
系统必须监听小红书登录状态，检测用户是否完成两次扫码。

#### 场景：用户完成第一次扫码
- **当** 用户用手机小红书 APP 扫描第一个二维码并确认
- **那么** 系统在 60 秒内检测到第一次扫码成功
- **那么** 系统生成并显示第二个二维码

#### 场景：用户完成第二次扫码
- **当** 用户扫描第二个二维码并确认登录
- **那么** 系统在 60 秒内检测到登录成功
- **那么** 系统提取完整的 Cookie 字符串

#### 场景：第一次扫码超时
- **当** 5 分钟内用户未完成第一次扫码
- **那么** 系统判定为超时
- **那么** 系统删除第一个二维码图片
- **那么** 系统返回错误消息"扫码超时"

#### 场景：第二次扫码超时
- **当** 第一次扫码成功后 5 分钟内未完成第二次扫码
- **那么** 系统判定为超时
- **那么** 系统删除两个二维码图片
- **那么** 系统返回错误消息"第二次扫码超时"

### 需求：提取并验证 Cookie
系统必须从浏览器上下文中提取 Cookie 并验证完整性。

#### 场景：Cookie 提取成功
- **当** 登录成功
- **那么** 系统提取所有 Cookie 字段
- **那么** 系统验证包含必需字段（a1, web_session, id_token）
- **那么** 系统将 Cookie 转换为字符串格式

#### 场景：Cookie 字段缺失
- **当** Cookie 缺少必需字段
- **那么** 系统判定为无效 Cookie
- **那么** 系统返回错误消息"Cookie 不完整"
