## 新增需求

### 需求：Cookie 明文存储
系统必须将 Cookie 明文存储到数据库，无需加密。

#### 场景：保存 Cookie
- **当** 获取到新的 Cookie
- **那么** 系统直接将 Cookie 字符串存储到数据库
- **那么** 字段名为 `xiaohongshu_cookie`
- **那么** 系统记录 `last_refresh_time` 为当前时间
- **那么** 系统计算 `next_refresh_time` 为当前时间 + 24 小时

#### 场景：读取 Cookie
- **当** 搜索服务请求 Cookie
- **那么** 系统从数据库读取明文 Cookie
- **那么** 系统直接返回 Cookie 字符串

### 需求：Cookie 版本管理
系统必须支持 Cookie 版本管理，记录每次更新。

#### 场景：Cookie 更新
- **当** 刷新任务获取到新 Cookie
- **那么** 系统版本号加 1
- **那么** 系统记录 `last_refresh_time` 为当前时间
- **那么** 系统计算 `next_refresh_time` 为当前时间 + 24 小时
- **那么** 系统将旧 Cookie 版本保存到历史记录（可选）

#### 场景：查询当前版本
- **当** 搜索服务请求 Cookie
- **那么** 系统返回最新版本的 Cookie
- **那么** 系统记录读取日志（可选）

### 需求：Cookie 读取
系统必须提供 Cookie 读取接口。

#### 场景：读取成功
- **当** 搜索服务请求 Cookie
- **那么** 系统从数据库读取明文 Cookie
- **那么** 系统返回原始 Cookie 字符串

#### 场景：Cookie 不存在
- **当** 数据库中没有 Cookie
- **那么** 系统返回空值
- **那么** 系统提示用户先执行扫码登录或手工配置

### 需求：刷新历史记录
系统必须记录每次 Cookie 刷新的详细信息。

#### 场景：刷新成功
- **当** Cookie 刷新完成
- **那么** 系统记录刷新时间、Cookie 版本、耗时
- **那么** 系统记录成功状态
- **那么** 历史记录保留 30 天

#### 场景：刷新失败
- **当** Cookie 刷新失败
- **那么** 系统记录失败时间、错误消息
- **那么** 系统记录失败状态
- **那么** 系统保留旧 Cookie 继续使用
