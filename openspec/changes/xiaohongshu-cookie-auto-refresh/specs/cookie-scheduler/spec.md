## 新增需求

### 需求：定时调度器
系统必须支持基于 Cron 表达式的定时任务调度，自动触发 Cookie 刷新。

#### 场景：配置定时任务
- **当** 系统启动时
- **那么** 系统读取配置中的 Cron 表达式（默认 `0 2 * * *`）
- **那么** 系统注册定时任务
- **那么** 系统在指定时间触发刷新

#### 场景：定时任务触发
- **当** 到达设定的 Cron 时间
- **那么** 系统检查是否有正在运行的刷新任务
- **那么** 如果没有，启动新的刷新任务
- **那么** 如果有，跳过本次执行并记录日志

### 需求：并发控制
系统必须防止多个刷新任务同时运行。

#### 场景：任务正在运行
- **当** 定时任务触发时已有刷新任务在运行
- **那么** 系统跳过本次执行
- **那么** 系统记录"任务跳过：已有任务运行中"

#### 场景：任务卡死处理
- **当** 刷新任务运行超过 10 分钟未完成
- **那么** 系统判定为卡死
- **那么** 系统强制终止任务
- **那么** 系统允许下一次定时任务正常执行

### 需求：手动触发刷新
系统必须提供 API 接口供用户手动触发 Cookie 刷新。

#### 场景：API 调用成功
- **当** 用户调用 `POST /api/cookie/refresh`
- **那么** 系统立即启动刷新任务
- **那么** 系统返回任务 ID
- **那么** 系统异步执行刷新

#### 场景：API 调用时任务正在运行
- **当** 手动触发时已有刷新任务在运行
- **那么** 系统返回错误 409
- **那么** 错误消息为"刷新任务正在运行中"
