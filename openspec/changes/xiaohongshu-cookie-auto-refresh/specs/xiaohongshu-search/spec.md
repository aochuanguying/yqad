## 修改需求

### 需求：从数据库动态读取 Cookie
小红书搜索服务必须从数据库动态读取 Cookie，而非从环境变量静态读取。

#### 场景：搜索前获取 Cookie
- **当** 搜索服务初始化时
- **那么** 服务从数据库读取最新版本的 Cookie
- **那么** 服务解密 Cookie 并缓存到内存
- **那么** 服务使用 Cookie 初始化小红书客户端

#### 场景：Cookie 自动刷新后
- **当** 定时任务完成 Cookie 刷新
- **那么** 搜索服务在下一次搜索时使用新 Cookie
- **那么** 服务无需重启

#### 场景：Cookie 读取失败
- **当** 数据库连接失败或 Cookie 不存在
- **那么** 服务返回错误"Cookie 未配置"
- **那么** 服务提示用户先执行扫码登录

### 需求：连接测试集成
小红书连接测试必须集成到 Cookie 管理系统，使用最新的重试和错误处理逻辑。

#### 场景：测试连接成功
- **当** 用户调用连接测试 API
- **那么** 系统从数据库读取 Cookie
- **那么** 系统使用 XiaohongshuSearch 类的 testConnection 方法
- **那么** 系统返回测试结果（成功/失败）

#### 场景：测试连接失败
- **当** Cookie 已过期或无效
- **那么** 系统返回错误消息
- **那么** 系统提示用户重新扫码
