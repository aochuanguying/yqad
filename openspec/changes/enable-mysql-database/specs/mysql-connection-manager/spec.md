## 新增需求

### 需求:MySQL 连接管理

系统必须提供 MySQL 数据库连接管理功能，包括连接池管理、事务支持、健康检查和优雅关闭。

#### 场景:初始化连接池
- **当** 应用启动时
- **那么** 系统必须根据环境配置创建 MySQL 连接池
- **那么** 系统必须设置合理的连接数限制（测试环境 5 个，生产环境 10 个）
- **那么** 系统必须配置连接超时（10 秒）和查询超时（30 秒）

#### 场景:获取数据库连接
- **当** 业务代码需要执行 SQL 查询时
- **那么** 系统必须从连接池分配一个可用连接
- **那么** 系统必须在查询完成后自动释放连接回连接池
- **那么** 系统必须在连接超时时抛出错误

#### 场景:事务管理
- **当** 业务操作需要原子性保证时
- **那么** 系统必须支持开启事务
- **那么** 系统必须支持提交事务
- **那么** 系统必须支持回滚事务
- **那么** 系统必须在异常发生时自动回滚事务

#### 场景:健康检查
- **当** 定期健康检查触发时
- **那么** 系统必须执行简单的 SQL 查询（SELECT 1）验证连接
- **那么** 系统必须记录健康检查结果
- **那么** 系统必须在连续失败时触发告警

#### 场景:优雅关闭
- **当** 应用关闭时
- **那么** 系统必须等待所有活跃查询完成
- **那么** 系统必须释放所有连接池资源
- **那么** 系统必须在 30 秒内完成关闭或强制终止

### 需求:环境配置管理

系统必须支持根据环境变量动态加载 MySQL 配置，区分测试和生产环境。

#### 场景:加载测试环境配置
- **当** NODE_ENV=test 时
- **那么** 系统必须使用 localhost:3306 连接 MySQL
- **那么** 系统必须使用数据库名 yqad_test
- **那么** 系统必须使用较小的连接池（5 个连接）

#### 场景:加载生产环境配置
- **当** NODE_ENV=production 时
- **那么** 系统必须使用容器名 mysql:3306 连接 MySQL
- **那么** 系统必须使用数据库名 yqad_prod
- **那么** 系统必须使用较大的连接池（10 个连接）

#### 场景:环境变量覆盖
- **当** 设置 MYSQL_HOST 环境变量时
- **那么** 系统必须使用环境变量覆盖配置文件中的主机地址
- **当** 设置 MYSQL_PASSWORD 环境变量时
- **那么** 系统必须使用环境变量作为数据库密码

### 需求:降级策略

系统必须实现 MySQL 不可用时的优雅降级机制，保证业务连续性。

#### 场景:MySQL 连接失败
- **当** MySQL 服务器不可用时
- **那么** 系统必须记录错误日志
- **那么** 系统必须切换到降级模式
- **那么** 系统必须尝试从 Redis 缓存读取数据（只读）

#### 场景:降级到 Redis 缓存
- **当** MySQL 不可用且 Redis 缓存存在时
- **那么** 系统必须返回 Redis 缓存数据
- **那么** 系统必须记录降级日志

#### 场景:完全不可用
- **当** MySQL 和 Redis 都不可用时
- **那么** 系统必须返回 null 或错误
- **那么** 系统必须记录完整的错误堆栈
