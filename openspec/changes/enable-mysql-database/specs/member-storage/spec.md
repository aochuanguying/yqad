## 新增需求

### 需求:会员信息 MySQL 存储

系统必须使用 MySQL 数据库存储会员信息，包括用户基本信息、权限等级、有效期等数据。

#### 场景:创建会员
- **当** 新用户注册时
- **那么** 系统必须在 members 表中插入一条新记录
- **那么** 系统必须生成唯一的会员 ID（UUID）
- **那么** 系统必须加密存储密码（bcrypt）
- **那么** 系统必须设置默认会员等级为 free
- **那么** 系统必须记录创建时间

#### 场景:查询会员信息
- **当** 根据会员 ID 查询时
- **那么** 系统必须返回完整的会员信息
- **那么** 系统必须包含会员等级和有效期
- **那么** 系统必须不包含密码哈希

#### 场景:更新会员信息
- **当** 会员信息变更时
- **那么** 系统必须更新 members 表对应记录
- **那么** 系统必须记录更新时间
- **那么** 系统必须返回更新后的会员信息

#### 场景:更新会员等级
- **当** 会员购买服务或升级时
- **那么** 系统必须更新 member_level 字段
- **那么** 系统必须更新 expires_at 字段
- **那么** 系统必须记录操作日志

#### 场景:删除会员
- **当** 会员注销时
- **那么** 系统必须软删除会员（标记 deleted_at）
- **那么** 系统必须保留历史数据用于审计
- **那么** 系统必须级联删除关联的帖子和评论

#### 场景:查询会员列表
- **当** 管理员查询会员列表时
- **那么** 系统必须支持分页查询
- **那么** 系统必须支持按会员等级筛选
- **那么** 系统必须支持按注册时间排序
- **那么** 系统必须支持按过期时间排序

### 需求:会员认证 MySQL 存储

系统必须使用 MySQL 数据库存储会员认证相关信息，支持登录验证和 Token 刷新。

#### 场景:登录验证
- **当** 用户提交用户名和密码时
- **那么** 系统必须查询 members 表验证用户名
- **那么** 系统必须比对密码哈希
- **那么** 系统必须检查会员状态（是否被禁用）
- **那么** 系统必须检查会员有效期
- **那么** 系统必须返回 JWT Token

#### 场景:Token 刷新
- **当** Token 即将过期时
- **那么** 系统必须验证 Refresh Token
- **那么** 系统必须验证会员状态
- **那么** 系统必须生成新的 JWT Token
- **那么** 系统必须更新最后登录时间

#### 场景:密码重置
- **当** 用户申请密码重置时
- **那么** 系统必须验证用户身份
- **那么** 系统必须生成重置 Token
- **那么** 系统必须设置 Token 过期时间（1 小时）
- **当** 用户提交新密码时
- **那么** 系统必须验证重置 Token
- **那么** 系统必须加密存储新密码
- **那么** 系统必须使重置 Token 失效

### 需求:会员数据完整性

系统必须保证会员数据的完整性和一致性，使用外键约束和事务。

#### 场景:外键约束
- **当** 删除会员时
- **那么** 系统必须级联删除该会员的帖子
- **那么** 系统必须级联删除该会员的评论
- **那么** 系统必须禁止删除还有关联数据的会员（如果未设置级联）

#### 场景:唯一约束
- **当** 创建会员时
- **那么** 系统必须保证 username 唯一
- **那么** 系统必须保证 email 唯一
- **当** 重复创建时
- **那么** 系统必须抛出唯一约束错误

#### 场景:事务一致性
- **当** 跨表操作时（如发帖并更新计数）
- **那么** 系统必须使用事务保证原子性
- **那么** 系统必须在失败时回滚所有操作
- **那么** 系统必须保证数据最终一致性
