## 新增需求

### 需求：手机配置页签导航

系统必须在首页导航栏中添加"手机配置"页签。

页签位置：与其他现有页签并列（如"首页"、"AutoJS"等）。

#### 场景：显示手机配置页签
- **当** 用户访问首页
- **那么** 导航栏显示"手机配置"页签

#### 场景：切换到手机配置页面
- **当** 用户点击"手机配置"页签
- **那么** 页面切换到手机配置内容区域

### 需求：手机配置页面结构

系统必须在手机配置页面中显示三个子功能入口：
1. 手机短信
2. 未接电话
3. AutoJS 配置

页面采用页签或卡片形式组织这三个子功能。

#### 场景：显示三个子功能
- **当** 用户进入手机配置页面
- **那么** 页面显示"手机短信"、"未接电话"、"AutoJS 配置"三个子功能入口

### 需求：复用 AutoJS 配置

系统必须将现有的 AutoJS 配置页面（`autojs-config.html`）集成到手机配置页面中作为子功能。

#### 场景：访问 AutoJS 配置
- **当** 用户在手机配置页面点击"AutoJS 配置"
- **那么** 显示与原 `autojs-config.html` 相同的功能界面
