## 新增需求

### 需求：系统必须记录 Pipeline 各步骤的执行时长

系统必须在发帖 Pipeline 的每个关键步骤记录执行时长，用于性能分析和瓶颈识别。

#### 场景：记录步骤开始和结束时间

- **当** Pipeline 执行到任意关键步骤（子方向选择、内容生成、素材选择、图片上传、话题匹配、多样化变换、合规检查、发布）
- **那么** 系统必须记录该步骤的开始时间戳和结束时间戳

#### 场景：计算步骤执行时长

- **当** 步骤执行完成
- **那么** 系统必须计算并记录该步骤的执行时长（毫秒）

#### 场景：记录步骤中间状态

- **当** 步骤执行过程中产生重要中间结果
- **那么** 系统必须记录中间状态（如：选择的子方向索引、生成的标题、匹配的素材数量等）

### 需求：系统必须记录发帖过程的详细上下文

系统必须记录发帖过程的完整上下文信息，便于故障排查和数据分析。

#### 场景：记录发帖模式信息

- **当** 开始执行发帖任务
- **那么** 系统必须记录发帖模式（主题发帖/自由发帖）、触发方式（自动/手动）、发帖等级（普通/精华）

#### 场景：记录配置参数

- **当** 执行发帖任务
- **那么** 系统必须记录使用的配置参数（如：每日限额、精华帖配置、素材选择策略等）

#### 场景：记录重试信息

- **当** 发帖过程中发生重试
- **那么** 系统必须记录重试次数、重试原因、每次重试的结果

### 需求：系统必须以异步方式记录日志

系统必须采用异步非阻塞方式记录日志，确保不影响发帖主流程的性能。

#### 场景：异步执行日志记录

- **当** 需要记录日志时
- **那么** 系统必须使用 `setImmediate` 或 `process.nextTick` 异步执行，不阻塞主流程

#### 场景：日志记录超时保护

- **当** 日志记录操作超过 5 秒未完成
- **那么** 系统必须终止日志记录操作，避免长时间占用资源

#### 场景：日志记录失败处理

- **当** 日志记录失败时
- **那么** 系统必须记录错误到控制台，但不影响发帖流程
