## 新增需求

### 需求：系统必须提供发帖成功率统计

系统必须提供发帖成功率的多维度统计，用于评估系统健康度。

#### 场景：按发帖模式统计成功率

- **当** 查询发帖成功率时
- **那么** 系统必须分别返回主题发帖和自由发帖的成功率

#### 场景：按时间维度统计成功率

- **当** 查询成功率趋势时
- **那么** 系统必须支持按小时、天、周维度统计成功率

#### 场景：按失败原因分类统计

- **当** 分析发帖失败原因时
- **那么** 系统必须按失败原因分类统计（合规检查失败、图片上传失败、发布失败等）

### 需求：系统必须提供发帖耗时统计

系统必须提供发帖过程的耗时统计，用于性能监控和优化。

#### 场景：统计平均发帖耗时

- **当** 查询性能指标时
- **那么** 系统必须返回平均发帖耗时（从开始到发布的总时长）

#### 场景：统计各环节耗时分布

- **当** 分析性能瓶颈时
- **那么** 系统必须返回 Pipeline 各环节的耗时分布（P50、P90、P99）

#### 场景：统计耗时趋势

- **当** 查询性能趋势时
- **那么** 系统必须返回按时间维度统计的耗时趋势图数据

### 需求：系统必须提供环节转化率统计

系统必须提供 Pipeline 各环节的转化率统计，用于识别流程瓶颈。

#### 场景：统计各环节通过率

- **当** 查询转化率时
- **那么** 系统必须返回每个 Pipeline 步骤的通过率（通过数/总尝试数）

#### 场景：统计降级发生率

- **当** 查询质量指标时
- **那么** 系统必须返回精华帖降级为普通帖的发生率和原因分布

#### 场景：统计重试成功率

- **当** 查询重试效果时
- **那么** 系统必须返回重试操作的成功率（重试后成功数/总重试数）

### 需求：系统必须提供实时监控指标

系统必须提供关键指标的实时监控，支持系统健康度评估。

#### 场景：查询当前小时指标

- **当** 查询实时状态时
- **那么** 系统必须返回当前小时的发帖数量、成功率、平均耗时

#### 场景：查询今日累计指标

- **当** 查询今日状态时
- **那么** 系统必须返回今日累计发帖数量、成功率、与昨日的对比

#### 场景：查询异常告警

- **当** 指标异常时（成功率低于阈值、耗时超过阈值）
- **那么** 系统必须记录告警日志，便于及时发现和处理问题
