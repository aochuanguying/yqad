## 新增需求

### 需求：日志表必须包含性能指标字段

`post_logs` 表必须扩展性能指标相关字段，支持详细的性能分析。

#### 场景：记录 Pipeline 步骤耗时

- **当** 创建或更新日志记录时
- **那么** 系统必须支持存储 Pipeline 各步骤的耗时信息（JSON 格式）

#### 场景：记录总执行时长

- **当** 发帖任务完成时
- **那么** 系统必须记录从开始到结束的总执行时长（毫秒）

#### 场景：记录资源使用情况

- **当** 发帖任务完成时
- **那么** 系统必须记录资源使用情况（如：图片上传数量、API 调用次数等）

### 需求：日志表必须包含调试信息字段

`post_logs` 表必须扩展调试信息相关字段，支持深度问题排查。

#### 场景：记录错误堆栈

- **当** 发帖失败时
- **那么** 系统必须支持存储完整的错误堆栈信息（TEXT 类型）

#### 场景：记录上下文快照

- **当** 记录日志时
- **那么** 系统必须支持存储关键上下文信息的快照（JSON 格式）

#### 场景：记录重试历史

- **当** 发帖过程中发生重试时
- **那么** 系统必须支持存储重试历史记录（JSON 数组）

### 需求：日志表必须添加常用查询索引

`post_logs` 表必须添加常用查询维度的索引，提升查询性能。

#### 场景：按时间范围查询

- **当** 按时间范围查询日志时
- **那么** 系统必须使用 `created_at` 索引快速定位数据

#### 场景：按状态和触发方式组合查询

- **当** 按状态和触发方式组合筛选时
- **那么** 系统必须使用复合索引 `(status, trigger_type, created_at)` 加速查询

#### 场景：按任务 ID 查询

- **当** 通过 `task_id` 查询日志时
- **那么** 系统必须使用 `task_id` 索引快速定位数据

## 修改需求

### 需求：日志记录接口必须支持扩展字段

**修改说明**：扩展 `createPostLog` 和 `updatePostLog` 接口，支持新增的性能指标和调试信息字段。

#### 场景：创建包含性能指标的日志

- **当** 调用 `createPostLog` 创建日志时
- **那么** 系统必须支持传入 `pipelineTimings`、`totalDuration` 等性能指标字段

#### 场景：更新包含调试信息的日志

- **当** 调用 `updatePostLog` 更新日志时
- **那么** 系统必须支持更新 `errorStack`、`contextSnapshot`、`retryHistory` 等调试信息字段

#### 场景：查询包含扩展信息的日志

- **当** 调用 `getPostLogById` 或 `queryPostLogs` 查询日志时
- **那么** 系统必须返回所有扩展字段的信息

### 需求：日志查询接口必须支持性能指标筛选

**修改说明**：扩展日志查询接口，支持按性能指标进行筛选和排序。

#### 场景：按执行时长筛选

- **当** 查询日志时
- **那么** 系统必须支持按 `totalDuration` 进行范围筛选（如：查询耗时超过 100 秒的记录）

#### 场景：按 Pipeline 步骤耗时排序

- **当** 分析性能瓶颈时
- **那么** 系统必须支持按特定 Pipeline 步骤的耗时进行排序

#### 场景：统计性能指标

- **当** 调用 `getStats` 接口时
- **那么** 系统必须返回性能指标统计（平均耗时、成功率、各环节转化率等）
