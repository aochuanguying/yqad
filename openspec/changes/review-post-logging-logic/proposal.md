## 为什么

当前发帖日志系统虽已实现基础功能，但在实际运行中存在多个潜在问题：日志记录不完整导致故障排查困难、缺少关键性能指标监控、异常场景覆盖不足、以及与发帖主流程的协同存在断层。近期多次发帖失败后无法快速定位根因，亟需对发帖日志逻辑进行全面复盘和优化。

## 变更内容

本次变更将对发帖日志系统进行全面复盘和优化：

1. **完善日志记录维度**：增加 Pipeline 各关键步骤的执行时长、中间状态、失败原因等详细信息
2. **增强错误追踪能力**：记录完整的错误堆栈、上下文参数、重试次数等调试信息
3. **优化日志查询性能**：添加常用查询维度的索引、优化分页查询效率
4. **补充监控指标**：记录发帖成功率、平均耗时、各环节转化率等关键指标
5. **改进异常处理**：增加超时保护、降级策略、异常日志记录

## 功能 (Capabilities)

### 新增功能
- `logging-enhancement`: 增强日志记录维度，包括 Pipeline 步骤耗时、中间状态、详细错误信息
- `monitoring-metrics`: 添加发帖过程监控指标，包括成功率、耗时统计、环节转化率
- `error-tracking`: 完善错误追踪机制，包括堆栈记录、上下文快照、重试历史

### 修改功能
- `post-logging`: 扩展日志表结构，增加性能指标字段和调试信息字段
