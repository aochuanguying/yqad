## 新增需求

### 需求：系统必须提供远程发帖内容生成 API
系统必须提供 RESTful API 接口，供手机端调用以生成发帖内容（标题、正文、图片下载地址），保留所有核心发帖逻辑（主题选择、内容生成、图片匹配、精华策略）。

#### 场景：同步生成单篇帖子
- **当** 手机端发送 POST 请求到 `/api/posts/generate` 并提供有效 token
- **那么** 系统必须生成一篇帖子并返回标题、正文、图片下载地址、发帖模式和元数据

#### 场景：使用主题生成帖子
- **当** 存在未使用的主题配置且 `useTopic` 参数为 true（默认）
- **那么** 系统必须优先使用主题方向、提纲和关联素材生成帖子

#### 场景：自由生成帖子
- **当** 无可用主题或 `useTopic` 参数为 false
- **那么** 系统必须使用互联网参考模式自由生成帖子

#### 场景：精华帖模式生成
- **当** 配置 `featuredPosting.enabled` 为 true 且请求未指定模式
- **那么** 系统必须尝试生成精华帖，若不达标则降级为普通帖

#### 场景：指定发帖模式
- **当** 请求中明确指定 `mode` 参数（'featured' 或 'normal'）
- **那么** 系统必须使用指定模式生成帖子

#### 场景：推荐话题关联
- **当** 帖子生成成功
- **那么** 系统必须基于 AI 语义匹配推荐 0-5 个话题供手机端参考

#### 场景：API 鉴权
- **当** 请求头缺少 Authorization 或 token 无效
- **那么** 系统必须返回 401 错误

#### 场景：频率限制
- **当** 同一设备 1 小时内请求超过 10 次
- **那么** 系统必须返回 429 错误

### 需求：系统必须提供异步批量生成 API
系统必须提供异步任务接口，支持批量生成多篇帖子，手机端可通过任务 ID 查询进度和获取结果。

#### 场景：创建批量任务
- **当** 手机端发送 POST 请求到 `/api/posts/batch` 并指定 `count` 参数
- **那么** 系统必须创建异步任务并返回任务 ID

#### 场景：查询任务状态
- **当** 手机端发送 GET 请求到 `/api/posts/tasks/:id`
- **那么** 系统必须返回任务状态（pending/processing/completed/failed）和进度信息

#### 场景：获取任务结果
- **当** 任务状态为 completed
- **那么** 系统必须返回所有生成的帖子内容列表

#### 场景：任务过期清理
- **当** 任务创建超过 30 分钟
- **那么** 系统必须自���清理该任务数据

## 修改需求

### 需求：发帖服务从定时触发改为 API 触发
系统必须支持通过配置项 `post.mode` 控制发帖触发方式，当模式为 'api' 时禁用定时发帖任务。

#### 场景：API 模式配置
- **当** 配置 `post.mode` 为 'api'
- **那么** 系统必须禁用定时发帖任务，仅响应 API 请求

#### 场景：定时模式配置（向后兼容）
- **当** 配置 `post.mode` 为 'scheduled' 或未配置
- **那么** 系统必须保留原有的定时发帖功能

#### 场景：模式切换
- **当** 配置从 'scheduled' 改为 'api'
- **那么** 系统必须在下一次定时任务触发前停止发帖
