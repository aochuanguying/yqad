## 新增需求

### 需求：系统必须提供 API Token 配置管理功能
系统必须提供 API Token 的生成、存储、读取和重置功能，支持通过配置文件持久化 Token。

#### 场景：生成新 Token
- **当** 调用 Token 生成函数
- **那么** 系统必须生成一个加密安全的随机 Token（32 字节，hex 编码，格式：`api_token_<64 字符>`）

#### 场景：保存 Token
- **当** 生成或重置 Token 后
- **那么** 系统必须将 Token 保存到配置文件（`data/api-token.json`），包含 `token`、`createdAt` 字段

#### 场景：读取 Token
- **当** 验证中间件需要验证 Token 时
- **那么** 系统必须能够从配置文件中读取已存储的 Token

#### 场景：重置 Token
- **当** 调用 Token 重置函数
- **那么** 系统必须生成新 Token 并覆盖旧 Token，同时更新 `createdAt` 时间戳

#### 场景：Token 文件权限
- **当** 创建或更新 Token 配置文件时
- **那么** 文件权限必须设置为 600（仅所有者可读写）

### 需求：系统必须提供 API Token 状态查询功能
系统必须提供查询 API Token 配置状态的接口，包括是否已配置、创建时间等信息（不暴露完整 Token）。

#### 场景：查询 Token 状态
- **当** 调用 Token 状态查询函数
- **那么** 系统必须返回 Token 是否存在、创建时间、最后使用时间（不返回 Token 本身）
