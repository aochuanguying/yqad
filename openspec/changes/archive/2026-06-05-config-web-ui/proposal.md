## 为什么

当前所有配置项存储在 `config/default.yaml` 中，修改配置需要直接编辑 YAML 文件并重启服务。对于非技术人员或远程维护场景，这种方式不够友好，容易因格式错误导致服务异常。需要一个可视化的网页界面来查看和管理配置项，降低运维门槛。

## 变更内容

- **新增**：基于 Express 的轻量级 Web 服务器，提供配置管理的 HTTP API
- **新增**：单页面 Web UI，以表单方式展示和编辑所有配置项（API、认证、AI模型、签到、评论、发帖、调度、日志等）
- **新增**：配置项分组展示，支持实时编辑和保存到 `config/default.yaml`
- **新增**：配置变更后的服务热重载能力

## 功能 (Capabilities)

### 新增功能

- `config-web-ui`: 提供配置管理的 Web 界面，包含 HTTP API 和前端页面，支持配置的查看、编辑和持久化

### 修改功能

- `scheduler`: 调度器需要支持配置热重载，当通过 Web UI 修改调度配置后无需重启即可生效

## 影响

- **依赖新增**：`express`（HTTP 服务器）
- **代码新增**：`src/web/` 目录，包含路由、控制器和静态资源
- **配置文件**：`config/default.yaml` 将通过 API 被读写，需确保并发安全
- **端口占用**：新增 HTTP 服务端口（默认 3000）
- **启动流程**：主入口需集成 Web 服务器的启动
