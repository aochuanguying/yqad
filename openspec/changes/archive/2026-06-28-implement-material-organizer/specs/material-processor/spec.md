## 新增需求

### 需求：提取图片元数据
系统必须为每个新素材提取基础元数据，包括文件尺寸、分辨率、格式。

#### 场景：成功提取元数据
- **当** 处理新图片文件
- **那么** 系统使用 sharp 库提取 width、height、format、fileSize 信息

#### 场景：HEIC 格式处理
- **当** 处理 HEIC 格式图片
- **那么** 系统记录格式信息，如 sharp 不支持则跳过元数据提取，不影响录入

#### 场景：损坏文件处理
- **当** 图片文件损坏无法读取
- **那么** 系统记录错误日志，跳过元数据提取，继续处理其他文件

### 需求：AI 生成描述
系统必须使用 AI 为每个新素材生成自然语言描述，用于语义搜索。

#### 场景：成功生成描述
- **当** 处理新素材且元数据提取完成
- **那么** 系统调用 AI 生成 1-2 句描述，说明图片内容、场景、特点

#### 场景：AI 服务不可用
- **当** AI 服务调用失败
- **那么** 系统使用文件名作为降级描述，记录警告日志

#### 场景：描述长度控制
- **当** AI 生成描述超过 200 字
- **那么** 系统截断到 200 字以内

### 需求：AI 生成标签
系统必须为每个新素材生成 3-5 个标签，用于分类和检索。

#### 场景：成功生成标签
- **当** 处理新素材
- **那么** 系统调用 AI 生成 3-5 个标签，涵盖场景、物体、情感维度

#### 场景：标签去重
- **当** AI 生成重复或相似标签
- **那么** 系统去重，保持唯一性

#### 场景：标签格式标准化
- **当** AI 生成标签包含特殊字符
- **那么** 系统清洗标签，只保留中文、英文、数字

### 需求：批量处理支持
系统必须支持批量处理素材，控制并发数和内存使用。

#### 场景：分批处理
- **当** 处理超过 50 个素材
- **那么** 系统分批次处理，每批最多 50 个素材

#### 场景：批次间延迟
- **当** 完成一批处理
- **那么** 系统暂停 1 秒，避免 AI 服务限流

#### 场景：进度日志
- **当** 处理每个素材
- **那么** 系统记录进度（当前/总数）和耗时
