## 上下文

**当前状态**：
- HybridMaterialService 已实现素材选择和混合策略
- MySQL material_records 表已创建，支持素材记录存储
- ChromaDB prod_materials 集合已创建，支持向量搜索
- 调度器支持间隔模式和 Cron 模式触发素材整理任务
- **缺失**：素材扫描、素材处理、向量同步的实际实现

**约束**：
- 保持与现有 HybridMaterialService 接口兼容
- 使用现有数据库 Schema（无需迁移）
- 支持现有配置结构（scheduler.materialProcessing）
- 遵循项目现有的日志、错误处理模式

**利益相关者**：
- 自动发帖系统（依赖素材选择）
- Web 管理界面（素材管理功能）
- 定时调度器（素材整理任务）

## 目标 / 非目标

**目标：**
1. 实现文件系统扫描器，自动发现并录入新素材
2. 实现素材处理器，提取元数据、生成描述和标签
3. 实现完整的素材整理 Pipeline
4. 批量同步历史素材到 ChromaDB
5. 支持并发控制和错误恢复

**非目标：**
1. 不实现复杂的图像识别（使用 AI 文本生成描述）
2. 不实现素材去重（留给后续优化）
3. 不实现素材质量自动提升（裁剪、增强等）
4. 不修改现有发帖流程

## 决策

### 1. 扫描策略：轮询 vs 监听

**选择：轮询模式**

**理由**：
- 简单可靠，不依赖文件系统事件（在不同 OS 上行为不一致）
- 与现有调度器架构一致（定时/间隔触发）
- 易于控制扫描频率，避免频繁 IO
- 支持批量处理，效率更高

**替代方案**：
- chokidar 监听：实时性好，但事件风暴处理复杂，跨平台兼容性差

### 2. 素材处理：本地处理 vs AI 生成

**选择：AI 生成描述和标签**

**理由**：
- 项目已有 AI 内容生成基础设施
- 描述质量更高，更适合语义搜索
- 自动生成标签更准确（场景、物体、情感）

**实现**：
- 使用现有 generateContent() API
- 提取基础元数据（尺寸、格式、文件大小）使用 sharp
- AI 生成：描述、标签、OCR 文本（如果有）

### 3. 向量生成：实时 vs 批量

**选择：批量同步**

**理由**：
- 素材整理是批量操作，批量生成向量更高效
- 减少 ChromaDB 连接开销
- 支持错误重试和断点续传
- 可以控制批次大小，避免内存溢出

**实现**：
- 新录入素材：实时生成单个向量
- 历史素材：批量任务，每批 100 条
- 使用 embedding-vectorizer 统一生成

### 4. 并发控制：数据库锁 vs 内存标志

**选择：内存标志 + 数据库事务**

**理由**：
- 单实例部署，内存标志足够
- 简单高效，避免数据库锁开销
- 关键操作（录入、更新）使用事务保证一致性

**实现**：
- scheduler.materialRunning 标志（已存在）
- 扫描器：基于 hash 去重，避免重复录入
- 事务：upsertMaterialRecord 已支持

### 5. 错误处理：严格失败 vs 容错继续

**选择：容错继续 + 详细日志**

**理由**：
- 素材整理是后台任务，不应阻塞整个流程
- 单个素材失败不影响其他素材
- 详细日志便于后续排查和手动修复

**实现**：
- try-catch 包裹每个素材处理
- 记录失败素材 ID 和错误信息
- 返回统计：成功数、失败数、跳过数

## 风险 / 权衡

### [风险] AI 生成描述成本高
**缓解**：
- 限制描述生成仅用于新素材
- 批量处理时使用批量 API（如果支持）
- 缓存结果，避免重复生成

### [风险] 大量素材扫描耗时
**缓解**：
- 基于 hash 去重，跳过已存在素材
- 分批处理，每批 50 个
- 支持增量扫描（仅扫描新文件）

### [风险] ChromaDB 向量生成失败
**缓解**：
- 向量生成失败不影响 MySQL 录入
- 支持后续手动同步（工具函数）
- 记录失败素材，便于重试

### [风险] 图片处理依赖 sharp
**缓解**：
- sharp 是成熟的 Node.js 图片处理库
- 降级方案：无法提取元数据时跳过，不影响录入
- Docker 镜像已包含 sharp 依赖

## 迁移计划

**部署步骤**：
1. 安装依赖：sharp（图片处理）
2. 部署新代码
3. 配置扫描路径（config.default.yaml）
4. 手动触发一次素材整理，验证功能
5. 启用定时任务（可选）
6. 运行批量向量同步工具（一次性）

**回滚策略**：
- 新代码向后兼容，回滚无特殊要求
- 已录入素材保留在数据库，可手动清理
