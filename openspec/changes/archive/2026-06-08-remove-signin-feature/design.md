## 上下文

当前系统包含签到功能，但由于真实 API 缺少必要的签名算法（x-fawvw-sign），签到功能实际上无法使用。配置中已默认禁用签到（`signin.enabled: false`）。

## 目标 / 非目标

**目标：**
- 完全移除签到相关的所有代码和配置
- 保持系统其他功能正常运行
- 简化代码库，减少维护负担

**非目标：**
- 不实现新的签到方案
- 不修改其他功能的逻辑（除了移除对签到的引用）

## 决策

**完全删除而非保留代码**
- 签到 API 因缺少签名算法无法工作，且短期内无解决方案
- 保留无用代码会增加维护成本和理解难度
- 如未来需要签到功能，可基于新的 API 文档重新实现

**更新每日摘要结构**
- 移除 `DailySummary` 接口中的 `signin` 字段
- 调整 `generateDailySummary` 函数签名，移除 `signinResult` 参数
- 简化 `checkAlerts` 函数，移除签到相关的告警逻辑

## 风险 / 权衡

**风险：** 用户可能依赖签到配置项
- **缓解措施：** 签到功能从未实际工作过，且默认禁用，影响范围极小

**风险：** 编译后的 dist 文件需要同步清理
- **缓解措施：** 手动删除 dist 目录中的签到相关文件
