## 修改需求

### 需求: 系统必须自动生成并发布优质帖子
系统必须定时发布帖子到社区。发帖时必须优先使用已配置且未使用的主题（含关联素材和提纲），当所有主题用完或无主题配置时，回退到基于 AI 自由生成的模式。系统必须在可用时优先按“精华策略”生成精华候选帖；当精华候选无法满足最低门槛时，系统必须自动降级为普通帖子发布。

#### 场景: 优先使用已配置主题发帖
- **当** 自动发帖任务触发且存在"未使用"状态的主题
- **那么** 系统必须选取一个未使用的主题，使用其方向描述和内容提纲作为 AI 生成的约束，生成帖子内容，并附带主题关联的素材图片

#### 场景: 无可用主题时自由生成
- **当** 自动发帖任务触发且所有主题均已使用（或无主题配置）
- **那么** 系统必须回退到原有模式：从内容分析结果中选取热点话题，调用 AI 自由生成帖子

#### 场景: 优先按精华策略生成精华候选帖
- **当** 自动发帖任务准备生成帖子内容且配置 `featuredPosting.enabled` 为启用
- **那么** 系统必须优先使用精华策略生成“精华候选帖”（更严格的写作约束与图片数量目标），并在发布前对候选帖进行精华门槛校验

#### 场景: 精华候选不达标时自动降级为普通帖
- **当** 精华候选帖不满足精华最低门槛（至少包含：正文字符数不足、或上传成功图片数量不足）
- **那么** 系统必须切换为普通发帖策略继续执行，并记录降级原因

#### 场景: 发帖后标记主题已使用
- **当** 使用预配置主题成功发帖
- **那么** 系统必须将该主题标记为"已使用"并记录使用时间

#### 场景: 发布帖子
- **当** 帖子内容生成完毕且（若启用精华策略）已完成精华门槛校验与必要的降级决策
- **那么** 系统必须调用发帖 API 将帖子发布到社区

#### 场景: 每日发帖数量可配置
- **当** 执行每日发帖任务
- **那么** 系统必须根据配置的每日发帖数量上限（默认1篇）执行发帖

#### 场景: 发帖主题配置使用文件选择器指定素材
- **当** 用户在 Web 管理界面配置发帖主题并需要附加素材
- **那么** 素材字段必须通过文件选择器从素材库中选取，配置中存储相对路径（如 `images/banner.jpg`），禁止存储绝对路径或手动输入任意字符串

#### 场景: 发帖时读取素材文件
- **当** 发帖任务执行且主题配置中包含素材相对路径
- **那么** 系统必须将相对路径与素材库根目录（`materials.basePath`）拼接得到完整路径后读取文件
