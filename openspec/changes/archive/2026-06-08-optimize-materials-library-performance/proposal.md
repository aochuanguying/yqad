## 为什么

当前素材库页面在素材数量较多（100+ 张）时出现明显卡顿，尤其是在滚动浏览时。问题根源在于一次性渲染所有素材卡片（150+ 个 DOM 节点）和同时加载所有图片，导致浏览器渲染压力大、内存占用高、滚动帧率低。

## 变更内容

- **新增**：素材列表分页加载功能，初始仅渲染首屏素材（24 个），支持"加载更多"按钮逐步加载
- **新增**：图片懒加载增强，使用 Intersection Observer API 实现更精确的懒加载控制
- **优化**：素材卡片渲染逻辑，支持按页渲染和进度显示
- **优化**：减少首屏 DOM 节点数量和图片并发请求数

## 功能 (Capabilities)

### 新增功能
- `materials-pagination`: 素材列表分页加载功能，支持按需加载和进度显示

### 修改功能
- `post-materials`: Web UI 素材库浏览界面的渲染方式从一次性全量渲染改为分页渲染

## 影响

- **Web UI**: 素材库 Tab 的渲染逻辑需要支持分页和懒加载
- **性能**: 首屏渲染时间预计减少 80%，滚动帧率显著提升，内存占用降低 90%
- **用户体验**: 用户可感知加载进度，操作更加流畅
