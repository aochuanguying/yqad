## 新增需求

### 需求：系统必须智能推荐最优搜索平台

系统应当根据发帖目的、内容类型、目标受众等因素，智能推荐最适合的搜索平台，以提升素材质量和发帖效果。

#### 场景：种草推荐场景
- **当** 发帖目的为"种草推荐"或"生活分享"时
- **那么** 系统必须优先推荐小红书平台，其次为汽车之家，最后为知乎

#### 场景：专业分析场景
- **当** 发帖目的为"专业分析"或"技术解读"时
- **那么** 系统必须优先推荐知乎平台，其次为汽车之家，最后为小红书

#### 场景：车主口碑场景
- **当** 发帖目的为"车主口碑"或"实用分享"时
- **那么** 系统必须优先推荐汽车之家平台，其次为小红书，最后为知乎

#### 场景：提车日记场景
- **当** 内容类型为"提车日记"且图片质量高时
- **那么** 系统必须推荐小红书平台

#### 场景：提车日记场景价格详细
- **当** 内容类型为"提车日记"且价格配置详细时
- **那么** 系统必须推荐汽车之家平台

### 需求：系统必须实现优先级 + 轮询混合选择策略

系统应当采用"智能推荐 + 轮询"的混合策略选择搜索平台，避免单一平台过度使用和频率限制问题。

#### 场景：计算基础优先级
- **当** 选择搜索平台时
- **那么** 系统必须根据发帖目的查询配置表，获取各平台的基础优先级（1-10 分）

#### 场景：频率限制调整
- **当** 某平台剩余查询次数不足时
- **那么** 系统必须降低该平台的优先级，剩余次数越少优先级降低越多

#### 场景：成功率调整
- **当** 某平台历史搜索成功率较高时
- **那么** 系统必须提升该平台的优先级，成功率越高优先级提升越多

#### 场景：最近使用惩罚
- **当** 某平台刚被使用过时
- **那么** 系统必须在下一次选择时降低该平台的优先级，避免连续使用同一平台

#### 场景：权重随机选择
- **当** 完成所有优先级调整后
- **那么** 系统必须使用权重随机算法选择平台，优先级越高被选中的概率越大

### 需求：系统必须记录平台使用统计

系统应当记录各平台的使用情况，包括查询次数、成功率、素材质量等指标，用于优化选择策略。

#### 场景：记录查询次数
- **当** 使用��平台进行搜索时
- **那么** 系统必须记录该平台的查询次数，并更新到数据库和 Redis 缓存

#### 场景：记录成功率
- **当** 搜索完成后
- **那么** 系统必须记录该次搜索是否成功，并更新该平台的累计成功率

#### 场景：记录素材质量
- **当** 搜索到的素材被选中使用时
- **那么** 系统必须记录该平台的素材质量评分，用于评估平台效果

#### 场景：统计重置
- **当** 每天零点时
- **那么** 系统必须重置各平台的每小时查询次数计数器

### 需求：系统必须支持平台优先级配置

系统应当支持通过数据库配置各平台的基础优先级，并支持动态更新。

#### 场景：查询平台优先级
- **当** 调用 `getPlatformPriorities()` 接口时
- **那么** 系统必须返回所有平台的当前优先级配置

#### 场景：更新平台优先级
- **当** 调用 `updatePlatformPriority(platform, priority)` 接口时
- **那么** 系统必须更新数据库中该平台的优先级，并清除 Redis 缓存

#### 场景：优先级范围校验
- **当** 设置的优先级不在 1-10 范围内时
- **那么** 系统必须拒绝更新并返回错误提示
