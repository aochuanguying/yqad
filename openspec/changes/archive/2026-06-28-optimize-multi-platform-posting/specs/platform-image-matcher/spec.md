## 新增需求

### 需求：系统必须根据平台偏好选择图片

系统应当根据不同平台的图片风格偏好（小红书精美风、知乎信息图、汽车之家实拍图）智能选择素材图片，以提升发帖效果。

#### 场景：小红书图片选择
- **当** 目标平台为小红书时
- **那么** 系统必须优先选择高清（清晰度>15）、有滤镜、构图精美的图片，按质量总分降序排列

#### 场景：知乎图片选择
- **当** 目标平台为知乎时
- **那么** 系统必须优先选择包含图表、数据、示意图的图片，关键词匹配"图表"或"数据"

#### 场景：汽车之家图片选择
- **当** 目标平台为汽车之家时
- **那么** 系统必须优先选择实拍图、细节图、改装图的图片，关键词匹配"实拍"或"细节"

#### 场景：图片数量不足
- **当** 符合平台偏好的图片数量不足时
- **那么** 系统必须降低筛选标准，先按质量分数补充图片

### 需求：系统必须支持图片质量评分

系统应当对每张图片进行质量评分，包括清晰度、美观度、信息量等维度，用于图片选择排序。

#### 场景：清晰度评分
- **当** 评估图片清晰度时
- **那么** 系统必须根据图片分辨率、噪点程度、对焦准确度给出 1-20 分的清晰度评分

#### 场景：美观度评分
- **当** 评估图片美观度时
- **那么** 系统必须根据构图、色彩、光影效果给出 1-20 分的美观度评分

#### 场景：信息量评分
- **当** 评估图片信息量时
- **那么** 系统必须根据图片包含的产品细节、场景信息、文字说明给出 1-20 分的信息量评分

#### 场景：总分计算
- **当** 计算图片总分时
- **那么** 系统必须使用加权平均：总分 = 清晰度*0.4 + 美观度*0.3 + 信息量*0.3

### 需求：系统必须支持图片关键词匹配

系统应当为每张图片提取关键词标签，用于平台偏好匹配和搜索。

#### 场景：提取图片关键词
- **当** 处理新图片时
- **那么** 系统必须使用图像识别或元数据分析提取图片关键词，如"实拍"、"图表"、"改装"等

#### 场景：关键词权重
- **当** 多个关键词匹配时
- **那么** 系统必须根据关键词与平台偏好的相关度计算权重，优先选择高权重关键词

#### 场景：关键词更新
- **当** 手动标注图片关键词时
- **那么** 系统必须更新数据库中的关键词记录，并同步到 ChromaDB 向量库

### 需求：系统必须支持图片降级策略

系统应当在不同场景下实现图片降级策略，确保发帖流程不被图片问题阻断。

#### 场景：高清图不足降级
- **当** 高清图片数量不足时
- **那么** 系统必须降低清晰度要求，选择中等清晰度的图片

#### 场景：平台偏好图不足降级
- **当** 符合平台偏好的图片不足时
- **那么** 系统必须回退到按质量总分选择，不限制图片风格

#### 场景：无图降级
- **当** 没有任何可用图片时
- **那么** 系统必须生成纯文字内容，并记录告警日志
