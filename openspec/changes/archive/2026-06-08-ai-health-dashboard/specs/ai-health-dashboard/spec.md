## 新增需求

### 需求:系统必须提供 AI Provider 健康状态 API
系统必须提供 REST API 接口，用于获取所有 AI Provider 的实时健康状态和详细指标数据。

#### 场景:获取所有 Provider 健康状态
- **当** 客户端发送 GET 请求到 `/api/ai/health` 端点
- **那么** 系统返回所有 Provider 的健康状态数组，包含熔断器状态、速率限制状态、成功率、平均响应时间

#### 场景:获取单个 Provider 详细指标
- **当** 客户端发送 GET 请求到 `/api/ai/health/:provider` 端点
- **那么** 系统返回指定 Provider 的详细指标，包括请求总数、成功请求数、失败请求数、平均响应时间、熔断器状态历史

### 需求:系统必须提供 AI 健康仪表盘前端页面
系统必须提供 Web 前端页面，用于可视化展示 AI Provider 的健康状态和关键指标。

#### 场景:访问仪表盘页面
- **当** 用户访问 `/ai-health` 路由
- **那么** 系统渲染 AI 健康仪表盘页面，显示所有 Provider 的状态卡片

#### 场景:自动刷新数据
- **当** 仪表盘页面加载完成
- **那么** 系统每 30 秒自动调用健康状态 API 并刷新显示

#### 场景:手动刷新数据
- **当** 用户点击"刷新"按钮
- **那么** 系统立即调用健康状态 API 并更新显示

### 需求:系统必须对异常状态进行视觉告警
系统必须通过颜色编码和图标对 Provider 的异常状态进行明显的视觉告警。

#### 场景:熔断器打开状态告警
- **当** Provider 的熔断器状态为 OPEN
- **那么** 状态卡片显示红色背景并展示熔断警告图标

#### 场景:低成功率告警
- **当** Provider 的成功率低于 80%
- **那么** 状态卡片显示橙色背景并展示警告图标

#### 场景:健康状态正常
- **当** Provider 的熔断器状态为 CLOSED 且成功率高于 90%
- **那么** 状态卡片显示绿色背景并展示正常图标

### 需求:系统必须展示实时指标数据
系统必须实时展示每个 Provider 的关键性能指标，包括但不限于请求总数、成功率、平均响应时间、速率限制状态。

#### 场景:展示请求统计
- **当** 健康状态数据加载完成
- **那么** 每个 Provider 卡片显示总请求数、成功请求数、失败请求数

#### 场景:展示响应时间指标
- **当** 健康状态数据加载完成
- **那么** 每个 Provider 卡片显示平均响应时间（毫秒）

#### 场景:展示速率限制状态
- **当** 健康状态数据加载完成
- **那么** 每个 Provider 卡片显示当前可用 token 数和是否在白名单
