## 为什么

当前服务的所有发帖动作都是由外部 autojs 脚本触发的，服务本身只负责接收调用并执行逻辑后返回帖子内容。但代码中仍保留了自主发帖的定时调度逻辑（包括发帖定时任务、调度器注册等），这些代码已经不再使用，需要清理以简化代码结构。

## 变更内容

清理自主发帖相关的定时调度逻辑，包括：
- 移除调度器中的发帖定时任务注册
- 移除调度器配置中的发帖相关配置项
- 清理与定时发帖相关的 handler 逻辑
- 保留 API 触发发帖的所有功能（供 autojs 远程调用）

**BREAKING**: 移除定时发帖功能后，系统将不再自动执行定时发帖任务，所有发帖都必须通过外部 autojs 脚本触发。

## 功能 (Capabilities)

### 新增功能
无新增功能，本次变更是清理性质。

### 修改功能
- `scheduler`: 调度器不再注册和管理发帖定时任务，仅保留评论、素材整理、车辆监控等定时任务

## 影响

**受影响的代码**:
- `src/scheduler/index.ts`: 移除发帖定时任务注册和相关 handler
- `src/index.ts`: 移除发帖 handler 的创建和传递
- `src/storage/mysql/scheduler-config-storage.ts`: 移除发帖相关的配置项

**API 影响**:
- 远程发帖 API (`/api/posts/generate`) 不受影响，继续正常工作
- 定时发帖功能移除，不再支持自动定时发帖

**系统影响**:
- 系统启动时不再加载发帖定时任务
- 配置表中发帖相关的 cron 配置将不再使用
