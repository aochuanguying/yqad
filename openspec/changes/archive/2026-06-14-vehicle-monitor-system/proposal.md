## 为什么

目前车辆信息监控依赖手动查看 iOS 小组件，无法实现自动化的异常状态监控和报警。当车辆发生异常移动、车窗未关等异常情况时，用户无法及时获知。需要构建一个自动化的车辆监控系统，实现 24 小时不间断监控，并在异常情况下自动报警。

## 变更内容

**新增功能：**

1. **车辆登录与 Token 管理** - 实现手机号 + 密码自动登录，支持 Token 的自动刷新和持久化存储
2. **车辆状态定时监控** - 系统按可配置的时间间隔自动获取车辆位置和状态信息
3. **异常状态判定** - 基于现有脚本的异常判定逻辑（车窗/车门未关、车辆移动、设防状态等）进行智能判断
4. **手机位置联动** - 当车辆状态异常时，自动获取 Home Assistant 中绑定手机设备的位置信息
5. **安全距离报警** - 计算车辆与手机的实时距离，超出可配置的安全距离阈值时触发电话报警
6. **电话报警接口** - 预留电话报警接口，当安全距离超标时自动拨打指定手机号

## 功能 (Capabilities)

### 新增功能

- `vehicle-auth`: 车辆登录认证、Token 管理和自动刷新机制
- `vehicle-monitor`: 定时监控车辆状态和位置，支持可配置的监控间隔
- `anomaly-detection`: 异常状态判定，包括车辆移动、门窗未关、设防状态等
- `device-location`: 从 Home Assistant 获取手机设备位置信息
- `distance-alert`: 计算车辆与手机的距离，判断是否超出安全距离
- `phone-call`: 电话报警接口（预留实现）

## 影响

**受影响的代码：**
- **新增服务模块**：`src/services/vehicle-monitor-service.ts`
- **修改调度器**：`src/scheduler/index.ts`（添加车辆监控任务注册）
- **修改配置**：`config/default.yaml`（添加 `vehicleMonitor` 配置块）
- **新增数据文件**：`data/vehicle-token.json`（存储车辆 API Token）
- 复用现有 `aujie-widget.js` 中的登录、车辆信息获取、OBD 车况获取等核心逻辑
- 复用 Home Assistant API 文档中的设备位置获取逻辑

**依赖项：**
- 复用现有依赖（axios, node-cron, winston, yaml）
- Home Assistant API 访问权限
- 奥捷智行 API 访问权限

**系统影响：**
- 复用现有调度器框架，无需额外进程
- 复用现有日志系统（Winston）和配置系统（YAML）
- 电话报警接口需要外部服务支持（预留接口）
