## 修改需求

### 需求：系统必须在检测到车辆异常时触发告警通知

当车辆监控系统检测到异常状态时，系统必须调用告警通知服务，通过短信和电话方式通知用户。

#### 场景：异常触发告警
- **当** 车辆监控系统检测到异常状态（如车窗未关、车辆移动、安全距离超标等）
- **那么** 系统调用告警通知服务
- **那么** 告警通知服务发送短信到配置的手机号
- **那么** 告警通知服务在短信发送后拨打电话

#### 场景：冷却时间抑制告警
- **当** 距离上次告警不足 30 分钟
- **那么** 系统跳过本次告警通知
- **那么** 系统记录异常状态但不执行通知
- **那么** 系统记录冷却时间剩余时长

### 需求：系统必须验证 Telecom API 配置的有效性

在调用告警通知前，系统必须验证 Android Telecom API 的配置（API 地址和 Token）是否已正确配置。

#### 场景：配置有效时执行告警
- **当** API 地址和 Token 均已正确配置
- **那么** 系统允许调用告警通知服务
- **那么** 系统执行短信发送和电话拨打

#### 场景：配置缺失时禁止告警
- **当** API 地址或 Token 未配置
- **那么** 系统禁止调用告警通知服务
- **那么** 系统记录警告日志，提示配置缺失
- **那么** 系统继续监控但不执行通知
