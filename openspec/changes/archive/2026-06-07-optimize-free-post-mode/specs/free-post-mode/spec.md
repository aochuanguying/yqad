## 移除需求

### 需求：自由发帖模式必须移除论坛参考逻辑

**Reason**: 论坛参考增加了系统复杂性和外部依赖，互联网参考模式已能提供足够的素材来源

**Migration**: 使用互联网参考模式替代，系统会从小红书等平台获取参考素材，不再依赖论坛帖子

#### 场景：移除论坛参考查询

- **当** 自由发帖模式启动时
- **那么** 不再调用论坛 API 查询参考帖子

#### 场景：移除论坛参考处理逻辑

- **当** 处理参考素材时
- **那么** 不再解析和整合论坛帖子内容

## 修改需求

### 需求：自由发帖模式必须只保留互联网参考和社区分析两种模式

自由发帖模式必须简化为两种模式：互联网参考模式（优先）和社区分析模式（回退）。互联网参考模式必须从小红书等平台获取参考素材，社区分析模式必须基于社区历史帖子分析数据。

#### 场景：优先使用互联网参考模式

- **当** 没有可用的预配置主题时
- **那么** 系统必须首先尝试互联网参考模式

#### 场景：回退到社区分析模式

- **当** 互联网参考模式失败、超限或未返回结果时
- **那么** 系统必须回退到社区分析模式

### 需求：自由发帖必须应用精华帖标准

自由发帖模式必须应用与主题发帖相同的精华帖标准，包括最小内容字符数、最小图片数量等质量要求。即使在自由发帖模式下，也必须确保发帖质量符合精华帖标准。

#### 场景：应用精华帖内容标准

- **当** 自由发帖模式生成内容时
- **那么** 必须满足配置的最小内容字符数要求

#### 场景：应用精华帖图片标准

- **当** 自由发帖模式选取图片时
- **那么** 必须满足配置的最小图片数量要求

#### 场景：质量检查失败重试

- **当** 生成的内容不满足精华帖标准时
- **那么** 必须重新生成直到满足要求或达到最大重试次数
