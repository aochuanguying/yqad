## 新增需求

### 需求：系统必须支持图片去水印功能

系统必须提供图片去水印服务，能够识别并移除互联网参考图片中的水印，包括文字水印、logo 水印等常见类型。去水印功能必须支持批量处理，并提供失败降级策略。

#### 场景：成功去水印

- **当** 互联网参考服务获取到带水印的图片
- **那么** 系统调用去水印服务处理图片并返回无水印版本

#### 场景：去水印失败降级

- **当** 去水印服务处理失败或超时
- **那么** 系统使用原始图片并记录警告日志，继续发帖流程

#### 场景：批量去水印处理

- **当** 多篇参考帖子需要处理图片时
- **那么** 系统并行处理所有图片去水印请求，在配置的超时时间内完成

### 需求：去水印功能必须可配置

系统必须提供去水印功能的配置选项，包括启用开关、超时时间、失败降级策略等。配置必须在 config.json 中可调整。

#### 场景：启用去水印功能

- **当** 配置中 `watermarkRemoval.enabled` 为 true
- **那么** 系统在获取互联网参考图片时自动调用去水印服务

#### 场景：禁用去水印功能

- **当** 配置中 `watermarkRemoval.enabled` 为 false
- **那么** 系统直接使用原始图片，跳过水印处理

### 需求：去水印服务必须使用 AI 模型

系统必须使用 AI 模型的图像理解能力来识别和移除水印，而不是传统的图像处理算法。AI 模型必须能够处理复杂的水印类型和背景。

#### 场景：AI 识别水印

- **当** 输入图片包含水印时
- **那么** AI 模型必须准确识别水印位置和类型

#### 场景：AI 移除水印

- **当** 水印被识别后
- **那么** AI 模型必须生成无水印的修复版本，保持图片质量

## 修改需求

### 需求：互联网参考服务必须集成去水印功能

互联网参考服务在获取参考帖子后，必须检查配置并调用去水印服务处理图片。处理后的图片必须传递给内容生成器使用。

#### 场景：处理参考帖子图片

- **当** 互联网参考服务获取到参考帖子时
- **那么** 必须检查每张图片是否需要去水印处理

#### 场景：传递处理后的图片

- **当** 图片去水印完成后
- **那么** 必须将处理后的图片 URL 传递给内容生成器
