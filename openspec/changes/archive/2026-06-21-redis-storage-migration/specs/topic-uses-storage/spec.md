## 新增需求

### 需求：系统必须提供主题可用次数的 Redis 存储功能

系统必须从零开始实现 Redis 存储，使用 Redis String 存储主题可用次数，支持原子增减和 TTL 自动过期。

**约束**：
- 项目从零开始，没有历史数据需要迁移
- 使用 String 数据结构（简单整数计数）
- 不支持 Sorted Set（不需要排序功能，避免过度设计）

#### 场景：设置主题可用次数
- **当** 调用 `setUses(topicId, uses)` 方法时
- **那么** 系统必须使用 `SET` 命令将可用次数存储为字符串
- **并且** 必须设置 7 天 TTL（使用 `EXPIRE` 命令）

#### 场景：获取主题可用次数
- **当** 调用 `getUses(topicId)` 方法时
- **那么** 系统必须使用 `GET` 命令查询可用次数
- **并且** 必须将字符串转换为数字返回
- **并且** 如果键不存在必须返回 0

#### 场景：增加可用次数
- **当** 调用 `incrementUses(topicId, delta)` 方法时
- **那么** 系统必须使用 `INCRBY` 命令原子地增加可用次数
- **并且** 必须返回增加后的新值

#### 场景：减少可用次数
- **当** 调用 `decrementUses(topicId, delta)` 方法时
- **那么** 系统必须使用 `DECRBY` 命令原子地减少可用次数
- **并且** 必须确保结果不为负数（最小为 0）

#### 场景：批量获取可用次数
- **当** 调用 `getUsesBatch(topicIds)` 方法时
- **那么** 系统必须使用 Pipeline 批量执行 `GET` 命令
- **并且** 必须返回 Map<string, number> 格式的结果

### 需求：系统必须支持降级到内存存储

系统必须在 Redis 不可用时自动降级到内存 Map 存储，保证业务连续性。

#### 场景：Redis 不可用降级
- **当** Redis 连接失败或超时时
- **那么** 系统必须自动切换到内存 Map 存储
- **并且** 必须记录警告日志

#### 场景：恢复 Redis 连接
- **当** Redis 连接恢复时
- **那么** 系统必须将内存数据同步到 Redis
- **并且** 必须切换回 Redis 存储模式
