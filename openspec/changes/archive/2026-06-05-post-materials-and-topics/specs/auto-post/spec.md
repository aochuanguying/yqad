## 修改需求

### 需求: 系统必须自动生成并发布优质帖子
系统必须定时发布帖子到社区。发帖时必须优先使用已配置且未使用的主题（含关联素材和提纲），当所有主题用完或无主题配置时，回退到基于 AI 自由生成的模式。

#### 场景: 优先使用已配置主题发帖
- **当** 自动发帖任务触发且存在"未使用"状态的主题
- **那么** 系统必须选取一个未使用的主题，使用其方向描述和内容提纲作为 AI 生成的约束，生成帖子内容，并附带主题关联的素材图片

#### 场景: 无可用主题时自由生成
- **当** 自动发帖任务触发且所有主题均已使用（或无主题配置）
- **那么** 系统必须回退到原有模式：从内容分析结果中选取热点话题，调用 AI 自由生成帖子

#### 场景: 发帖后标记主题已使用
- **当** 使用预配置主题成功发帖
- **那么** 系统必须将该主题标记为"已使用"并记录使用时间

#### 场景: 发布帖子
- **当** 帖子内容生成完毕
- **那么** 系统必须调用发帖 API 将帖子发布到社区

#### 场景: 每日发帖数量可配置
- **当** 执行每日发帖任务
- **那么** 系统必须根据配置的每日发帖数量上限（默认1篇）执行发帖
