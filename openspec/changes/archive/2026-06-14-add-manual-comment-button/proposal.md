## 为什么

当前评论功能仅支持定时自动执行，用户无法在 Web 界面手动触发立即评论。这导致用户需要等待定时任务执行，无法灵活控制评论时机，降低了用户体验和操作灵活性。

## 变更内容

在 Web 管理界面的"评论"配置页签中，添加一个"立即评论"按钮，位于"保存配置"按钮的左侧。点击后：
- 调用后端 API 立即执行一次评论任务
- 显示执行状态（执行中、成功、失败）
- 支持查看执行日志和结果

## 功能 (Capabilities)

### 新增功能
- `manual-comment-trigger`: 在 Web 界面添加手动触发评论的按钮，支持立即执行评论任务并显示执行状态

## 影响

- **前端代码**: `src/web/public/index.html` - 评论 Tab 需要添加按钮和状态显示逻辑
- **后端 API**: 需要新增或复用现有的评论执行 API 端点
- **服务层**: `src/services/auto-comment.ts` - 可能需要暴露公开方法供 API 调用
- **用户体验**: 提升操作灵活性，用户可随时手动触发评论任务
