## 新增需求

### 需求: 系统必须记录所有任务执行的详细日志
系统必须对每次任务执行进行日志记录，包括执行时间、结果状态和关键数据。

#### 场景: 记录成功执行日志
- **当** 任何任务（签到/评论/发帖）执行成功
- **那么** 系统必须记录INFO级别日志，包含任务类型、执行时间、关键结果（如获得积分数、评论ID等）

#### 场景: 记录失败执行日志
- **当** 任何任务执行失败
- **那么** 系统必须记录ERROR级别日志，包含任务类型、失败原因、重试次数

#### 场景: 日志文件按日轮转
- **当** 日期变更时
- **那么** 系统必须创建新的日志文件（格式：YYYY-MM-DD.log），保留最近30天的日志

### 需求: 系统必须提供执行状态摘要
系统必须能够输出每日执行的汇总状态，便于快速了解系统运行情况。

#### 场景: 每日执行摘要
- **当** 所有每日任务执行完毕
- **那么** 系统必须生成当日摘要，包含：签到状态、评论数量、发帖数量、失败任务列表

#### 场景: 异常状态通知
- **当** 连续2天签到失败或任务异常率超过50%
- **那么** 系统必须输出WARN级别告警日志（后续可扩展为推送通知）
