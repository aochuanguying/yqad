## 新增需求

### 需求: 系统必须自动完成每日签到
系统必须在每天指定时间自动执行登录和签到流程，确保每日签到不遗漏。

#### 场景: 正常签到流程
- **当** 调度器触发每日签到任务
- **那么** 系统必须先执行登录获取token，再调用签到接口完成签到

#### 场景: token过期自动刷新
- **当** 签到时发现token已过期
- **那么** 系统必须自动使用refresh_token获取新token并重试签到

#### 场景: 签到失败重试
- **当** 签到接口返回失败（网络异常或服务端错误）
- **那么** 系统必须在等待随机间隔后重试，最多重试3次

### 需求: 系统必须管理认证token的生命周期
系统必须安全存储token，跟踪过期时间，并在需要时自动刷新。

#### 场景: token持久化存储
- **当** 登录成功获取到新token
- **那么** 系统必须将token及其过期时间保存到本地存储

#### 场景: 启动时检查token有效性
- **当** 系统启动或执行任务前
- **那么** 系统必须检查已存储token是否仍在有效期内，过期则自动刷新
