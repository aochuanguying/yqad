## 新增需求

### 需求: Mock API客户端必须模拟一汽奥迪APP全部核心接口
系统必须提供Mock实现，覆盖登录、签到、获取帖子列表、获取评论列表、发布评论、发布帖子等接口。Mock实现必须返回符合真实API格式的响应数据。

#### 场景: 登录接口返回token
- **当** 调用登录接口并传入用户名和密码
- **那么** 系统必须返回包含access_token和refresh_token的响应对象

#### 场景: 签到接口返回成功状态
- **当** 使用有效token调用签到接口
- **那么** 系统必须返回签到成功状态及获得的积分信息

#### 场景: 获取帖子列表
- **当** 调用帖子列表接口并指定页码和数量
- **那么** 系统必须返回包含帖子标题、内容摘要、作者、发布时间的帖子数组

#### 场景: 获取帖子评论列表
- **当** 调用评论接口并传入帖子ID
- **那么** 系统必须返回该帖子下的评论列表，包含评论内容、作者、时间

#### 场景: 发布评论
- **当** 调用发布评论接口并传入帖子ID和评论内容
- **那么** 系统必须返回评论发布成功状态及评论ID

#### 场景: 发布帖子
- **当** 调用发帖接口并传入标题和正文内容
- **那么** 系统必须返回发帖成功状态及帖子ID

### 需求: Mock API必须通过接口抽象实现可替换
系统必须定义统一的API接口（Interface），Mock实现和未来的真实HTTP实现都必须遵循同一接口契约，实现可无缝切换。

#### 场景: 通过配置切换API实现
- **当** 配置文件中将api.mode设置为"mock"
- **那么** 系统必须使用Mock实现处理所有API调用

#### 场景: 接口契约一致性
- **当** Mock实现和真实实现都实现同一接口
- **那么** 两者的方法签名、参数类型和返回类型必须完全一致
