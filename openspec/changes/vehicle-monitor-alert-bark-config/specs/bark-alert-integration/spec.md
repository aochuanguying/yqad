## 新增需求

### 需求:Bark 推送告警
系统必须支持通过 Bark 推送服务发送告警通知。当配置了 Bark 键时，系统应当调用 Bark API 发送告警消息。

#### 场景:Bark 键配置有效
- **当** 告警配置中包含有效的 Bark 键
- **那么** 系统调用 Bark API 发送告警推送通知

#### 场景:Bark 键未配置
- **当** 告警配置中 Bark 键为空字符串
- **那么** 系统不调用 Bark API，跳过 Bark 告警

#### 场景:Bark 推送成功
- **当** Bark API 返回成功响应
- **那么** 记录 Bark 告警状态为成功，并记录日志

#### 场景:Bark 推送失败
- **当** Bark API 返回错误响应或请求超时
- **那么** 记录 Bark 告警状态为失败，并记录错误日志，但不影响其他告警渠道的执行

### 需求:Bark 告警消息格式
Bark 告警消息必须包含告警类型、时间戳和位置信息（如有）。

#### 场景:构造告警消息
- **当** 触发告警时
- **那么** Bark 消息标题为"车辆告警"，正文包含异常类型列表和时间戳

#### 场景:包含位置信息
- **当** 告警包含车辆位置信息时
- **那么** Bark 消息正文必须包含位置地址或坐标

### 需求:Bark 客户端配置
系统必须支持配置 Bark 服务端地址和认证键。

#### 场景:使用默认 Bark 服务器
- **当** 未配置自定义 Bark 服务器地址时
- **那么** 使用 Bark 官方服务器 `https://api.day.app`

#### 场景:使用自定义 Bark 服务器
- **当** 配置了自定义 Bark 服务器地址时
- **那么** 使用配置的服务器地址发送推送
