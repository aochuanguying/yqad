## 为什么

当前 Web 管理界面开放外网访问时缺乏认证保护，任何能够访问服务器地址的人都可以查看和修改配置、管理素材、触发任务等敏感操作。这存在严重的安全风险，需要为 Web 界面增加登录校验功能，确保只有授权用户才能访问管理功能。

## 变更内容

为 Web 管理界面增加基于 Session 的登录认证机制：

- **新增功能**：
  - Web 登录页面和会话管理
  - 管理员账号配置（用户名 + 密码）
  - 登录状态验证中间件
  - 登出功能
  
- **修改功能**：
  - 所有 API 端点增加认证检查（公开接口除外）
  - Web 界面根据登录状态显示/隐藏功能

## 功能 (Capabilities)

### 新增功能
- `web-session-auth`: Web 界面的 Session 登录认证机制，包括登录页面、会话管理、认证中间件
- `admin-credentials-config`: 管理员账号配置功能，支持通过配置文件或环境变量设置用户名和密码
- `auth-middleware`: Express 中间件，验证请求的登录状态，保护 API 和 Web 资源

### 修改功能
- `api-routes`: 所有 API 端点需要增加认证检查（除公开接口如登录 API 本身）

## 影响

- **代码影响**：
  - 新增认证中间件模块
  - 新增登录/登出 API 路由
  - 修改 server.ts 应用中间件
  - 所有现有 API 路由需要应用认证保护
  
- **配置影响**：
  - 需要在 config 中增加管理员账号配置项
  - 可能需要设置环境变量（推荐用于生产环境）
  
- **用户影响**：
  - 访问 Web 界面需要先登录
  - 登录状态过期后需要重新登录
  - 现有功能使用方式不变，仅增加访问控制

- **安全影响**：
  - 防止未授权访问
  - 保护配置数据和操作权限
  - 支持外网安全访问
