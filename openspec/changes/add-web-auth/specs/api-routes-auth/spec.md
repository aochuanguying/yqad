## 新增需求

### 需求：所有 API 端点必须默认要求认证
除公开白名单路由外，所有 `/api/*` 端点必须要求用户通过 Session 认证才能访问。

#### 场景：已认证用户访问 API
- **当** 已登录用户请求 `/api/config`、`/api/materials` 等受保护端点
- **那么** API 必须正常处理请求，返回数据

#### 场景：未认证用户访问 API
- **当** 未登录用户请求受保护的 API 端点
- **那么** API 必须返回 401 状态码和错误响应 `{"error": "未授权访问", "code": "UNAUTHORIZED"}`

#### 场景：Session 过期后访问 API
- **当** 用户使用过期的 Session 访问 API
- **那么** API 必须返回 401 状态码，客户端需要重新登录

### 需求：公开 API 白名单必须可配置
系统必须支持配置不需要认证的公开 API 路由，用于登录接口和特殊用途接口。

#### 场景：访问登录 API
- **当** 用户请求 `POST /api/auth/login`
- **那么** API 必须允许访问，无需认证

#### 场景：访问远程发帖 API
- **当** 用户请求 `/api/posts/token/*` 端点
- **那么** API 必须允许访问，使用独立的 API Token 认证（不由 Session 中间件处理）

#### 场景：访问登出 API
- **当** 用户请求 `POST /api/auth/logout`
- **那么** API 必须允许访问（但需要有效的 Session 才能执行登出操作）
