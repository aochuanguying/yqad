## 需求
### 需求：扫描素材目录
系统必须能够扫描配置的素材目录，发现所有符合格式的图片文件。

#### 场景：成功扫描本地素材目录
- **当** 素材整理任务启动
- **那么** 系统遍历配置的基础目录及其子目录，找到所有支持的图片格式（.jpg, .jpeg, .png, .heic）

#### 场景：跳过已存在的素材
- **当** 发现图片文件的 hash 已存在于 material_records 表
- **那么** 系统跳过该文件，不重复录入

#### 场景：处理空目录
- **当** 配置的素材目录不存在或为空
- **那么** 系统记录警告日志，不抛出错误，返回空列表

### 需求：计算文件哈希
系统必须为每个发现的图片文件计算唯一哈希值（SHA-256），用于去重。

#### 场景：计算新文件哈希
- **当** 发现新图片文件
- **那么** 系统读取文件内容并计算 SHA-256 哈希值

#### 场景：哈希碰撞处理
- **当** 两个不同文件产生相同哈希值（极罕见）
- **那么** 系统记录警告，使用文件路径作为辅助区分

### 需求：过滤不支持的文件
系统必须过滤掉不支持的文件格式和非文件项（文件夹、符号链接）。

#### 场景：过滤非图片文件
- **当** 扫描目录发现非图片文件（.txt, .mp4 等）
- **那么** 系统忽略这些文件，不记录日志

#### 场景：过滤隐藏文件
- **当** 发现以 . 开头的隐藏文件
- **那么** 系统跳过这些文件

#### 场景：过滤符号链接
- **当** 发现符号链接
- **那么** 系统跳过，避免循环引用
