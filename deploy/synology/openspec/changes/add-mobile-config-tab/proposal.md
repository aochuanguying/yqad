## 为什么

当前系统缺少对手机相关数据的管理功能，用户需要手动记录短信和未接电话信息。通过新增"手机配置"页签，可以集中管理手机相关配置和数据，提供更完整的移动端支持能力。

## 变更内容

- **新增**：在首页添加"手机配置"页签
- **新增**："手机短信"功能 - 记录电话号码、短信内容、接收时间
- **新增**："未接电话"功能 - 记录电话号码、接收时间
- **复用**："AutoJS 配置" - 将现有 AutoJS 配置页面集成到新页签下
- **新增**：MySQL 数据表存储短信和未接电话记录
- **新增**：API 接口供外部系统调用（使用 API Token 认证）

## 功能 (Capabilities)

### 新增功能
- `mobile-sms`: 手机短信记录存储和 API 接口
- `missed-call`: 未接电话记录存储和 API 接口
- `mobile-config-ui`: 手机配置页面 UI（包含三个子功能入口）

### 修改功能
- `autojs-config`: AutoJS 配置从独立页面改为"手机配置"页签下的子功能

## 影响

- **数据库**: 新增 `mobile_sms` 和 `missed_calls` 表
- **API**: 新增 `/api/posts/mobile/sms` 和 `/api/posts/mobile/calls/missed` 接口
- **前端**: 首页导航新增"手机配置"页签，整合原 AutoJS 配置页面
- **后端路由**: 新增 mobile 相关路由文件
