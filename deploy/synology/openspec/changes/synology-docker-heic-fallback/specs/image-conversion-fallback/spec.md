## 新增需求

### 需求:Linux 容器中的 HEIC/HEIF 兜底转换
素材处理模块在 Linux 容器中处理 HEIC/HEIF 文件时，必须优先使用 `sharp` 主转换链路；当主链路失败时，必须自动执行可用的命令行兜底转换，并最终生成 JPEG 输出。

#### 场景:sharp 直接转换成功
- **当** 系统处理 HEIC/HEIF 文件且 `sharp` 可直接完成解码与转换
- **那么** 系统必须直接使用 `sharp` 输出 JPEG 文件
- **并且** 不得调用额外的命令行兜底工具

#### 场景:sharp 转换失败后执行 Linux 兜底
- **当** 系统处理 HEIC/HEIF 文件且 `sharp` 主转换链路失败
- **那么** 系统必须自动调用已配置的 Linux 兜底转换工具生成临时 JPEG
- **并且** 系统必须继续执行旋转与最终 JPEG 质量输出
- **并且** 最终输出文件路径和格式必须与正常成功路径保持一致

#### 场景:兜底工具不可用
- **当** 系统需要执行 HEIC/HEIF 兜底转换但目标工具不可用、超时或执行失败
- **那么** 系统必须将当前文件标记为失败并记录可诊断错误
- **并且** 不得导致整个素材处理任务进程崩溃

### 需求:图片转换配置
系统必须为 JPEG 输出与 HEIC/HEIF 兜底行为提供可配置参数，并必须在未显式设置时提供安全默认值。

#### 场景:配置 JPEG 输出质量
- **当** 用户设置 `materials.processing.jpegQuality`
- **那么** 系统必须使用该值控制最终 JPEG 输出质量

#### 场景:启用或关闭 HEIC 兜底
- **当** 用户设置 HEIC/HEIF 兜底开关
- **那么** 系统必须据此决定是否执行命令行兜底转换
- **并且** 默认配置必须在 Linux 容器部署场景下启用该能力

#### 场景:限制兜底执行时间
- **当** 用户设置 HEIC/HEIF 兜底超时
- **那么** 系统必须在超时后中断本次兜底命令
- **并且** 必须记录超时错误以便排查
