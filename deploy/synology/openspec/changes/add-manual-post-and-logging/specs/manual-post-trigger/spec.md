## 新增需求

### 需求:用户可以手动触发发帖任务
系统必须允许用户在 Web 管理页面手动触发发帖任务，立即执行一次发帖操作。

#### 场景:用户点击立即发帖按钮
- **当** 用户在 Web 管理页面点击"立即发帖"按钮
- **那么** 系统调用 `POST /api/posts/execute` 接口触发发帖任务

#### 场景:发帖成功
- **当** 发帖任务成功完成
- **那么** 系统返回成功状态，显示发帖标题和简要信息

#### 场景:发帖失败
- **当** 发帖任务执行失败（如无可用主题、API 调用失败等）
- **那么** 系统返回错误状态，显示具体的错误信息

#### 场景:发帖任务正在执行中
- **当** 用户点击立即发帖按钮但已有发帖任务在执行
- **那么** 系统返回提示"发帖任务正在执行中，请稍后再试"，不启动新任务

### 需求:手工发帖 API 接口
系统必须提供 `POST /api/posts/execute` 接口，用于手动触发发帖任务。

#### 场景:调用手工发帖接口
- **当** 客户端发送 POST 请求到 `/api/posts/execute`
- **那么** 系统验证用户登录状态，执行发帖任务并返回结果

#### 场景:接口鉴权失败
- **当** 用户未登录或登录已过期
- **那么** 系统返回 401 未授权错误

### 需求：手工发帖扣减主题次数
手工发帖成功后立即扣减主题使用次数（如使用主题），无需等待回调。

#### 场景：手工发帖使用主题
- **当** 手工发帖使用预配置主题生成内容并成功发布
- **那么** 系统立即扣减该主题的使用次数，并标记为已使用

#### 场景：手工发帖自由生成
- **当** 手工发帖使用自由生成模式（无主题）
- **那么** 系统不扣减任何主题次数
