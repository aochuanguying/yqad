## 新增需求

### 需求：系统必须提供 Web 界面配置 Telecom API 地址

系统必须提供 Web 管理界面，允许用户配置 Android Telecom API 的访问地址（如 `http://192.168.1.100:5000`）。配置必须持久化保存到配置文件中。

#### 场景：配置 API 地址
- **当** 用户在 Web 界面输入有效的 API 地址
- **那么** 系统验证地址格式（必须包含协议和端口）
- **那么** 系统将配置保存到配置文件
- **那么** 系统显示配置成功提示

#### 场景：配置无效的 API 地址
- **当** 用户输入无效的 API 地址格式
- **那么** 系统拒绝保存并显示错误提示
- **那么** 系统保持原有配置不变

### 需求：系统必须提供 Web 界面配置 Telecom API Token

系统必须提供 Web 管理界面，允许用户配置 Android Telecom API 的认证 Token。Token 在界面中应部分掩码显示以保护安全。

#### 场景：配置 Token
- **当** 用户在 Web 界面输入有效的 Token
- **那么** 系统将 Token 保存到配置文件
- **那么** 系统在界面中显示 Token 的前 8 位和后 4 位，中间用 `***` 替代
- **那么** 系统显示配置成功提示

#### 场景：查看现有 Token
- **当** Token 已配置
- **那么** 系统在配置界面显示掩码后的 Token
- **那么** 系统提供"修改"按钮允许重新配置

### 需求：系统必须提供 Web 界面配置告警接收人手机号

系统必须提供 Web 管理界面，允许用户配置接收告警短信和电话的手机号。手机号必须符合中国大陆手机号格式（11 位数字，以 1 开头）。

#### 场景：配置告警手机号
- **当** 用户输入有效的 11 位中国大陆手机号
- **那么** 系统将手机号保存到配置文件
- **那么** 系统显示配置成功提示

#### 场景：配置无效的手机号
- **当** 用户输入不符合格式的手机号
- **那么** 系统拒绝保存并显示错误提示
- **那么** 系统提示正确的手机号格式

### 需求：系统必须提供 API 连接测试功能

系统必须提供 API 连接测试功能，允许用户验证配置的 Telecom API 地址和 Token 是否有效。

#### 场景：API 连接测试成功
- **当** 用户点击"测试连接"按钮
- **那么** 系统调用 Telecom API 的健康检查接口（`/health`）
- **那么** 如果 API 响应正常，系统显示"连接成功"提示

#### 场景：API 连接测试失败
- **当** 用户点击"测试连接"按钮
- **那么** 系统调用 Telecom API 的健康检查接口
- **那么** 如果 API 不可用或 Token 无效，系统显示"连接失败"提示及错误原因
