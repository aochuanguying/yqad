## 新增需求

### 需求：系统必须记录每次告警通知的详细信息

系统必须记录每次告警通知的详细信息，包括告警时间、异常类型、通知方式（短信/电话）、通知状态和接收人手机号。

#### 场景：记录告警历史
- **当** 系统执行告警通知（短信或电话）
- **那么** 系统记录告警时间（ISO 8601 格式）
- **那么** 系统记录异常类型列表
- **那么** 系统记录通知方式（"sms" 或 "call"）
- **那么** 系统记录通知状态（"success"、"failed"、"timeout"）
- **那么** 系统记录接收人手机号（掩码显示）

#### 场景：告警历史记录数量限制
- **当** 告警记录数量超过 100 条
- **那么** 系统自动删除最早的记录
- **那么** 系统保持最近 100 条记录

### 需求：系统必须提供告警历史查询 API

系统必须提供 REST API 接口，允许用户查询最近的告警历史记录。API 应支持分页和数量限制参数。

#### 场景：查询告警历史
- **当** 用户请求 GET `/api/alert-history` 接口
- **那么** 系统返回最近的告警记录列表
- **那么** 系统默认返回最近 10 条记录
- **那么** 系统支持 `limit` 参数指定返回数量（最大 100）

#### 场景：告警历史响应格式
- **当** 系统返回告警历史
- **那么** 响应必须包含记录数组和总数
- **那么** 每条记录必须包含时间、异常类型、通知方式、状态字段

### 需求：系统必须在 Web 界面显示告警历史

系统必须在车辆监控 Web 界面中显示告警历史记录，以表格形式展示，支持按时间倒序排列。

#### 场景：显示告警历史列表
- **当** 用户访问车辆监控页面
- **那么** 系统显示"告警历史"区域
- **那么** 系统以表格形式展示最近 10 条告警记录
- **那么** 系统显示字段包括：时间、异常类型、通知方式、状态

#### 场景：告警状态可视化
- **当** 显示告警记录
- **那么** 成功状态使用绿色标识
- **那么** 失败状态使用红色标识
- **那么** 超时状态使用黄色标识

### 需求：系统必须提供告警统计信息

系统必须提供告警统计信息，包括今日告警次数、本周告警次数和最常见的异常类型。

#### 场景：计算告警统计
- **当** 用户请求统计信息
- **那么** 系统计算今日（0 点至今）的告警次数
- **那么** 系统计算本周（周一至今）的告警次数
- **那么** 系统统计最常见的 3 种异常类型

#### 场景：显示告警统计
- **当** 用户访问车辆监控页面
- **那么** 系统在告警历史区域显示统计卡片
- **那么** 系统展示今日告警次数、本周告警次数
- **那么** 系统展示最常见的异常类型
