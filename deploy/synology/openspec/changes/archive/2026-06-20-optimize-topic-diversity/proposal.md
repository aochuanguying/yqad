# 优化主题发帖内容多样化

## 为什么

当前主题发帖功能存在以下问题：

1. **内容重复**：同一主题多次发帖容易使用相似的提纲和素材，导致内容高度重复
2. **模式化严重**：子方向顺序使用，缺乏轮换机制，导致发帖模式可预测
3. **素材单一**：同一主题的圖片过于相似，缺乏交叉使用策略
4. **结构雷同**：内容结构固定，缺乏变化，影响用户体验
5. **标题单调**：标题风格单一，缺乏吸引力，影响点击率

虽然已有基础的主题和子方向机制，但缺乏多样化优化，导致长期发帖效果下降。需要在合规性检查（第一步优化）的基础上，进一步增强内容多样化能力，提升帖子质量和吸引力。

## 变更内容

- **新增**：基于使用次数的子方向选择策略（优先使用使用次数少的子方向）
- **新增**：素材交叉使用策略（记录已用素材，优先使用未用过的）
- **新增**：AI 提纲变体生成（基于原提纲生成不同版本）
- **新增**：标题风格多样化（提问式、分享式、评测式、对比式等）
- **新增**：发帖内容变体生成（调整叙述角度、结构顺序）
- **修改**：`topics-service.ts` 的子方向选择逻辑，从顺序选择改为基于使用次数的加权选择
- **修改**：`auto-post.ts` 的内容生成流程，集成多样化策略
- **修改**：`content-generator.ts` 的标题生成逻辑，支持多种风格

## 功能 (Capabilities)

### 新增功能
- `sub-direction-rotation`: 基于使用次数的子方向轮换，优先选择使用次数少的子方向
- `material-rotation`: 素材交叉使用策略，记录并轮换使用素材
- `outline-variation`: AI 提纲变体生成，基于原提纲创建不同版本
- `title-style-diversity`: 标题风格多样化，支持提问式、分享式、评测式、对比式等
- `content-variation`: 发帖内容变体生成，调整叙述角度和结构顺序

### 修改功能
- `topic-selection`: 主题服务需要支持基于使用次数的子方向选择
- `auto-post`: 发帖服务需要集成多样化策略到内容生成流程
- `content-generation`: 内容生成器需要支持多种标题风格和内容变体

## 影响

- **代码影响**：
  - `src/web/services/topics-service.ts`：修改子方向选择逻辑，增加使用次数追踪
  - `src/services/auto-post.ts`：集成多样化策略到发帖流程
  - `src/ai/content-generator.ts`：增加标题风格和内容变体生成
  - `src/ai/prompts.ts`：更新 AI 提示词以支持多样化生成
- **数据结构影响**：
  - `Topic` 接口需要增加 `subDirectionUsageCount` 字段
  - 需要新增 `MaterialUsageRecord` 接口记录素材使用情况
- **配置影响**：
  - 可能需要增加多样化相关的配置参数（如标题风格权重、变体生成策略等）
- **发帖流程**：
  - 发帖流程增加多样化策略应用步骤
  - 生成时间可能略有增加（AI 变体生成）

## 技术实现要点

### 1. 子方向轮换机制
- 在 `Topic` 数据结构中增加 `subDirectionUsageCount: number[]` 数组
- 每次使用子方向后，递增对应索引的计数
- 选择时采用加权随机策略：使用次数越少，权重越高
- 支持重置机制（当所有子方向使用次数差异较小时）

### 2. 素材交叉使用
- 为每个主题记录 `materialUsageHistory: { path: string, usedCount: number, lastUsedAt?: string }[]`
- 优先选择使用次数少的素材
- 支持同主题内素材轮换和跨主题素材交叉

### 3. 提纲变体生成
- 调用 AI 接口，基于原提纲生成 2-3 个变体版本
- 变体策略：调整层次结构、替换同义表达、改变详略程度
- 随机选择一个变体用于内容生成

### 4. 标题风格多样化
- 定义多种标题风格模板：
  - **提问式**：「如何评价...」「...是什么体验？」
  - **分享式**：「分享我的...」「...使用心得」
  - **评测式**：「深度评测...」「...优缺点分析」
  - **对比式**：「...vs...」「...和...哪个更好？」
  - **数字式**：「...的 5 个技巧」「3 分钟了解...」
  - **悬念式**：「没想到...」「...居然可以这样」
- 根据主题类型和历史发帖记录，智能选择风格
- 避免短期内重复使用同一风格

### 5. 内容变体生成
- 调整叙述角度：第一人称 vs 第三人称，主观体验 vs 客观分析
- 调整结构顺序：总分总 vs 分总 vs 总分
- 调整语言风格：正式 vs 轻松，详细 vs 简洁
- 每次发帖随机选择变体策略

## 验收标准

1. **子方向轮换**：
   - 同一主题的子方向使用次数差异不超过 3 次
   - 连续发帖不会顺序使用同一子方向

2. **素材交叉**：
   - 同一素材在 5 次发帖内不重复使用（除非素材耗尽）
   - 同主题素材使用分布均匀

3. **提纲变体**：
   - 每次生成的提纲与原提纲有明显差异（AI 评估）
   - 变体保持原提纲的核心要点

4. **标题风格**：
   - 支持至少 6 种标题风格
   - 连续 3 次发帖不使用同一风格
   - 标题吸引力评分提升 20%（通过点击率评估）

5. **内容变体**：
   - 内容结构有明显变化（通过 AI 检测）
   - 叙述角度多样化
   - 用户反馈内容重复度降低

## 风险和依赖

- **依赖**：建立在第一步合规性检查的基础上，需要确保合规性检查已完成
- **风险**：AI 变体生成可能增加 API 调用成本和响应时间
- **缓解**：设置变体生成的超时和降级策略，确保发帖流程不被阻塞

## 后续优化建议

- 增加用户反馈收集机制，根据点赞、评论数据优化多样化策略
- 引入 A/B 测试，对比不同标题风格的实际效果
- 建立内容质量评估模型，自动调整多样化参数
