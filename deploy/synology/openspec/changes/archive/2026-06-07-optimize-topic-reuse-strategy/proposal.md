## 为什么

当前主题复用策略存在两个问题：
1. **标题过于严格**：AI 生成标题时与主题标题完全一致才算重复，但用户希望标题仅作为参考，不需要严格保持一致
2. **内容分配不均**：当一个主题可被复用多次（如 maxUseCount=5）时，当前逻辑没有规划如何均衡分配标题、方向描述和内容提纲，可能导致一个主题的内容在一次发帖中就用完，后续复用缺乏素材

需要优化主题复用策略，确保多次复用时内容可持续、标题多样化。

## 变更内容

1. **标题策略调整**：
   - 主题标题仅作为参考，AI 生成时可以灵活变化
   - 去重逻辑从"与主题标题完全一致"改为"与该主题历史发帖标题比对"
   - 允许 AI 基于方向描述和提纲自由生成标题

2. **内容均衡分配机制**：
   - 为每个主题建立"内容池"概念，包含多个子方向
   - 每次发帖从内容池中选取未使用或部分重叠的子方向
   - 确保同一主题的多次发帖内容有递进性和多样性

3. **发帖历史追踪增强**：
   - 记录每次发帖使用的具体子方向和提纲
   - 下次复用时避免重复使用相同的内容组合

## 功能 (Capabilities)

### 新增功能
- `topic-content-pool`: 主题内容池机制，支持将主题方向拆分为多个子方向，每次发帖选取不同子方向
- `smart-title-generation`: 智能标题生成，标题仅作为参考，AI 可基于方向自由生成，去重仅比对同主题历史发帖

### 修改功能
- 无

## 影响

- **代码影响**：
  - `src/services/auto-post.ts`: 修改 `generatePostWithDedup()` 的标题去重逻辑
  - `src/web/services/topics-service.ts`: 扩展 Topic 数据结构，增加内容池相关字段
  - `src/ai/content-generator.ts`: 调整 Prompt 构建逻辑，支持灵活标题生成
  - `src/ai/prompts.ts`: 更新发帖 Prompt 模板

- **数据结构影响**：
  - `topics.json` 需要增加子方向池字段
  - `postHistory` 需要记录使用的子方向信息

- **向后兼容**：
  - 旧格式 topics.json 自动补充默认字段
  - 无 maxUseCount 的主题默认保持原有行为
