## 修改需求

### 需求：系统必须基于精华帖规则评估发帖准备度
系统必须将精华帖创作规则抽象为可执行的评估，并在发帖前对候选内容进行检查，输出可解释的未达标原因。评估必须涵盖标题质量、内容结构、图片质量等多个维度。

#### 场景：评估精华帖准备度并返回多维度原因
- **当** 系统生成一篇"精华候选帖"（含标题、正文、候选图片）准备发布
- **那么** 系统必须基于配置的精华门槛对候选帖进行 readiness 评估，并返回 `eligible`（是否达标）以及 `reasons`（未达标原因列表）
- **并且** 评估必须包含标题质量、内容结构、图片质量维度的检查结果

### 需求：精华帖候选必须满足标题质量要求
系统必须将标题质量作为精华候选的评估维度之一，包括字数、关键词、句式等要求。

#### 场景：标题字数不达标时判定不合格
- **当** 候选帖标题字数少于 10 字或超过 20 字
- **那么** readiness 评估结果必须为不达标，并包含"标题字数不符合要求"的原因

#### 场景：标题缺少热点关键词时记录提醒
- **当** 候选帖标题不包含任何热点关键词
- **那么** readiness 评估必须记录提醒，但不强制降级（作为软性约束）

### 需求：精华帖候选必须满足内容结构要求
系统必须将内容结构作为精华候选的评估维度之一，包括开头引入、分段逻辑、结尾互动等要求。

#### 场景：内容缺少开头引入时记录提醒
- **当** 候选帖正文开头未使用提问、数据或痛点引入
- **那么** readiness 评估必须记录提醒，但不强制降级（作为软性约束）

#### 场景：内容缺少分段逻辑时记录提醒
- **当** 候选帖正文不包含任何小标题或编号列表
- **那么** readiness 评估必须记录提醒，但不强制降级（作为软性约束）

#### 场景：内容缺少结尾互动引导时记录提醒
- **当** 候选帖正文结尾不包含互动引导语
- **那么** readiness 评估必须记录提醒，但不强制降级（作为软性约束）

### 需求：精华帖候选必须满足图片质量要求
系统必须将图片质量作为精华候选的评估维度之一，包括图片数量、多角度覆盖等要求。

#### 场景：图片数量达到推荐值时给予加分
- **当** 候选帖图片数量达到 6-9 张
- **那么** readiness 评估必须记录为"图片质量优秀"

#### 场景：图片数量仅满足最低要求时记录提醒
- **当** 候选帖图片数量为 4-5 张（满足最低要求但未达推荐值）
- **那么** readiness 评估必须记录提醒，但不强制降级

### 需求：精华帖候选必须满足最低字数门槛
系统必须将"正文不少于 250 字"作为精华候选的最低门槛，并在不满足时触发降级逻辑。

#### 场景：正文字数不足时判定不达标
- **当** 候选帖正文字符数小于 `featuredPosting.minContentChars`（默认 250）
- **那么** readiness 评估结果必须为不达标，并包含"字数不足"的原因

### 需求：精华帖候选必须满足最低图片数量门槛
系统必须将"图片不少于 4 张"作为精华候选的最低门槛，并以"上传成功的图片 URL 数量"为准进行判定。

#### 场景：上传成功图片不足时判定不达标
- **当** 候选帖最终上传成功的图片 URL 数量小于 `featuredPosting.minImages`（默认 4）
- **那么** readiness 评估结果必须为不达标，并包含"图片不足/上传失败导致不足"的原因

### 需求：系统必须支持精华策略的重试与降级
系统必须在��华策略无法满足最低门槛时自动降级为普通发帖策略，保证发帖任务可持续执行。

#### 场景：精华候选经重试仍不达标时降级
- **当** 精华候选在达到 `featuredPosting.maxGenerateRetries` 或 `featuredPosting.maxImageUploadRetries` 后仍不达标
- **那么** 系统必须将该次发帖切换为普通策略继续执行，并记录降级原因

### 需求：精华策略门槛必须可配置
系统必须允许通过配置调整精华策略的关键门槛与重试上限，以适配不同账号状态与素材库规模。

#### 场景：使用配置覆盖默认精华门槛
- **当** 用户在配置中设置 `featuredPosting.*`（例如最少字数、最少图片数、重试次数）
- **那么** 系统必须在评估与流程控制中使用这些配置值而不是固定常量
