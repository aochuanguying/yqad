## 新增需求

### 需求：系统必须检测帖子内容与历史内容的相似度

系统必须在发帖前将待发布内容与最近 7-14 天内的历史发帖进行相似度比对，当相似度超过阈值时判定为重复内容并拒绝发布。

#### 场景：相似度低于阈值
- **当** 待发布内容与历史发帖的最高相似度低于配置阈值（默认 0.75）
- **那么** 系统必须判定内容通过去重检查，允许继续发布流程

#### 场景：相似度超过阈值
- **当** 待发布内容与任一历史发帖的相似度超过配置阈值
- **那么** 系统必须判定内容为重复内容，拒绝发布并返回错误信息
- **并且** 错误信息必须包含相似度最高的历史发帖 ID 和相似度数值

#### 场景：无历史发帖记录
- **当** 系统中没有最近 7 天内的历史发帖记录
- **那么** 系统必须跳过相似度检测，判定内容通过去重检查

#### 场景：可配置的比对时间范围
- **当** 配置文件中设置了 `contentDeduplication.lookbackDays` 参数
- **那么** 系统必须使用配置的天数作为历史内容比对范围（默认 7 天，最大 30 天）

#### 场景：可配置的相似度阈值
- **当** 配置文件中设置了 `contentDeduplication.similarityThreshold` 参数
- **那么** 系统必须使用配置的阈值作为重复判定标准（默认 0.75，范围 0.5-0.95）

### 需求：系统必须使用 TF-IDF + 余弦相似度算法

系统必须使用 TF-IDF 算法将文本内容向量化，并使用余弦相似度计算向量间的相似度。

#### 场景：文本向量化
- **当** 系统接收到待检测的帖子内容
- **那么** 系统必须使用 TF-IDF 算法将文本转换为特征向量
- **并且** 必须对标题和正文分别赋予不同权重（标题权重更高）

#### 场景：相似度计算
- **当** 系统需要计算两段内容的相似度
- **那么** 系统必须使用余弦相似度公式计算向量夹角余弦值
- **并且** 相似度值必须在 0 到 1 之间（1 表示完全相同）

#### 场景：性能优化
- **当** 历史发帖数量超过 100 条
- **那么** 系统必须使用向量索引缓存，避免重复计算 TF-IDF 向量
- **并且** 单次相似度检测的耗时必须控制在 500ms 以内

### 需求：系统必须提供相似度检测 API

系统必须提供 HTTP API 供外部调用相似度检测功能。

#### 场景：调用相似度检测 API
- **当** 客户端发送 POST 请求到 `/api/compliance/check-similarity`
- **并且** 请求体包含 `content` 字段（帖子正文）和 `title` 字段（标题）
- **那么** 系统必须返回相似度检测结果，包含最高相似度值和匹配的历史发帖 ID

#### 场景：批量相似度检测
- **当** 客户端发送 POST 请求到 `/api/compliance/check-similarity/batch`
- **并且** 请求体包含多个帖子内容的数组
- **那么** 系统必须返回每个帖子的相似度检测结果
- **并且** 响应必须包含内部重复检测（检测批量内容之间的重复）
