## 修改需求

### 需求：系统必须自动生成并发布优质帖子

**修改说明**：增加发帖前合规性检查流程

系统必须定时发布帖子到社区。发帖时必须优先使用已配置且未使用的主题（含关联素材和提纲），当所有主题用完或无主题配置时，回退到基于 AI 自由生成的模式。系统必须在可用时优先按"精华策略"生成精华候选帖；当精华候选无法满足最低门槛时，系统必须自动降级为普通帖子发布。**每次发帖前，系统必须执行合规性检查（内容去重、敏感词过滤、质量评分、发帖间隔控制），只有检查通过的帖子才能发布。每次发帖完成后，系统必须记录发帖日志，包括触发方式（auto）、发帖模式、使用的主题、标题、内容、图片、状态、合规性检查报告 ID 等信息。**

#### 场景：优先使用已配置主题发帖

- **当** 自动发帖任务触发且存在"未使用"状态的主题
- **那么** 系统必须选取一个未使用的主题，使用其方向描述和内容提纲作为 AI 生成的约束，生成帖子内容，并附带主题关联的素材图片
- **并且** 系统必须在发帖前执行合规性检查
- **并且** 合规性检查必须包含：内容相似度检测、敏感词过滤、内容质量评分、主题发帖间隔控制
- **并且** 只有合规性检查通过时，系统才能发布帖子
- **并且** 发帖完成后必须记录日志，triggerType 为"auto"，postType 为"topic"，包含 topicId 和 topicName，complianceReportId 为合规性检查报告 ID

#### 场景：合规性检查拒绝发帖

- **当** 自动发帖任务生成的帖子内容未通过合规性检查
- **那么** 系统必须拒绝发布该帖子
- **并且** 系统必须记录拒绝日志，包含拒绝原因（如：相似度过高、包含敏感词、质量评分过低、发帖间隔不足）
- **并且** 系统必须生成合规性检查报告并存储
- **并且** 系统必须将该主题标记为"本次尝试失败"，但不标记为"已使用"
- **并且** 系统可以选择下一个可用主题重新尝试（最多尝试 3 次）

#### 场景：敏感词自动替换后发帖

- **当** 自动发帖任务生成的帖子内容包含"替换"级别的敏感词
- **那么** 系统必须自动替换敏感词
- **并且** 替换后的内容必须重新执行合规性检查
- **并且** 如果替换后内容通过所有检查，系统必须发布帖子
- **并且** 发帖日志必须标记 `sensitiveWordsReplaced: true`，记录替换的词汇数量

#### 场景：无可用主题时自由生成

- **当** 自动发帖任务触发且所有主题均已使用（或无主题配置）
- **那么** 系统必须回退到原有模式：从内容分析结果中选取热点话题，调用 AI 自由生成帖子
- **并且** 系统必须在发帖前执行合规性检查（跳过主题发帖间隔控制）
- **并且** 只有合规性检查通过时，系统才能发布帖子
- **并且** 发帖完成后必须记录日志，triggerType 为"auto"，postType 为"free"，topicId 和 topicName 为空，complianceReportId 为合规性检查报告 ID

#### 场景：发帖后标记主题已使用

- **当** 使用预配置主题成功发帖
- **那么** 系统必须将该主题标记为"已使用"并记录使用时间
- **并且** 必须在日志中记录使用的主题 ID 和主题名称
- **并且** 必须更新主题的 `lastPostDate` 字段为当前时间

#### 场景：合规性检查超时处理

- **当** 合规性检查执行时间超过配置超时时间（默认 5 秒）
- **那么** 系统必须记录警告日志
- **并且** 系统可以选择��过超时检查项或拒绝发布（由配置决定）
- **并且** 必须在合规性检查报告中标记超时的检查项

### 需求：远程发帖 API 必须执行合规性检查

**修改说明**：远程发帖 API（`/api/posts/generate`、`/api/posts/execute`）在生成或发布帖子前必须执行合规性检查。

#### 场景：远程 API 生成内容并检查合规性

- **当** 远程 API 调用 `POST /api/posts/generate`
- **那么** 系统必须生成帖子内容
- **并且** 系统必须在返回内容前执行合规性检查
- **并且** 响应必须包含 `complianceCheck` 字段，包含检查结果和报告 ID
- **并且** 如果检查不通过，API 必须返回 400 错误，包含拒绝原因

#### 场景：远程 API 执行发帖并检查合规性

- **当** 远程 API 调用 `POST /api/posts/execute`
- **那么** 系统必须生成帖子内容并执行发帖操作
- **并且** 系统必须在发帖前执行合规性检查
- **并且** 只有检查通过时，系统才能执行发帖
- **并且** 响应必须包含 `complianceReportId` 字段

#### 场景：远程 API 批量发帖的合规性检查

- **当** 远程 API 调用 `POST /api/posts/batch`
- **那么** 系统必须为每篇待发布内容独立执行合规性检查
- **并且** 系统必须只发布通过合规性检查的内容
- **并且** 响应必须包含每篇内容的检查结果和发布状态
