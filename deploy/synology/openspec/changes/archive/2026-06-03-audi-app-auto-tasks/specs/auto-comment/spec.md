## 新增需求

### 需求: 系统必须自动选择帖子并发布AI生成的评论
系统必须从帖子列表中智能选择适合评论的帖子，生成评论内容并发布。

#### 场景: 选择评论目标帖子
- **当** 自动评论任务触发
- **那么** 系统必须从最新帖子中选择尚未评论过的帖子作为评论目标

#### 场景: 生成并发布评论
- **当** 确定评论目标帖子后
- **那么** 系统必须调用AI内容生成模块生成评论，然后调用发布评论API发布

#### 场景: 每日评论数量可配置
- **当** 执行每日评论任务
- **那么** 系统必须根据配置的每日评论数量上限（默认3条）执行评论

### 需求: 自动评论必须避免重复和异常行为
系统必须记录已评论的帖子，避免重复评论，并在评论间加入随机延时。

#### 场景: 避免重复评论同一帖子
- **当** 系统选择评论目标时
- **那么** 系统必须排除已经评论过的帖子（基于本地记录）

#### 场景: 评论间随机延时
- **当** 连续发布多条评论时
- **那么** 每条评论之间必须等待随机时间间隔（配置范围，默认30-120秒）
