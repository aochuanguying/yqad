## 新增需求

### 需求: 系统必须通过大模型生成符合社区风格的内容
系统必须调用ChatGPT等大模型API，基于内容分析结果生成自然、贴合社区氛围的评论和帖子。

#### 场景: 生成评论内容
- **当** 系统需要对某帖子发布评论
- **那么** 系统必须将帖子内容和社区分析摘要作为上下文，调用大模型生成一条相关评论

#### 场景: 生成帖子内容
- **当** 系统需要发布新帖子
- **那么** 系统必须基于热点话题和社区风格，调用大模型生成包含标题和正文的帖子

#### 场景: 内容质量约束
- **当** 大模型返回生成内容
- **那么** 生成的评论长度必须在20-200字之间，帖子正文必须在100-800字之间

### 需求: 大模型调用必须支持配置和容错
系统必须支持配置不同的大模型服务地址和模型名称，并处理调用失败的情况。

#### 场景: 可配置的模型参数
- **当** 读取配置文件时
- **那么** 系统必须支持配置apiKey、baseUrl、model名称、temperature等参数

#### 场景: API调用失败处理
- **当** 大模型API调用失败（超时、限流、服务不可用）
- **那么** 系统必须记录错误日志，等待后重试，最多重试2次；若仍失败则跳过本次生成

#### 场景: 支持多种兼容API
- **当** 配置了非OpenAI的兼容服务（如Azure OpenAI、本地模型）
- **那么** 系统必须通过统一的OpenAI协议格式正常调用并获取响应
