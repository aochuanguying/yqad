## 新增需求

### 需求：系统必须提供图片下载服务
系统必须将本地素材库中的图片转换为 HTTP 可访问的下载链接，供手机端远程下载，不再上传至奥迪服务器。

#### 场景：静态文件服务
- **当** 系统启动且 `web.enabled` 为 true
- **那么** 系统必须在 `/images` 路径下提供素材库的静态文件访问

#### 场景：图片下载 URL 生成
- **当** 生成帖子内容并选中图片
- **那么** 系统必须为每张图片生成完整的下载 URL（格式：`http://host:port/images/relative/path.jpg`）

#### 场景：相对路径转换
- **当** 图片路径为相对路径（如 `images/topic1/photo.jpg`）
- **那么** 系统必须将其转换为相对于 `materials.processedPath` 的 URL

#### 场景：文件不存在处理
- **当** 请求的图片文件不存在
- **那么** 系统必须返回 404 错误

#### 场景：非图片文件访问
- **当** 请求的文件不是图片格式
- **那么** 系统必须返回 403 错误

### 需求：系统必须提供图片信息元数据
系统必须在返回发帖内容时提供图片的完整元数据，包括下载 URL、相对路径、文件名和文件大小。

#### 场景：返回图片元数据
- **当** API 返回发帖内容
- **那么** 系统必须为每张图片提供 `url`、`relativePath`、`filename` 和 `size` 字段

#### 场景：计算文件大小
- **当** 生成图片信息
- **那么** 系统必须读取文件并计算其字节大小

## 修改需求

### 需求：图片上传逻辑变更
系统不再将图片上传至奥迪 CDN，而是提供本地下载 URL，由手机端下载后上传。

#### 场景：取消图片上传
- **当** 生成帖子内容时
- **那么** 系统不再调用 `uploadImages` API 上传图片到奥迪服务器

#### 场景：提供下载 URL
- **当** 选中图片后
- **那么** 系统必须生成可远程访问的 HTTP 下载 URL

#### 场景：图片选择逻辑保留
- **当** 选择图片时
- **那么** 系统必须保留现有的加权匹配算法、多角度策略和精华选图优化
