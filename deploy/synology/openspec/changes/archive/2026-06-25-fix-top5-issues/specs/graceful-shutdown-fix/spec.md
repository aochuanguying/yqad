## 新增需求

### 需求:优雅退出时必须关闭所有数据库连接

系统收到 SIGINT 或 SIGTERM 信号时，必须在停止调度器之后、退出进程之前，依次关闭 MySQL 连接池、Redis 连接和 ChromaDB 连接。

#### 场景:收到 SIGINT 信号时依次关闭所有连接
- **当** 系统收到 SIGINT 信号
- **那么** 系统依次执行：停止调度器 → 关闭 MySQL 连接池 → 断开 Redis → 关闭 ChromaDB 连接 → 退出进程

#### 场景:收到 SIGTERM 信号时依次关闭所有连接
- **当** 系统收到 SIGTERM 信号
- **那么** 系统依次执行：停止调度器 → 关闭 MySQL 连接池 → 断开 Redis → 关闭 ChromaDB 连接 → 退出进程

#### 场景:某个连接关闭失败时继续关闭其他连接
- **当** 关闭某个数据库连接时发生错误
- **那么** 系统记录错误日志并继续关闭剩余连接，不因单个失败而中断整个关闭流程
