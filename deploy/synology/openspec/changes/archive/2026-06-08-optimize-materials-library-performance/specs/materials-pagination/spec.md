## 新增需求

### 需求: Web UI 必须提供素材分页加载功能
系统必须在 Web 管理界面的素材库浏览功能中支持分页加载，以减少首屏渲染压力和提升滚动流畅度。

#### 场景: 初始加载素材
- **当** 用户点击"素材库"Tab
- **那么** 页面必须仅加载并渲染前 24 个素材（首屏 + 缓冲区），而非全部素材
- **并且** 页面必须显示当前加载进度（例如"已显示 24/150 个素材"）

#### 场景: 加载更多素材
- **当** 用户点击"加载更多"按钮
- **那么** 页面必须追加渲染下 24 个素材
- **并且** 页面必须更新加载进度显示
- **并且** 当所有素材加载完毕时，必须隐藏"加载更多"按钮

#### 场景: 切换目录筛选
- **当** 用户切换目录筛选器
- **那么** 页面必须重置分页状态，重新从第一页开始加载
- **并且** 页面必须仅显示当前选中目录下的素材

### 需求: Web UI 必须提供图片懒加载增强功能
系统必须使用 Intersection Observer API 实现更精确的图片懒加载控制，减少首屏图片并发请求数。

#### 场景: 图片进入视口
- **当** 素材卡片滚动进入视口（或接近视口）
- **那么** 系统必须开始加载该素材的图片

#### 场景: 图片离开视口
- **当** 素材卡片滚动离开视口
- **那么** 系统可以暂停或取消该图片的加载（如尚未完成）

## 修改需求

### 需求: Web UI 必须提供素材库浏览界面
系统必须在 Web 管理界面中提供素材库的可视化浏览功能。

#### 场景: 素材库 Tab 页展示
- **当** 用户点击"素材库"Tab
- **那么** 页面必须以网格缩略图方式展示所有可用素材
- **并且** 页面在"全部目录"模式下必须按素材的 directory（目录）分组展示（每个分组展示目录名称，根目录显示为"根目录"）
- **并且** 每个目录分组内素材必须按 filename 升序展示
- **并且** 页面必须采用分页加载策略，初始仅渲染首屏素材（24 个），支持"加载更多"按钮逐步加载
- **并且** 页面必须显示加载进度（例如"已显示 24/150 个素材"）
- **并且** 对浏览器无法直接预览的素材格式（例如部分环境下的 heic/heif），页面必须提供可用的降级方式（例如展示可点击的"打开文件"入口），禁止导致整页素材区域不可用
