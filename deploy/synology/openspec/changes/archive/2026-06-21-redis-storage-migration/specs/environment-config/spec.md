## 新增需求

### 需求：系统必须提供环境配置管理功能

系统必须根据环境变量自动切换测试和生产环境的数据库配置，实现环境隔离。

#### 场景：加载测试环境配置
- **当** `NODE_ENV=test` 时
- **那么** 系统必须使用测试环境配置（host: 192.168.50.50, db: 0, keyPrefix: "test:"）
- **并且** 必须通过 IP 地址访问 Redis

#### 场景：加载生产环境配置
- **当** `NODE_ENV=production` 时
- **那么** 系统必须使用生产环境配置（host: redis, db: 1, keyPrefix: "prod:"）
- **并且** 必须通过容器名访问 Redis（需加入 bridge 网络）

#### 场景：环境变量覆盖
- **当** 设置 `REDIS_HOST` 环境变量时
- **那么** 系统必须使用环境变量覆盖配置文件中的 host
- **并且** 必须优先于配置文件

#### 场景：加载默认配置
- **当** 未指定 NODE_ENV 时
- **那么** 系统必须使用默认配置（开发环境，host: localhost, db: 0）
- **并且** 必须记录警告日志提示未指定环境

### 需求：系统必须提供配置验证功能

系统必须在启动时验证配置的正确性，防止配置错误导致运行时故障。

#### 场景：验证 Redis 连接
- **当** 应用启动时
- **那么** 系统必须测试 Redis 连接
- **并且** 必须在连接失败时抛出错误并阻止应用启动

#### 场景：验证配置完整性
- **当** 加载配置时
- **那么** 系统必须检查必需字段（host, port, db）是否存在
- **并且** 必须在字段缺失时抛出错误

#### 场景：验证数据库编号
- **当** 配置 db 参数时
- **那么** 系统必须检查是否在有效范围（0-15）
- **并且** 必须在超出范围时抛出错误
