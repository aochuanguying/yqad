## 新增需求

### 需求：系统必须提供 API Token 的 Redis 存储功能

系统必须从零开始使用 Redis String 存储 API Token，支持加密存储、安全读取和自动过期。

**约束**：
- 项目从零开始，没有历史数据需要迁移
- 只使用 Redis 存储，不涉及文件存储
- 使用 String 数据结构（简单键值对）

#### 场景：保存 API Token
- **当** 调用 `saveToken(token, options)` 方法时
- **那么** 系统必须使用 `SET` 命令存储 Token
- **并且** 必须使用 AES-256 加密后存储
- **并且** 必须设置过期时间（如果指定）

#### 场景：读取 API Token
- **当** 调用 `getToken()` 方法时
- **那么** 系统必须使用 `GET` 命令获取 Token
- **并且** 必须解密后返回明文

#### 场景：删除 API Token
- **当** 调用 `deleteToken()` 方法时
- **那么** 系统必须使用 `DEL` 命令删除 Token
- **并且** 必须返回删除是否成功

#### 场景：检查 Token 是否存在
- **当** 调用 `hasToken()` 方法时
- **那么** 系统必须使用 `EXISTS` 命令检查
- **并且** 必须返回布尔值结果

#### 场景：获取 Token 元数据
- **当** 调用 `getTokenMetadata()` 方法时
- **那么** 系统必须返回 Token 的创建时间、过期时间、剩余 TTL
- **并且** 必须使用 `TTL` 命令获取剩余生存时间

### 需求：系统必须支持 Token 加密存储

系统必须对 API Token 进行加密存储，防止数据泄露。

#### 场景：加密 Token
- **当** 保存 Token 时
- **那么** 系统必须使用 AES-256-GCM 算法加密
- **并且** 必须使用环境变量 `TOKEN_ENCRYPTION_KEY` 作为密钥

#### 场景：解密 Token
- **当** 读取 Token 时
- **那么** 系统必须使用相同的密钥和算法解密
- **并且** 必须在解密失败时抛出错误
