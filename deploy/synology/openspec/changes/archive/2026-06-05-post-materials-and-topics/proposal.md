## 为什么

当前自动发帖完全依赖 AI 自由生成，缺乏对发帖内容方向的精细控制。需要引入素材库和主题配置机制，让用户能够预设帖子的主题方向、内容提纲和配图素材，使发帖内容更有针对性和真实感。同时，系统运行在内网环境中，Web 管理界面无需认证和字段脱敏，需要简化访问流程。

## 变更内容

- **新增**：素材库管理 —— 配置一个 NAS 路径作为素材目录，系统扫描并管理其中的图片资源（旅行照片、车辆图片等），在 Web UI 中提供素材浏览和选择功能
- **新增**：发帖主题配置 —— 用户可在 Web UI 中创建多个发帖主题，每个主题包含方向描述、内容提纲、关联的素材图片；发帖时优先消费已配置且未使用的主题，所有主题用完或无主题配置时回退到 AI 自由生成
- **修改**：移除 Web 管理界面的 Basic Auth 认证（内网系统无需登录）
- **修改**：移除 Web 前端的敏感字段脱敏（password 类型输入改为普通 text 输入）
- **修改**：自动发帖流程调整为优先使用预配置主题

## 功能 (Capabilities)

### 新增功能

- `post-materials`: 素材库管理功能，包括素材目录配置、图片扫描索引、素材浏览 API 和 Web UI 展示
- `post-topics`: 发帖主题配置功能，包括主题 CRUD、主题与素材关联、主题使用状态追踪、Web UI 管理界面

### 修改功能

- `auto-post`: 发帖逻辑调整为优先消费已配置主题（含关联素材和提纲），全部用完后回退到原有的 AI 自由生成模式
- `config-web-ui`: 移除 Basic Auth 认证和敏感字段脱敏，简化内网访问

## 影响

- **配置新增**：`materials.basePath`（素材库 NAS 路径）、`topics` 数据存储
- **API 新增**：素材浏览 API (`/api/materials`)、主题管理 API (`/api/topics`)
- **数据存储**：新增 `data/topics.json` 存储主题配置和使用状态
- **Web UI**：新增素材库浏览页面和主题管理页面（Tab 扩展）
- **代码修改**：`src/web/middleware/auth.ts` 移除、`src/services/auto-post.ts` 接入主题优先逻辑
- **文件系统依赖**：系统需要对配置的 NAS 路径有读取权限
