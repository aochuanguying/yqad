## 为什么

当前 Web 管理端“素材库”在素材数量较多时展示顺序不稳定，出现目录与图片混排、刷新后顺序变化等“显示错乱”的观感问题，影响素材查找与选用效率。

## 变更内容

- **修改**：素材库列表默认按“分类（目录）”分组展示，同一目录下素材按稳定规则排序
- **修改**：素材库 API 返回的素材列表顺序稳定（同目录内按名称排序，目录间按名称排序），避免前端依赖文件系统扫描顺序

## 功能 (Capabilities)

### 新增功能

<!-- 无 -->

### 修改功能

- `post-materials`: 调整素材库浏览行为，明确 Web UI 默认按目录分组展示，并保证素材返回顺序稳定

## 影响

- **Web UI**：素材库 Tab 的展示逻辑需要支持“按目录分组”的渲染
- **API**：`GET /api/materials` 的返回 `items` 顺序将变为稳定排序（不改变字段结构，仅调整顺序）
