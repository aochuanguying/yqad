## 新增需求

### 需求：基于帖子特征构建评论 Prompt
系统必须根据实时分析的帖子特征构建评论生成的系统 Prompt，替代原有的社区分析摘要。

#### 场景：构建完整特征 Prompt
- **当** 帖子分析包含完整的类型、主题、情感、关键词信息
- **那么** Prompt 必须包含所有特征信息，格式为：
  ```
  【帖子特征】
  - 类型：{postType}
  - 主题：{topic}
  - 情感：{sentiment}
  - 关键词：{keywords}
  ```

#### 场景：部分特征缺失
- **当** 帖子分析部分特征缺失或置信度低
- **那么** 仅包含可信特征，省略不确定信息

### 需求：保留评论风格轮换
系统必须保留原有的评论风格轮换机制（共鸣、提问、经验分享、吐槽、简短）。

#### 场景：风格轮换
- **当** 生成多条评论时
- **那么** 按 batchIndex 轮换使用不同的风格模板

#### 场景：风格指令注入
- **当** 构建 Prompt 时
- **那么** 必须包含选定的风格指令：
  ```
  【你这次的评论风格】
  {selectedStyle.instruction}
  ```

### 需求：保留拟人化策略
系统必须保留原有的拟人化策略（口语化、避免废话开头、避免重复）。

#### 场景：口语化注入
- **当** 随机数 < 0.3 时
- **那么** Prompt 必须包含口语化要求：
  ```
  - 这次可以故意用一些口语化表达，比如"哈哈"、"emmm"、"绝了"等
  ```

#### 场景：避免重复开头
- **当** 提供最近 avoidRepeatDays 天内的评论开头列表
- **那么** Prompt 必须包含禁止使用的开头方式列表

#### 场景：兜底模式去重
- **当** 兜底模式下且提供上次评论内容
- **那么** Prompt 必须包含去重提示：
  ```
  【注意】你之前对这个帖子评论过："XXX"。这次必须换一个完全不同的角度
  ```

### 需求：核心规则保持不变
系统必须保留原有的核心评论规则。

#### 场景：核心规则注入
- **当** 构建 Prompt 时
- **那么** 必须包含以下核心规则：
  - 必须针对帖子内容中的具体细节来评论
  - 不要用"感谢分享"、"写得好"这类万能废话开头
  - 不要用排比句、列条目
  - 像真人一样，可以只说半句话、可以跑题
  - 评论长度 20-200 字，偏短更自然

### 需求：长度约束
系统必须确保生成的评论长度在配置范围内。

#### 场景：应用长度约束
- **当** 评论生成完成后
- **那么** 必须检查长度是否在 `contentLimits.comment.min` 和 `contentLimits.comment.max` 之间

#### 场景：超长截断
- **当** 评论长度超过最大值
- **那么** 截断到最近的句末（句号、感叹号、问号）

#### 场景：过短处理
- **当** 评论长度小于最小值
- **那么** 记录警告日志，使用原文
