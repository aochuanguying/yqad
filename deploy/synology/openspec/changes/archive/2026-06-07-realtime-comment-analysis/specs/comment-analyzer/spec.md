## 新增需求

### 需求：帖子类型识别
系统必须能够识别帖子的内容类型（纯文字、长图文、图文混合）。

#### 场景：纯文字帖子识别
- **当** 帖子 content 字段长度 >= 30 且 images 数组为空
- **那么** 返回类型为 "text-only"

#### 场景：长图文帖子识别
- **当** 帖子 content 字段长度 < 30 且 images 数组非空
- **那么** 返回类型为 "image-text-long"

#### 场景：图文混合帖子识别
- **当** 帖子 content 字段长度 >= 30 且 images 数组非空
- **那么** 返回类型为 "image-text-mixed"

### 需求：关键词提取
系统必须从帖子标题和内容中提取关键词，用于评论生成参考。

#### 场景：成功提取关键词
- **当** 帖子包含标题和内容
- **那么** 提取 3-5 个关键词（去除停用词，保留 2 字以上词汇）

#### 场景：无内容帖子
- **当** 帖子内容为空或过短
- **那么** 仅从标题提取关键词

### 需求：情感倾向分析
系统必须分析帖子的情感倾向（正面、中性、负面）。

#### 场景：正面情感识别
- **当** 帖子包含正面词汇（如"满意"、"喜欢"、"推荐"等）且数量超过负面词汇
- **那么** 返回情感为 "positive"

#### 场景：负面情感识别
- **当** 帖子包含负面词汇（如"失望"、"问题"、"故障"等）且数量超过正面词汇
- **那么** 返回情感为 "negative"

#### 场景：中性情感识别
- **当** 帖子不包含明显情感词汇或正负面词汇数量相当
- **那么** 返回情感为 "neutral"

### 需求：内容主题分类
系统必须将帖子内容分类到预定义的主题类别中。

#### 场景：选车主题识别
- **当** 帖子包含"选车"、"对比"、"试驾"、"购车"等关键词
- **那么** 返回主题为 "选车购车"

#### 场景：用车体验主题识别
- **当** 帖子包含"油耗"、"驾驶"、"操控"、"动力"等关键词
- **那么** 返回主题为 "用车体验"

#### 场景：保养维修主题识别
- **当** 帖子包含"保养"、"维修"、"4S 店"、"配件"等关键词
- **那么** 返回主题为 "保养维修"

#### 场景：新车资讯主题识别
- **当** 帖子包含"新车"、"上市"、"发布"、"改款"等关键词
- **那么** 返回主题为 "新车资讯"

#### 场景：自驾游主题识别
- **当** 帖子包含"自驾"、"游记"、"旅行"、"路况"等关键词
- **那么** 返回主题为 "自驾游"

### 需求：综合特征分析
系统必须提供单一接口返回帖子的所有分析特征。

#### 场景：成功分析完整帖子
- **当** 提供包含标题、内容、图片的完整帖子
- **那么** 返回包含类型、关键词、情感、主题的 PostFeatures 对象

#### 场景：降级分析
- **当** 帖子部分内容缺失（如无图片、无内容）
- **那么** 基于可用信息进行分析，缺失字段返回默认值
