## 上下文

**当前状态：**
- `AutoCommentService` 依赖 `ContentAnalysisService.getSummary()` 获取社区分析摘要
- 内容分析定时执行（每天 9 点），分析 60 篇最新帖子和热门评论
- 分析结果持久化到 `data/analysis.json`，包含社区风格、热点话题、示例文本等
- 评论生成时使用这些信息构建 Prompt

**约束：**
- 内容分析服务后期会完全删除
- 不能依赖任何 `analysis.json` 或相关缓存文件
- 回帖逻辑需要自包含、独立运行

**利益相关者：**
- 回帖功能用户（需要保持或提升评论质量）
- 系统维护者（简化架构、减少定时任务）

## 目标 / 非目标

**目标：**
- 移除 `AutoCommentService` 对 `ContentAnalysisService` 的依赖
- 实现帖子级别的实时内容分析
- 保持评论生成的自然度和拟人化特性
- 简化系统架构，减少定时任务数量

**非目标：**
- 不保留内容分析服务的其他功能（如热点话题追踪、示例文本学习）
- 不引入新的外部依赖
- 不改变评论历史记录机制
- 不改变兜底模式逻辑

## 决策

### 1. 实时分析粒度

**决策：** 采用"单帖子实时分析"而非"批量分析"

**理由：**
- 每日回帖数量有限（默认 3 条），实时分析开销可接受
- 每个帖子独立分析，更精准捕捉个性化特征
- 代码逻辑更简单，无需缓存管理

**考虑过的替代方案：**
- 批量分析当日待回复帖子 → 增加复杂性，收益不明显
- 混合模式（优先缓存，失效时分析）→ 仍需依赖缓存文件

### 2. 分析特征设计

**决策：** 提取以下帖子特征用于评论生成：
- 帖子类型（纯文字/长图文/图文混合）
- 内容长度（字数）
- 关键词（从标题和内容提取）
- 情感倾向（正面/中性/负面）
- 内容主题（用车/保养/新车/改装等）

**理由：**
- 这些特征足以支撑个性化的评论生成
- 不依赖历史数据，完全基于帖子本身
- 计算成本低，可实时完成

### 3. Prompt 重构策略

**决策：** 将原有的"社区风格"Prompt 改为"帖子特征"Prompt

**原 Prompt 结构：**
```
【社区氛围】${summary.styleDescription}
【你这次的评论风格】${selectedStyle.instruction}
【核心规则】...
```

**新 Prompt 结构：**
```
【帖子特征】
- 类型：${postType}
- 主题：${topic}
- 情感：${sentiment}
- 关键词：${keywords}
【你这次的评论风格】${selectedStyle.instruction}
【核心规则】...
```

**理由：**
- 保持评论风格轮换、口语化等拟人化特性
- 用帖子自身特征替代社区风格描述
- 评论仍能针对帖子内容，不会变得通用化

### 4. 服务架构

**决策：** 新增独立的 `CommentAnalyzer` 服务

**职责：**
- 分析单个帖子的内容特征
- 返回结构化的 `PostFeatures` 对象
- 不依赖任何外部服务或缓存文件

**代码组织：**
```
src/services/
  ├── comment-analyzer.ts    (新增)
  ├── auto-comment.ts        (修改：移除 ContentAnalysisService 依赖)
  └── content-analysis.ts    (后续删除)
```

## 风险 / 权衡

**风险 1：评论质量可能下降**
- 原因：失去社区风格、示例文本等全局上下文
- 缓解措施：
  - 强化帖子特征提取（关键词、情感分析）
  - 在 Prompt 中强调"针对帖子具体细节评论"
  - 保留风格轮换、口语化等拟人化策略

**风险 2：实时分析增加延迟**
- 原因：每条评论生成前需先分析帖子
- 缓解措施：
  - 分析逻辑保持轻量���正则 + 简单统计）
  - 与 AI 生成时间相比，分析开销可忽略
  - 批量分析时可采用并行处理

**风险 3：情感分析准确性**
- 原因：简单规则可能误判复杂情感
- 缓解措施：
  - 初期使用简单规则（正面词/负面词计数）
  - 后期可引入 AI 情感分析（调用 Vision 模型）
  - 情感判断错误对评论质量影响有限

**权衡：失去跨帖子一致性**
- 原有内容分析能追踪热点话题、风格演变
- 新方案仅关注单个帖子，可能丢失宏观视角
- 接受此权衡，因为回帖场景下单帖子上下文更关键
