## 新增需求

### 需求:源码必须覆盖构建产物引用的模块
系统必须在源码目录中提供主项目构建所需的全部模块，禁止依赖仅存在于 `dist` 的残留文件才能完成编译。

#### 场景:从源码构建主项目
- **当** 开发者在项目根目录运行 `npm run build`
- **那么** TypeScript 编译必须能够解析业务入口、Web 服务、AI 模块、服务模块、工具模块和类型模块的所有导入

#### 场景:恢复缺失模块路径
- **当** `dist/**/*.js.map` 指向 `src` 下的源路径
- **那么** 对应源路径必须存在可维护的源文件或被构建配置明确纳入的源模块

### 需求:配置基线必须安全且可覆盖
系统必须提供可提交的默认配置结构，并禁止在默认配置中保存可直接复用的真实访问密钥。

#### 场景:读取默认配置
- **当** 系统读取 `config/default.yaml`
- **那么** 配置文件必须保留运行所需字段结构
- **并且** API Key、用户凭据和上游访问密钥必须为空值、占位符或通过本地配置/环境变量覆盖

### 需求:Web 静态资源必须随构建输出
系统必须在主项目构建后提供 Web 管理界面所需静态资源。

#### 场景:构建 Web 管理界面
- **当** 开发者运行主项目构建命令
- **那么** `src/web/public` 中的静态页面必须出现在 `dist/web/public` 中
