## 新增需求

### 需求：系统必须支持管理员账号配置
系统应当提供配置项，允许管理员设置用户名和密码（哈希形式），支持配置文件和环境变量两种方式。

#### 场景：通过配置文件设置管理员账号
- **当** 管理员在 `config/default.yaml` 中设置 `web.auth.username` 和 `web.auth.passwordHash`
- **那么** 系统使用配置的凭据进行登录验证

#### 场景：通过环境变量设置管理员账号
- **当** 管理员设置环境变量 `WEB_AUTH_USERNAME` 和 `WEB_AUTH_PASSWORD_HASH`
- **那么** 系统优先使用环境变量的值覆盖配置文件

#### 场景：密码必须以哈希形式存储
- **当** 管理员配置密码时
- **那么** 密码必须使用 bcrypt 算法进行哈希处理，禁止明文存储

#### 场景：配置缺失时
- **当** 未配置管理员账号（配置文件和环境变量都为空）
- **那么** 系统启动时发出警告，认证功能无法正常使用

### 需求：系统必须提供密码哈希生成工具
系统应当提供脚本或工具，帮助管理员生成 bcrypt 密码哈希，避免手动操作的复杂性。

#### 场景：使用脚本生成密码哈希
- **当** 管理员运行 `npm run generate-password-hash`
- **那么** 系统提示输入密码，输出对应的 bcrypt 哈希值

#### 场景：密码哈希格式验证
- **当** 生成的密码哈希
- **那么** 哈希值必须以 `$2b$` 或 `$2a$` 开头，符合 bcrypt 格式规范

### 需求：Session Secret 必须可配置
系统应当提供 Session 签名密钥的配置方式，支持配置文件和环境变量。

#### 场景：通过配置文件设置 Session Secret
- **当** 管理员在配置中设置 `web.auth.sessionSecret`
- **那么** 系统使用该值作为 Session 的签名密钥

#### 场景：通过环境变量设置 Session Secret
- **当** 管理员设置环境变量 `WEB_AUTH_SESSION_SECRET`
- **那么** 系统优先使用环境变量的值

#### 场景：Session Secret 缺失时
- **当** 未配置 Session Secret
- **那么** 系统使用默认值（开发环境）或启动失败（生产环境）并发出警告
