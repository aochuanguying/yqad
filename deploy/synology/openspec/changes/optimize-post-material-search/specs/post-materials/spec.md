## 修改需求

### 需求：系统必须提供素材浏览 API
系统必须通过 HTTP API 暴露素材库的浏览能力，优先读取 `.materials/index.json` 索引，索引不存在时降级扫描文件系统。

#### 场景：获取素材列表
- **当** 客户端发送 GET /api/materials 请求
- **那么** 系统必须优先读取 `.materials/index.json` 索引文件
- **并且** 索引存在且非空时，系统必须基于索引数据返回素材列表，包含文件名、相对路径、文件大小和缩略图 URL
- **并且** 索引不存在或为空时，系统必须降级扫描文件系统并返回结果
- **并且** 系统必须返回稳定排序的 items：先按 directory 升序排序（根目录视为最小值），再按 filename 升序排序
- **并且** 系统必须将以下扩展名视为图片素材并纳入列表：jpg、jpeg、png、gif、webp、heic、heif

#### 场景：索引缓存
- **当** 首次读取索引文件后
- **那么** 系统必须缓存索引数据到内存，基于文件 mtime 判断缓存有效性
- **当** 索引文件未修改时再次请求
- **那么** 系统必须直接返回缓存数据，不重新读取文件
- **当** 索引文件被修改或重建后
- **那么** 系统必须使缓存失效并重新读取
