## 新增需求

### 需求：系统必须提供素材索引缓存机制
系统必须为 `.materials/index.json` 提供基于文件 mtime 的内存缓存，避免重复 IO 读取。

#### 场景：首次读取索引
- **当** 首次调用 `loadMaterialIndex()` 函数
- **那么** 系统必须读取 `.materials/index.json` 文件并缓存到内存中，同时记录文件 mtime

#### 场景：缓存命中
- **当** 再次调用 `loadMaterialIndex()` 且索引文件未修改
- **那么** 系统必须直接返回缓存的索引数据，不重新读取文件

#### 场景：缓存失效
- **当** 索引文件被修改（mtime 变化）
- **那么** 系统必须重新读取文件并更新缓存

#### 场景：梳理后缓存刷新
- **当** `rebuildMaterialIndex()` 完成索引重建
- **那么** 系统必须主动清空缓存，确保下次读取获取最新数据
