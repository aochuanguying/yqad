## 新增需求

### 需求：系统必须基于帖子标题和内容进行话题关联

系统必须在发布精华帖时，使用 AI 语义匹配算法，根据帖子的标题和正文内容，从热门话题列表中自动选择最相关的话题进行关联。

#### 场景：成功匹配话题
- **当** 系统准备发布一篇精华帖
- **那么** 系统调用 AI 语义匹配功能，传入帖子标题和正文内容
- **那么** AI 从候选热门话题中选择 0-5 个与帖子内容语义相关的话题

#### 场景：无匹配话题时降级处理
- **当** AI 语义匹配未找到相关话题
- **那么** 系统以无话题方式继续发布帖子
- **那么** 发帖流程不中断

#### 场景：话题匹配 API 失败时降级处理
- **当** 热门话题 API 请求失败或超时
- **那么** 系统返回空的话题列表
- **那么** 系统以无话题方式继续发布帖子
- **那么** 发帖流程不中断

### 需求：话题匹配必须使用 AI 语义分析

系统必须使用 AI 大模型进行语义分析，而不是简单的关键词匹配，以确保话题关联的准确性和相关性。

#### 场景：AI 语义分析
- **当** 系统调用话题匹配功能
- **那么** 系统将帖子标题和正文内容提交给 AI 模型
- **那么** AI 模型分析帖子的语义内容
- **那么** AI 模型从候选话题中选择语义相关的话题

#### 场景：避免强行匹配
- **当** 候选话题与帖子内容无明显语义关联
- **那么** AI 应返回空数组，不强行匹配话题

### 需求：话题数量必须限制在 5 个以内

系统必须限制每个帖子关联的话题数量不超过 5 个，以避免过度关联。

#### 场景：AI 返回超过 5 个话题
- **当** AI 语义匹配返回超过 5 个话题
- **那么** 系统只保留前 5 个话题
- **那么** 系统丢弃多余的话题

### 需求：发帖时必须传递话题列表到 API

系统必须在发帖 API 调用中，将匹配的话题列表通过 `topicList` 字段传递给服务端，以实现话题关联。

#### 场景：发布带话题的帖子
- **当** 话题匹配成功并返回至少 1 个话题
- **那么** 系统在发帖请求中包含 `topicList` 字段
- **那么** `topicList` 包含话题的 `id` 和 `name`
- **那么** 服务端将帖子与话题关联

#### 场景：发布无话题的帖子
- **当** 话题匹配失败或返回空列表
- **那么** 系统在发帖请求中 `topicList` 字段为空数组
- **那么** 帖子以无话题方式发布
