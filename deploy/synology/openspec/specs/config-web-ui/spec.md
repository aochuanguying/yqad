## 目的

提供一个可视化的单页面配置管理界面，允许用户通过浏览器直接查看和编辑所有系统配置项，无需认证即可访问。

## 需求

### 需求: 系统必须提供配置管理的 Web 页面
系统必须提供一个可视化的单页面应用，以表单方式展示和编辑所有配置项。Web 界面无需认证即可直接访问。

#### 场景: 访问配置页面
- **当** 用户通过浏览器访问根路径 /
- **那么** 系统必须直接返回配置管理的 HTML 页面，禁止要求任何形式的认证

#### 场景: 分组展示配置
- **当** 页面加载完成
- **那么** 页面必须将配置项按以下分组展示：API设置、用户认证、AI模型、签到、评论、发帖、内容分析、调度、日志、内容长度约束、素材库、发帖主题

#### 场景: 编辑并保存配置
- **当** 用户修改表单字段并点击保存按钮
- **那么** 系统必须将修改的配置发送到后端 API 并显示保存成功或失败的提示

#### 场景: 所有字段明文展示
- **当** 页面展示任何配置字段（包括密码和 API Key）
- **那么** 系统必须以普通文本输入框方式展示所有字段值，禁止使用 password 类型隐藏
