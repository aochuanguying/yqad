## 需求
### 需求：脚本列表展示
系统必须能够在 AutoJS 配置页面展示 AutoJS API 服务器上所有可用脚本的列表。

#### 场景：页面加载时自动获取脚本列表
- **当** 用户打开 AutoJS API 配置页面
- **那么** 系统自动调用 AutoJS API 的 `/api/scripts` 接口获取脚本列表

#### 场景：成功获取脚本列表
- **当** AutoJS API 返回有效的脚本列表
- **那么** 在配置页面以列表形式展示所有脚本名称

#### 场景：获取脚本列表失败
- **当** AutoJS API 不可用或返回错误
- **那么** 显示友好的错误提示并提供刷新按钮

#### 场景：手动刷新脚本列表
- **当** 用户点击刷新按钮
- **那么** 系统重新调用 AutoJS API 获取最新脚本列表
