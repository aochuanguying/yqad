## 新增需求

### 需求: 系统必须基于精华帖规则评估发帖准备度
系统必须将精华帖创作规则抽象为可执行的评估，并在发帖前对候选内容进行检查，输出可解释的未达标原因。

#### 场景: 评估精华帖准备度并返回原因
- **当** 系统生成一篇“精华候选帖”（含标题、正文、候选图片）准备发布
- **那么** 系统必须基于配置的精华门槛对候选帖进行 readiness 评估，并返回 `eligible`（是否达标）以及 `reasons`（未达标原因列表）

### 需求: 精华帖候选必须满足最低字数门槛
系统必须将“正文不少于 250 字”作为精华候选的最低门槛，并在不满足时触发降级逻辑。

#### 场景: 正文字数不足时判定不达标
- **当** 候选帖正文字符数小于 `featuredPosting.minContentChars`（默认 250）
- **那么** readiness 评估结果必须为不达标，并包含“字数不足”的原因

### 需求: 精华帖候选必须满足最低图片数量门槛
系统必须将“图片不少于 4 张”作为精华候选的最低门槛，并以“上传成功的图片 URL 数量”为准进行判定。

#### 场景: 上传成功图片不足时判定不达标
- **当** 候选帖最终上传成功的图片 URL 数量小于 `featuredPosting.minImages`（默认 4）
- **那么** readiness 评估结果必须为不达标，并包含“图片不足/上传失败导致不足”的原因

### 需求: 系统必须支持精华策略的重试与降级
系统必须在精华策略无法满足最低门槛时自动降级为普通发帖策略，保证发帖任务可持续执行。

#### 场景: 精华候选经重试仍不达标时降级
- **当** 精华候选在达到 `featuredPosting.maxGenerateRetries` 或 `featuredPosting.maxImageUploadRetries` 后仍不达标
- **那么** 系统必须将该次发帖切换为普通策略继续执行，并记录降级原因

### 需求: 精华策略门槛必须可配置
系统必须允许通过配置调整精华策略的关键门槛与重试上限，以适配不同账号状态与素材库规模。

#### 场景: 使用配置覆盖默认精华门槛
- **当** 用户在配置中设置 `featuredPosting.*`（例如最少字数、最少图片数、重试次数）
- **那么** 系统必须在评估与流程控制中使用这些配置值而不是固定常量
