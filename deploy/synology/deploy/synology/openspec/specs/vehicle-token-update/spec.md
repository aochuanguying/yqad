## 目的

定义车辆监控 Token 更新 API 的功能规范，支持外部系统通过 API 远程更新 Token 并热加载。

## 需求

### 需求：外部系统可以通过 API 更新车辆监控 Token
系统必须提供 RESTful API 端点，允许经过 API Token 鉴权的外部系统更新车辆监控 Token。Token 更新后必须立即热加载到内存中，无需重启服务。

#### 场景：成功更新 Token
- **当** 外部系统发送 POST 请求到 `/api/vehicle-token/update` 端点
- **当** 请求头包含有效的 API Token（`Authorization: Bearer <token>`）
- **当** 请求体包含有效的 `token` 字段（非空字符串）
- **那么** 系统返回 200 成功响应
- **那么** 系统将新 Token 保存到 `data/vehicle-token.json` 文件
- **那么** 系统将新 Token 热加载到内存中
- **那么** 后续的车辆监控请求使用新 Token

#### 场景：API Token 鉴权失败
- **当** 请求头缺少 `Authorization` 字段
- **当** 请求头格式不正确（不是 `Bearer <token>` 格式）
- **当** 提供的 API Token 无效
- **那么** 系统返回 401 未授权响应
- **那么** 系统不修改任何 Token 数据

#### 场景：请求参数验证失败
- **当** 请求体缺少 `token` 字段
- **当** `token` 字段为空字符串
- **当** `token` 字段不是字符串类型
- **那么** 系统返回 400 错误响应
- **那么** 系统提示具体的验证错误信息
- **那么** 系统不修改任何 Token 数据

#### 场景：文件写入失败
- **当** Token 文件所在目录不存在或不可写
- **那么** 系统返回 500 服务器错误响应
- **那么** 系统记录详细的错误日志
- **那么** 系统不修改内存中的 Token
