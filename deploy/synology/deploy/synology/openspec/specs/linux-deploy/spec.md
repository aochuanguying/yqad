## 新增需求

### 需求：系统必须支持通过 Docker Compose 部署网关容器

系统必须支持使用 Docker Compose 部署 HiGPT 网关容器，容器绑定到本地回环地址 `127.0.0.1:3000`，不对外暴露端口。

#### 场景：成功部署容器
- **当** 用户在部署目录执行 `docker-compose up -d`
- **那么** 容器启动并监听 `127.0.0.1:3000`
- **那么** 容器可以访问外部 HiGPT API

#### 场景：容器健康检查
- **当** 容器启动完成后
- **那么** 执行 `curl http://127.0.0.1:3000/health` 返回 200 状态码

### 需求：系统必须支持通过 Nginx 反向代理访问网关

系统必须配置 Nginx 反向代理，将 `https://域名/higpt/` 路径的请求转发到网关容器 `http://127.0.0.1:3000/`。

#### 场景：HTTP 重定向到 HTTPS
- **当** 用户访问 `http://域名/higpt/`
- **那么** Nginx 返回 301 重定向到 `https://域名/higpt/`

#### 场景：HTTPS 代理请求
- **当** 用户访问 `https://域名/higpt/v1/chat/completions`
- **那么** Nginx 将请求转发到 `http://127.0.0.1:3000/v1/chat/completions`
- **那么** 响应正确返回给用户

#### 场景：WebSocket 支持
- **当** 客户端发起 WebSocket 连接请求
- **那么** Nginx 正确传递 `Upgrade` 和 `Connection` 头
- **那么** WebSocket 连接成功建立

### 需求：系统必须配置 HTTPS（Let's Encrypt SSL 证书）

系统必须使用 Let's Encrypt 为域名申请并配置 SSL 证书，启用 HTTPS 访问。

#### 场景：成功申请证书
- **当** 执行 Certbot 申请证书
- **那么** 证书保存到 `/etc/lets/live/域名/`
- **那么** 证书有效期为 90 天

#### 场景：HTTPS 访问
- **当** 用户访问 `https://域名/higpt/`
- **那么** 浏览器显示有效的 SSL 证书
- **那么** 连接使用 TLS 1.2 或更高版本

### 需求：系统必须提供一键部署脚本

系统必须提供一键部署脚本 `deploy.sh`，自动化完成所有部署步骤。

#### 场景：成功执行部署脚本
- **当** 用户执行 `./deploy.sh`
- **那么** 脚本检查前置条件（Docker、Nginx、域名解析）
- **那么** 脚本创建配置文件（`.env`、`local.yaml`、Nginx 配置）
- **那么** 脚本申请 SSL 证书
- **那么** 脚本启动 Docker 容器
- **那么** 脚本执行健康检查

#### 场景：部署失败回滚
- **当** 部署过程中某一步失败
- **那么** 脚本回滚已完成的变更
- **那��** 脚本输出错误信息和恢复建议

### 需求：系统必须提供健康检查接口

系统必须提供健康检查脚本 `health-check.sh`，验证部署是否成功。

#### 场景：健康检查通过
- **当** 执行 `./health-check.sh`
- **那么** 检查容器运行状态
- **那么** 检查 Nginx 配置
- **那么** 检查 SSL 证书有效期
- **那么** 执行 API 测试请求
- **那么** 所有检查通过，返回 0 状态码

#### 场景：健康检查失败
- **当** 某个检查项失败
- **那么** 脚本输出详细错误信息
- **那么** 脚本返回非 0 状态码
