## 新增需求

### 需求：系统必须提供加权素材匹配算法
系统必须实现基于字段权重的素材匹配算法，按 tags、intro、directory、filename 等字段区分权重计算匹配分。

#### 场景：tags 字段高权重
- **当** 主题关键词命中素材的 tags 字段
- **那么** 系统必须为该命中赋予 3.0 的权重系数

#### 场景：intro 字段中权重
- **当** 主题关键词命中素材的 intro 字段
- **那么** 系统必须为该命中赋予 2.0 的权重系数

#### 场景：directory 字段中低权重
- **当** 主题关键词命中素材的 directory 字段
- **那么** 系统必须为该命中赋予 1.5 的权重系数

#### 场景：filename 字段低权重
- **当** 主题关键词命中素材的 filename 字段
- **那么** 系统必须为该命中赋予 1.0 的权重系数

#### 场景：2 字 token 降权
- **当** 关键词 token 长度为 2 个字符
- **那么** 系统必须将该 token 的权重乘以 0.5 系数

#### 场景：3 字及以上 token 正常权重
- **当** 关键词 token 长度为 3 个及以上字符
- **那么** 系统必须保持该 token 的权重为 1.0

#### 场景：按匹配分排序
- **当** 完成所有 token 的匹配分计算
- **那么** 系统必须按匹配分降序排列素材，选取最高分的素材
