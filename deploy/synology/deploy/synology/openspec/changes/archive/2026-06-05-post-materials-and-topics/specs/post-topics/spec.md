## 新增需求

### 需求: 系统必须提供发帖主题配置管理
系统必须支持用户创建、编辑、删除发帖主题，每个主题包含方向描述、内容提纲和关联素材。

#### 场景: 创建发帖主题
- **当** 用户提交新主题配置（包含标题、方向描述、可选的内容提纲、可选的关联素材路径列表）
- **那么** 系统必须将主题保存到 data/topics.json，状态标记为"未使用"

#### 场景: 编辑已有主题
- **当** 用户修改一个已存在主题的内容
- **那么** 系统必须更新该主题的配置，保留其使用状态不变

#### 场景: 删除主题
- **当** 用户删除一个主题
- **那么** 系统必须从存储中移除该主题，禁止影响其他主题

#### 场景: 查看所有主题
- **当** 客户端发送 GET /api/topics 请求
- **那么** 系统必须返回所有已配置主题的列表，包含主题内容和使用状态

### 需求: 系统必须追踪主题使用状态
系统必须记录每个主题的使用情况，确保主题不被重复使用。

#### 场景: 标记主题已使用
- **当** 某个主题被用于发帖
- **那么** 系统必须将该主题标记为"已使用"，并记录使用时间

#### 场景: 重置主题状态
- **当** 用户手动重置一个主题的使用状态
- **那么** 系统必须将该主题重新标记为"未使用"，使其可再次被发帖消费

#### 场景: 重置所有主题
- **当** 用户点击"全部重置"
- **那么** 系统必须将所有已使用的主题重新标记为"未使用"

### 需求: Web UI 必须提供主题管理界面
系统必须在 Web 管理界面中提供主题的可视化管理功能。

#### 场景: 主题列表展示
- **当** 用户点击"发帖主题"Tab
- **那么** 页面必须展示所有已配置主题的卡片列表，区分"未使用"和"已使用"状态

#### 场景: 创建主题表单
- **当** 用户点击"新建主题"按钮
- **那么** 页面必须展示主题编辑表单，包含标题、方向描述、内容提纲（多行文本）和素材选择器

#### 场景: 素材关联选择
- **当** 用户在主题编辑表单中点击"选择素材"
- **那么** 页面必须弹出素材库浏览组件，允许用户选择一张或多张图片关联到该主题
