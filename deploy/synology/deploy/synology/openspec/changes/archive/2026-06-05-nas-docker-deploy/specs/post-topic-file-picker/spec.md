## 新增需求

### 需求: Web 管理界面必须提供素材文件浏览 API
后端必须提供接口，列出素材库目录下的可用文件，供前端文件选择器使用。

#### 场景: 获取素材文件列表
- **当** 前端请求 `GET /api/materials/files`
- **那么** 后端必须返回素材库目录下所有文件的名称和相对路径列表（JSON 格式）

#### 场景: 素材库为空时的处理
- **当** 素材库目录为空或不存在
- **那么** 接口必须返回空数组，禁止返回错误状态码

#### 场景: 仅返回文件不返回目录
- **当** 素材库目录下存在子目录
- **那么** 接口必须只列出文件，禁止将目录项包含在结果中

### 需求: 发帖主题配置中的素材字段必须使用文件选择器
Web 管理界面中配置发帖主题时，素材附件字段必须提供可视化文件选择器，而不是手动输入路径的文本框。

#### 场景: 打开文件选择器
- **当** 用户在发帖主题配置页点击素材选择区域
- **那么** 界面必须展示素材库中的文件列表，支持单选或多选

#### 场景: 选择文件后填充配置
- **当** 用户从列表中选择一个或多个文件
- **那么** 对应的素材路径必须自动填入配置字段，禁止要求用户手动输入路径

#### 场景: 已选文件的展示
- **当** 主题配置中已有素材路径
- **那么** 界面必须显示已选文件名（而非原始路径字符串），并提供清除选择的操作

#### 场景: 文件列表加载失败的处理
- **当** 获取素材文件列表的 API 请求失败
- **那么** 界面必须显示友好的错误提示，禁止整个配置页面崩溃
