# 优化主题发帖内容多样化 - 任务清单

## 任务列表

### 任务 1: 扩展 Topic 数据结构
**优先级**: High  
**状态**: Pending  
**预计工时**: 1h

**描述**:
在 `src/web/services/topics-service.ts` 中扩展 Topic 接口，增加多样化相关的字段。

**具体工作**:
1. 在 Topic 接口中增加以下字段：
   - `subDirectionUsageCount?: number[]` - 子方向使用次数计数
   - `materialUsageHistory?: MaterialUsageRecord[]` - 素材使用记录
   - `usedTitleStyles?: TitleStyleType[]` - 已使用的标题风格
   - `lastUsedOutlineVariantIndex?: number` - 上次使用的提纲变体索引

2. 新增 `MaterialUsageRecord` 接口：
   ```typescript
   export interface MaterialUsageRecord {
     path: string;
     usedCount: number;
     lastUsedAt?: string;
     postIds?: string[];
   }
   ```

3. 新增 `TitleStyleType` 类型定义：
   ```typescript
   export type TitleStyleType = 
     | 'question'
     | 'sharing'
     | 'review'
     | 'comparison'
     | 'numeric'
     | 'suspense';
   ```

4. 更新 `saveTopics` 和 `loadTopics` 函数，确保新字段正确持久化

**验收标准**:
- [ ] Topic 接口包含所有新增字段
- [ ] 数据类型定义正确，TypeScript 编译通过
- [ ] 现有代码不受影响（新字段均为可选）
- [ ] 数据持久化正常工作

### 任务 2: 实现子方向轮换机制
**优先级**: High  
**状态**: Pending  
**预计工时**: 2h

**描述**:
修改 `selectNextSubDirection` 函数，从顺序选择改为基于使用次数的加权随机选择。

**具体工作**:
1. 实现基于使用次数的权重计算逻辑：
   - 权重 = 1 / (使用次数 + 1)
   - 使用次数越少，被选中的概率越高

2. 实现轮盘赌选择算法

3. 增加 `resetSubDirectionCountsIfNeeded` 函数：
   - 当最大最小使用次数差小于阈值时，重置计数

4. 更新 `incrementUseCount` 函数，同步更新子方向使用计数

**验收标准**:
- [ ] 子方向选择符合加权概率分布
- [ ] 使用次数记录正确递增
- [ ] 重置机制正常工作
- [ ] 单元测试覆盖率达到 80%

## 风险点

1. **AI 调用成本**: 变体生成会增加 API 调用次数，需监控成本
2. **性能影响**: 多样化策略可能增加发帖时间，需优化
3. **数据兼容性**: 旧数据迁移需妥善处理
4. **质量把控**: 多样化不能以牺牲质量为代价，需严格验证

## 注意事项

1. 所有新字段均为可选，确保向后兼容
2. 多样化策略失败时，需有降级方案
3. 记录详细的日志，便于问题排查
4. 与第一步合规性检查功能保持独立，互不影响
