## 修改需求

### 需求: 系统必须提供灵活的定时任务调度能力
系统必须支持配置各任务的执行时间、频率和优先级，统一管理所有自动化任务的调度。调度器必须支持运行时配置热重载。

#### 场景: 配置签到任务时间
- **当** 系统启动时读取调度配置
- **那么** 系统必须按配置的cron表达式（默认每天8:00-9:00随机时间）触发签到任务

#### 场景: 配置评论任务时间
- **当** 系统启动时读取调度配置
- **那么** 系统必须按配置的cron表达式（默认每天10:00-20:00区间内随机分散）触发评论任务

#### 场景: 配置发帖任务时间
- **当** 系统启动时读取调度配置
- **那么** 系统必须按配置的cron表达式（默认每天12:00-18:00随机时间）触发发帖任务

#### 场景: 配置热重载后重新调度
- **当** 系统收到配置变更事件且调度相关配置发生变化
- **那么** 系统必须取消当前所有已注册的定时任务，使用新配置重新创建并注册定时任务，禁止丢失当天已执行的任务记录
