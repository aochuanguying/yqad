## 新增需求

### 需求: 调度器必须支持素材梳理任务的定时触发
调度器必须支持通过配置启用素材梳理任务，并按配置的 cron 表达式定时触发素材梳理流程。

#### 场景: 配置素材梳理任务时间
- **当** 系统启动时读取调度配置
- **并且** 配置中存在素材梳理任务的 cron 表达式（例如 scheduler.materialProcessing.cron）
- **那么** 系统必须按 cron 表达式定时触发一次素材梳理任务

#### 场景: 防止并发重复执行
- **当** 上一次素材梳理任务尚未结束
- **并且** 新一次素材梳理任务到达触发时间
- **那么** 系统必须禁止并发启动第二个素材梳理任务
- **并且** 系统必须记录告警日志

### 需求: 配置热重载后必须重新调度素材梳理任务
当调度相关配置发生变化时，调度器必须重新注册素材梳理任务，保证新配置生效。

#### 场景: 素材梳理任务 cron 变更后重新调度
- **当** 系统收到配置变更事件且素材梳理任务的 cron 配置发生变化
- **那么** 系统必须取消当前已注册的素材梳理任务
- **并且** 系统必须使用新配置重新创建并注册素材梳理任务
