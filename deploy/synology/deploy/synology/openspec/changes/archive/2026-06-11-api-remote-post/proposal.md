## 为什么

当前通过 API 发帖无法触发奥迪社区的奖励任务系统，导致发帖后无法获得相应的积分奖励。为了解决这个问题，我们已经通过手机自动化实现了模拟人工发帖。现在需要将发帖逻辑从服务端直接发布改为远程调用模式，使手机端能够接管实际的发布操作。

## 变更内容

- **新增**：远程发帖 API 接口，供手机端调用获取发帖内容（标题、正文、图片下载地址）
- **修改**：取消定时发帖任务，改为 API 触发模式
- **修改**：图片上传逻辑变更，不再上传至奥迪服务器，改为提供本地图片下载地址
- **保留**：核心发帖逻辑（主题选择、内容生成、图片匹配、精华策略）全部保留

## 功能 (Capabilities)

### 新增功能
- `remote-post-api`: 远程发帖 API 接口，包括发帖内容生成、图片下载地址提供、API 鉴权
- `image-download-service`: 图片下载服务，将本地素材转换为可远程访问的下载链接

### 修改功能
- `auto-post`: 发帖服务从定时触发改为 API 触发，保留所有核心逻辑
- `scheduler`: 移除定时发帖任务配置

## 影响

- **代码影响**：
  - 新增 Web API 路由：`POST /api/posts/generate`
  - 修改 `AutoPostService`：将发布逻辑改为返回发帖内容
  - 新增图片下载服务：提供 HTTP 可访问的图片下载链接
  - 修改 scheduler 配置：移除或禁用定时发帖任务
- **API 影响**：新增 RESTful 接口供手机端调用
- **配置影响**：需要配置图片下载服务的基础 URL 和端口
- **部署影响**：需要确保素材目录可通过 HTTP 访问（或配置 CDN）
