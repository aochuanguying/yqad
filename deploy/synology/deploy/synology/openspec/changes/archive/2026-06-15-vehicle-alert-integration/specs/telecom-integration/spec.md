## 新增需求

### 需求：系统必须能够调用 Android Telecom API 发送短信

系统必须提供短信发送功能，通过调用 Android Telecom API 的 `/api/v1/sms/send` 端点实现。短信内容必须包含车辆异常的详细信息。

#### 场景：成功发送短信
- **当** 系统调用短信发送接口并提供有效的手机号和异常信息
- **那么** 系统向 Android Telecom API 发送 POST 请求
- **那么** API 返回成功响应，表示短信已发送

#### 场景：短信发送失败
- **当** Android Telecom API 不可用或返回错误
- **那么** 系统记录错误日志
- **那么** 系统继续尝试拨打电话通知

### 需求：系统必须能够调用 Android Telecom API 拨打电话

系统必须提供电话拨打功能，通过调用 Android Telecom API 的 `/api/v1/call` 端点实现。电话用于提醒用户查看车辆异常信息。

#### 场景：成功拨打电话
- **当** 系统调用电话拨打接口并提供有效的手机号
- **那么** 系统向 Android Telecom API 发送 POST 请求
- **那么** API 返回成功响应，表示电话已拨出

#### 场景：电话拨打失败
- **当** Android Telecom API 不可用或返回错误
- **那么** 系统记录错误日志
- **那么** 系统不自动重试，等待下次告警周期

### 需求：系统必须验证 Telecom API 的配置

系统必须在使用 Telecom API 前验证配置的有效性，包括 API 地址和 Token 的配置检查。

#### 场景：配置验证通过
- **当** API 地址和 Token 均已正确配置
- **那么** 系统允许调用短信和电话功能

#### 场景：配置验证失败
- **当** API 地址或 Token 未配置或为空
- **那么** 系统禁止调用 Telecom API
- **那么** 系统记录警告日志，提示配置缺失
