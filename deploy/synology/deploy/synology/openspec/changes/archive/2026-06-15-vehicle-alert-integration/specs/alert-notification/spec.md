## 新增需求

### 需求：系统必须在检测到车辆异常时发送告警短信

当车辆监控系统检测到异常（如车窗未关、车辆移动、安全距离超标等）时，系统必须首先发送告警短信到配置的手机号。短信内容必须包含异常类型和发生时间。

#### 场景：异常检测触发短信告警
- **当** 车辆监控系统检测到异常状态
- **那么** 系统构造包含异常类型和时间的短信内容
- **那么** 系统调用 Telecom API 发送短信
- **那么** 系统记录短信发送状态

#### 场景：短信内容格式
- **当** 系统构造短信内容
- **那么** 短信必须包含异常类型（如"车窗未关"、"车辆移动"）
- **那么** 短信必须包含异常发生时间
- **那么** 短信必须包含车辆位置信息（如可获取）

### 需求：系统必须在发送短信后拨打电话提醒

在成功发送告警短信后，系统必须等待 5 秒，然后拨打电话到配置的手机号进行语音提醒。电话用于确保用户及时注意到告警信息。

#### 场景：短信发送成功后拨打电话
- **当** 短信成功发送
- **那么** 系统等待 5 秒间隔
- **那么** 系统调用 Telecom API 拨打电话
- **那么** 系统记录电话拨打状态

#### 场景：短信发送失败仍尝试电话通知
- **当** 短信发送失败
- **那么** 系统记录短信发送失败日志
- **那么** 系统仍然尝试拨打电话通知
- **那么** 系统确保至少一种通知方式被尝试

### 需求：系统必须实现告警冷却机制

系统必须实现 30 分钟的告警冷却时间，避免同一异常在短时间内重复通知用户。冷却时间从上次告警通知开始计算。

#### 场景：冷却时间内抑制告警
- **当** 距离上次告警不足 30 分钟
- **那么** 系统跳过本次告警通知
- **那么** 系统记录冷却时间剩余时长
- **那么** 系统继续记录异常状态但不通知

#### 场景：冷却时间结束后允许告警
- **当** 距离上次告警超过 30 分钟
- **那么** 系统允许发送新的告警通知
- **那么** 系统更新上次告警时间

### 需求：系统必须处理 Telecom API 调用超时

系统必须设置 10 秒的 API 调用超时时间，避免网络问题导致告警流程阻塞。超时时应记录错误并继续流程。

#### 场景：API 调用超时
- **当** Telecom API 在 10 秒内未响应
- **那么** 系统终止当前 API 调用
- **那么** 系统记录超时错误
- **那么** 系统继续尝试下一步通知（如短信超时则尝试电话）

#### 场景：API 调用正常响应
- **当** Telecom API 在超时时间内响应
- **那么** 系统处理 API 响应
- **那么** 系统记录通知结果
