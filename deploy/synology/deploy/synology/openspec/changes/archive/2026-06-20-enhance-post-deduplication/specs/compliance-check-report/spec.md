## 新增需求

### 需求：系统必须生成合规性检查报告

系统必须在每次发帖前生成合规性检查报告，记录所有检查项的结果。

#### 场景：生成完整的合规性检查报告
- **当** 系统完成发帖前的所有合规性检查
- **那么** 系统必须生成一份合规性检查报告
- **并且** 报告必须包含：检查时间、帖子 ID、检查结果（通过/拒绝）、各项检查得分和详情

#### 场景：报告包含相似度检测结果
- **当** 执行内容去重检查
- **那么** 报告必须包含：最高相似度值、匹配的历史发帖 ID、比对的历史发帖数量、比对时间范围

#### 场景：报告包含敏感词检测结果
- **当** 执行敏感词过滤检查
- **那么** 报告必须包含：检测到的敏感词列表（级别、位置、替换词）、敏感词总数、处理策略

#### 场景：报告包含质量评分结果
- **当** 执行内容质量评分
- **那么** 报告必须包含：各维度得分（完整性、原创性、多样性、吸引力）、权重、加权得分、总分、评估详情

#### 场景：报告包含发帖间隔检查结果
- **当** 执行发帖间隔控制检查
- **那么** 报告必须包含：主题 ID、主题名称、最后发帖时间、已间隔天数、最短间隔要求、是否通过

#### 场景：报告存储为独立 JSON 文件
- **当** 生成合规性检查报告
- **那么** 系统必须将报告存储为 `data/compliance-reports/:postId.json` 文件
- **并且** 文件命名必须使用帖子 ID，便于查询和回溯

### 需求：系统必须提供合规性检查报告查询 API

系统必须提供 HTTP API 供外部查询合规性检查报告。

#### 场景：查询单个报告详情
- **当** 客户端发送 GET 请求到 `/api/compliance/reports/:postId`
- **那么** 系统必须返回指定帖子 ID 的完整合规性检查报告
- **并且** 响应必须包含报告的所有字段

#### 场景：查询报告列表
- **当** 客户端发送 GET 请求到 `/api/compliance/reports?page=1&limit=20&status=rejected`
- **那么** 系统必须返回分页的合规性检查报告列表
- **并且** 响应必须支持按检查结果筛选（`status=passed` 或 `status=rejected`）
- **并且** 响应必须包含总记录数和当前页数据

#### 场景：按时间范围查询报告
- **当** 客户端发送 GET 请求到 `/api/compliance/reports?startDate=2026-06-01&endDate=2026-06-20`
- **那么** 系统必须返回指定时间范围内的合规性检查报告列表
- **并且** 响应必须按检查时间降序排序

#### 场景：查询统计信息
- **当** 客户端发送 GET 请求到 `/api/compliance/reports/stats?period=7d`
- **那么** 系统必须返回统计信息，包含：总检查数、通过率、平均质量评分、主要拒绝原因分布
- **并且** `period` 参数支持：`7d`、`30d`、`90d`

### 需求：系统必须清理过期的合规性检查报告

系统必须定期清理过期的合规性检查报告，避免存储空间无限增长。

#### 场景：定期清理过期报告
- **当** 系统运行定时清理任务（默认每天凌晨 2 点）
- **那么** 系统必须删除超过配置保留天数的合规性检查报告
- **并且** 默认保留天数为 90 天（可配置）

#### 场景：清理任务日志记录
- **当** 执行清理任务
- **那么** 系统必须记录清理日志，包含：删除的报告数量、保留的报告数量、执行时间

#### 场景：可配置的保留天数
- **当** 配置文件中设置了 `complianceCheckReport.retentionDays` 参数
- **那么** 系统必须使用配置的天数作为报告保留期限（默认 90 天，范围 30-365 天）

### 需求：系统必须在发帖日志中关联合规性检查报告

系统必须在发帖日志中记录合规性检查报告的 ID，便于追溯和审计。

#### 场景：发帖日志包含报告 ID
- **当** 系统记录发帖日志
- **那么** 日志必须包含 `complianceReportId` 字段，指向对应的合规性检查报告
- **并且** 对于发帖前检查被拒绝的内容，也必须记录日志和生成报告

#### 场景：发帖日志包含检查结果摘要
- **当** 系统记录发帖日志
- **那么** 日志必须包含检查结果摘要：是否通过、质量评分总分、检测到的敏感词数量
- **并且** 摘要信息必须便于快速浏览，无需查询完整报告
