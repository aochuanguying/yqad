## 修改需求

### 需求: 系统必须获取并分析社区现有内容
系统必须在业务按需触发时获取社区最新帖子和评论，提取热点话题、高频关键词和语言风格特征，为AI内容生成提供上下文。当评论查询不可用时，系统必须降级为仅分析帖子内容。系统禁止通过调度器定期触发社区内容分析。

#### 场景: 按需获取最新帖子进行分析
- **当** 内容分析模块被业务流程按需触发
- **那么** 系统必须获取最近N篇帖子（可配置，默认20篇）的标题和内容

#### 场景: 获取热门帖子的评论
- **当** 获取到帖子列表后
- **那么** 系统必须尝试获取互动量最高的帖子下的评论内容

#### 场景: 评论接口返回空结果时降级
- **当** getComments 返回空数组（真实模式下暂无评论查询端点）
- **那么** 系统必须仅基于帖子内容执行分析，禁止因缺少评论数据而中断流程

#### 场景: 提取内容特征
- **当** 帖子和评论数据获取完成
- **那么** 系统必须生成包含热点话题、常见表达方式、内容长度分布的分析摘要

#### 场景: 禁止定时触发内容分析
- **当** 系统启动调度器或执行遗漏任务检查
- **那么** 系统禁止定时或补执行内容分析模块

### 需求: 分析结果必须结构化存储供后续使用
系统必须将分析结果以结构化格式保存，供AI内容生成模块直接使用。

#### 场景: 保存分析摘要
- **当** 内容分析完成
- **那么** 系统必须将分析摘要保存为JSON格式，包含话题列表、风格描述、示例文本

#### 场景: 分析结果有时效性
- **当** 分析结果超过24小时未更新
- **那么** 系统必须在下次生成内容前重新执行分析
