## 新增需求

### 需求：标题参考策略
系统必须将主题标题视为参考而非约束，AI 生成标题时可以基于子方向、提纲和全局人设自由发挥，不需要与主题标题保持完全一致。

#### 场景：生成标题时的约束
- **当** AI 生成帖子标题时
- **那么** 必须仅将主题标题作为风格参考，实际标题应基于当前选取的子方向标题和提纲生成

#### 场景：标题风格一致性
- **当** 同一主题多次发帖时
- **那么** 生成的标题应保持风格一致性，但具体措辞和角度应有所差异

### 需求：智能标题去重
系统必须实现智能的标题去重机制，仅比对同主题历史发帖的标题，而非与主题标题本身比对，确保新标题与历史发帖不重复。

#### 场景：标题去重范围
- **当** 检查标题是否重复时
- **那么** 必须仅与同主题的 postHistory 中所有帖子的标题进行比对，不与主题标题比对

#### 场景：标题去重算法
- **当** 生成新标题后
- **那么** 必须遍历该主题的 postHistory，检查新标题是否与任意历史帖子的标题完全一致

#### 场景：标题重复重试
- **当** 生成的标题与历史帖子标题重复时
- **那么** 必须重新调用 AI 生成新标题，最多重试 2 次

#### 场景：重试耗尽处理
- **当** 重试 2 次后生成的标题仍然重复时
- **那么** 必须放弃该主题，返回失败并记录日志，由调度器选择下一个可用主题

### 需求：标题生成 Prompt 优化
系统必须优化标题生成的 Prompt，明确告知 AI 标题仅作为参考，鼓励基于子方向和提纲进行创新。

#### 场景：构建 Prompt 时
- **当** 为 AI 生成构建 System Prompt 时
- **那么** 必须包含以下说明：
  - "主题标题仅供参考，不需要保持一致"
  - "请根据子方向和内容提纲生成吸引人的标题"
  - "标题应体现帖子内容的核心亮点"

#### 场景：Prompt 差异化
- **当** 主题有多个子方向时
- **那么** Prompt 中必须突出当前子方向的独特性，帮助 AI 生成差异化的标题
