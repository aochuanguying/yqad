## 修改需求

### 需求：精华帖候选必须满足汇总字数门槛
系统必须将"标题 + 正文 + 话题名称"的汇总字数作为精华候选的评估标准，并使用配置的字数范围进行判定。

#### 场景：汇总字数符合配置范围
- **当** 候选帖的汇总字数（标题 + 正文 + 话题名称）在配置的 `featuredPosting.minContentChars` 和 `maxBodyLength` 范围内
- **那么** readiness 评估结果必须判定字数达标

#### 场景：汇总字数不足
- **当** 候选帖的汇总字数小于 `featuredPosting.minContentChars`（默认 250）
- **那么** readiness 评估结果必须为不达标，并包含"汇总字数不足"的原因

#### 场景：汇总字数超限
- **当** 候选帖的汇总字数超过 `maxBodyLength`（默认 480 字预留空间）
- **那么** readiness 评估结果必须为不达标，并包含"汇总字数超限"的原因

## 新增需求

### 需求：系统必须提供汇总字数计算功能
系统必须能够计算标题、正文和话题名称的总字符数，用于精华帖评估。

#### 场景：计算汇总字数
- **当** 进行精华帖评估时
- **那么** 系统必须计算 `title.length + content.length + sum(topic.name.length)` 作为汇总字数
