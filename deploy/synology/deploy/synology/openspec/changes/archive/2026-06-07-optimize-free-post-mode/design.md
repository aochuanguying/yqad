## 上下文

当前自由发帖模式在 `auto-post.ts` 的 `postFreeStyle()` 方法中实现了两种模式：互联网参考模式和社区分析模式。互联网参考模式通过 `internet-reference-service.ts` 从小红书等平台获取参考帖子，但存在以下问题：

1. **论坛参考依赖**：代码中存在论坛参考逻辑，增加了系统复杂性
2. **图片水印问题**：互联网参考图片直接使用，未处理水印，影响发帖质量
3. **精华帖标准不统一**：自由发帖模式下的质量标准不够明确

系统运行在内网环境，使用 TypeScript/Node.js 技术栈，已有 AI 内容生成服务和图片处理基础。

## 目标 / 非目标

**目标：**
- 简化自由发帖模式，移除论坛参考逻辑
- 实现图片去水印功能，提升图片质量
- 确保自由发帖也符合精华帖标准
- 保持互联网参考模式的核心功能

**非目标：**
- 不改变主题发帖模式的逻辑
- 不修改社区分析模式的实现
- 不改变现有的 AI 内容生成流程
- 不涉及 Web 管理界面的重大变更

## 决策

### 决策 1：移除论坛参考逻辑

**决策**：完全移除 `auto-post.ts` 中的论坛参考相关代码，只保留互联网参考模式和社区分析模式。

**理由**：
- 论坛参考增加了外部依赖和系统复杂性
- 互联网参考模式已能提供足够的素材来源
- 简化代码逻辑，降低维护成本

**考虑过的替代方案**：
- 保留论坛参考作为备选 →  rejected，因为会增加维护负担且收益有限

### 决策 2：图片去水印实现方案

**决策**：使用 AI 服务实现图片去水印功能，而不是传统的图像处理库。

**理由**：
- AI 去水印效果更好，能处理复杂水印类型
- 无需引入额外的图像处理依赖（如 OpenCV）
- 与现有 AI 服务架构一致，易于集成
- 支持多种水印类型（文字水印、logo 水印等）

**实现方式**：
- 创建 `watermark-removal-service.ts` 服务模块
- 使用 AI 模型的图像理解能力识别和移除水印
- 支持批量处理和失败降级策略

**考虑过的替代方案**：
- 使用 OpenCV 等传统图像处理库 → rejected，因为对复杂水印效果不佳
- 使用第三方去水印 API → rejected，因为涉及外部依赖和隐私问题

### 决策 3：精华帖标准强化

**决策**：在自由发帖模式中也应用精华帖标准，包括最小内容字符数、最小图片数量等。

**理由**：
- 确保所有发帖质量一致
- 提升账号整体内容质量
- 符合平台精华帖推荐机制

**实现方式**：
- 在 `postFreeStyle()` 中复用精华帖配置
- 内容生成时应用相同的质量检查
- 图片选取时使用相同的标准

## 风险 / 权衡

**风险 1：AI 去水印效果不稳定**
→ **缓解措施**：实现失败降级策略，去水印失败时使用原图并记录警告日志

**风险 2：去水印增加处理时间**
→ **缓解措施**：异步处理图片，设置合理的超时时间，支持跳过处理选项

**风险 3：移除论坛参考可能导致素材单一**
→ **缓解措施**：优化互联网参考的关键词策略，确保素材多样性

**风险 4：破坏现有功能**
→ **缓解措施**：保留社区分析模式作为回退方案，充分测试互联网参考模式
