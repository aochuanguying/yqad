## 新增需求

### 需求: 系统必须自动生成并发布优质帖子
系统必须定时基于社区热点话题生成高质量帖子并发布到社区。

#### 场景: 基于热点生成帖子
- **当** 自动发帖任务触发
- **那么** 系统必须从内容分析结果中选取热点话题，调用AI生成包含标题和正文的帖子

#### 场景: 发布帖子
- **当** 帖子内容生成完毕
- **那么** 系统必须调用发帖API将帖子发布到社区

#### 场景: 每日发帖数量可配置
- **当** 执行每日发帖任务
- **那么** 系统必须根据配置的每日发帖数量上限（默认1篇）执行发帖

### 需求: 自动发帖必须确保内容多样性
系统必须避免连续发布相似主题的帖子，确保内容多样性。

#### 场景: 避免话题重复
- **当** 生成新帖子时
- **那么** 系统必须检查最近7天内已发布帖子的话题，避免选取相同或高度相似的话题

#### 场景: 记录发帖历史
- **当** 帖子成功发布后
- **那么** 系统必须记录帖子标题、话题关键词和发布时间到本地历史记录
