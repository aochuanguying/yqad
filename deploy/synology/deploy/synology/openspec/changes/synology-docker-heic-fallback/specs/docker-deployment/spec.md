## 修改需求

### 需求:Dockerfile 配置
主应用的 Docker 打包文件必须面向群晖 `DS218+ / NAS218+` 的 `linux/amd64` 运行环境，并必须提供能够支撑 HEIC/HEIF 兜底转换的系统依赖。

#### 场景:构建群晖可运行镜像
- **当** 为主应用构建 Docker 镜像
- **那么** 打包文件必须使用 Debian 系官方 Node LTS 镜像
- **并且** 打包文件必须明确支持 `linux/amd64` 目标平台
- **并且** 打包文件必须安装 `sharp` 运行所需的图像依赖与 HEIC/HEIF 兜底转换工具

#### 场景:提供 HEIC/HEIF 兜底工具
- **当** 镜像完成构建并启动容器
- **那么** 容器内必须可调用 HEIC/HEIF 转换命令行工具
- **并且** 该工具必须可被素材处理模块用于方案 B 的兜底转换
- **并且** 文档必须给出验证该工具可用的方式

### 需求:Docker Compose 配置
项目必须提供适用于群晖 Container Manager 的 `docker-compose.yml`，并必须覆盖数据持久化、端口暴露和基础运行参数。

#### 场景:挂载持久化目录
- **当** 用户按推荐方式启动容器
- **那么** `config`、`data`、`logs` 目录必须分别挂载到容器中的对应路径
- **并且** 容器重启或重建后，token、素材处理结果和日志必须保持可用

#### 场景:对外暴露 Web 服务
- **当** 用户部署容器后需要从浏览器或手机访问服务
- **那么** Compose 配置必须映射 Web 服务端口
- **并且** 配置方式必须允许用户设置 NAS 对外可访问地址
- **并且** 部署说明必须要求将 `web.baseUrl` 设置为可公开访问的 NAS 地址

#### 场景:在 Apple Silicon 本机构建并群晖运行
- **当** 用户在 Mac 上构建镜像并部署到群晖
- **那么** Compose 或部署说明必须明确要求使用 `linux/amd64` 构建产物
- **并且** 禁止默认依赖宿主机构架自动推断目标平台

### 需求:部署文档
项目必须提供适用于群晖 DSM 7 Container Manager 的部署文档，并必须覆盖首次部署、验证、故障排查和回滚。

#### 场景:首次部署
- **当** 用户按照文档在群晖上完成构建或导入镜像、配置卷挂载并启动容器
- **那么** 用户必须能够在 30 分钟内完成主应用部署
- **并且** 文档必须包含构建镜像、导入或推送镜像、编辑配置、启动服务和访问 Web 页面的完整步骤

#### 场景:验证图片访问链路
- **当** 用户完成部署后执行验证步骤
- **那么** 文档必须指导用户验证 `web.baseUrl` 是否生效
- **并且** 文档必须指导用户验证 `/images` 链路可被外部设备访问

#### 场景:排查 HEIC 处理问题
- **当** 群晖容器中的 HEIC/HEIF 图片转换失败
- **那么** 文档必须提供检查容器依赖、查看日志和验证兜底工具的故障排查步骤
- **并且** 文档必须提供最小可执行的回滚方案
